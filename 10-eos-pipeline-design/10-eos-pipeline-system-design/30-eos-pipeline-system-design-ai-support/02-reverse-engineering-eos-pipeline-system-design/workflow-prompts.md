# 系统设计逆向工程：提示词模板集
**System Design Reverse Engineering: Workflow Prompt Templates**

**用途**：AI 按 8 步流程依次执行逆向推导，每个步骤使用对应的提示词模板。AI 将提示词中的 `[占位符]` 替换为实际内容后执行。

## 依据B说明

本提示词模板集适用于设计线（系统设计流水线）的逆向工程场景，默认演示"软件产品"类型的逆向推导过程。

**适用场景**：
- 对设计线（流水线）自身软件实现的逆向推导
- 对EOS（企业运营体系）的逆向推导
- 对其他软件产品的逆向推导

**使用时注意**：
- 根据目标产品的类型选择 PA 分类体系：软件产品 → `前端/后端/数据/基础设施`；方法论/工具产品 → `规范与准则/设计指南/AI辅助文档/任务定义`
- 所有提示词模板中的 `[占位符]` 在执行时替换为实际内容
- 模板中默认使用软件产品分类，适用于对 EOS或流水线软件实现的逆向推导

## 跨步骤上下文管理说明

### 上下文传递方式

8步逆向推导流程中，每步逆向推导的结果是下一步的输入。AI 在执行每一步前，应确保完整继承了上一步的关键决策：

| 继承内容 | 说明 | 保存位置 |
|---------|------|---------|
| 产物分析结果 | 产物梳理、PA 组件识别 | 第1-2步输出 |
| 架构定义 | PA 组件划分、SysReq 架构、BA IPO | 各步的产品数据文件 |
| 反向推导评级 | 推导结果的 AAA/B/C 评级汇总 | 上一步的决策记录 |
| 制品差异记录 | 设计制品之间的不一致项 | 差异清单 |
| 待确认项 | 上一步标记的 ⚠️ 项 | 汇总清单 |

### 决策记录机制

每步完成后，AI 将本步的关键决策记录在**本步决策记录模板**中，供后续步骤引用。

### 当需要回溯修正

如果在第 N 步发现问题需要修改第 M 步（M < N）的产出：
1. 评估影响范围（见[AI决策框架 -> 跨步骤影响范围表](../02-ai-decision-framework-eos-pipeline.md#72-跨步骤影响范围表)）
2. 逆向工程的影响范围更广，因为每步推导依赖前一步的结果
3. 修改后检查下游所有推导链路是否需要同步调整
4. 标注变更记录

## 前置准备：第1步——现有产物梳理与架构分析

> **前置说明**：本步骤仅在 PA 文档不存在或质量严重不足时执行。如果 PA 已存在且可用，可直接从第3步开始。

### 目标
全面梳理现有产物，理解产品的目录结构和文件组织方式，建立架构级别的全局视图。

### 上下文继承清单

```
从哪里来：
  - 直接输入：项目目录结构、现有设计文档/指南/AI辅助文档/产品数据
  - 参考文档：[产品上下文](../01-product-context-eos-pipeline.md)，了解设计线的背景
  - 涉及规范：产品架构规范 §2

预期产出：
  1. 产物总览清单（产物分类 + 架构层级映射）
  2. 覆盖盲区标记
  3. 初版架构评估报告

本步决策对后续的影响：
  - 产物分类结果直接影响第2步 PA 组件识别
  - 覆盖盲区标记影响第7步验证的检查范围
  - 架构层级映射为后续所有逆向推导提供分类基础
```

### 输入
- 项目目录结构
- 现有设计文档/指南/AI辅助文档/产品数据
- 产品代码和架构图纸（可选，仅在需要从代码重建PA时使用）

### 涉及规范
- [通用设计准则 §五.7 PA节点属性](../../../../00-general/10-general-system-design-standards/01-general-system-design-standards.md#57-pa节点属性)

### 决策框架参考

| 决策场景 | 处理方法 | 参考章节 |
|---------|---------|---------|
| 产物分析深度 | 按 L1/L2/L3 分级判断 | [8.1 代码分析深度判断](../02-ai-decision-framework-eos-pipeline.md#81-代码分析深度判断) |
| 组件识别粒度 | 可独立部署、1-2周开发周期 | 按规范执行 |

## 审查修复流程

### 审查输入
AI 完成指定步骤后，输出产物供 rule-conformance-review 审查。审查报告包含：
- 通过/未通过的项目清单
- 问题定位（涉及的具体文件、章节、编号）
- 问题类型（结构性/内容性/格式性/制品一致性）

### AI 修复流程
收到审查报告后，AI 按以下优先级处理：

1. **格式性问题**（编号错误、格式不规范）→ 直接修复，不需要人工确认
2. **内容缺失**（某节为空、映射不完整）→ 补充内容，标注修复说明
3. **制品一致性问题**（文档与PA不符）→ 以PA为准修正文档，标注"从PA修正"
4. **结构性问题**（粒度不匹配、分类错误）→ 提出修复方案，标注 ⚠️ 待确认
5. **跨步骤一致性问题** → 参考[AI决策框架 -> 跨步骤影响范围表](../02-ai-decision-framework-eos-pipeline.md#72-跨步骤影响范围表)进行影响范围评估

### 影响范围评估
| 问题定位 | 影响范围 | 修复策略 |
|---------|---------|---------|
| 前置准备偏差 | PA + SysReq + BA + SR + OR 全部受波及 | 修正前置分析后从第2步重新推导 |
| 第2步 PA 组件划分 | SysReq 9 级活动 + 后续所有映射 | 调整 PA 后重新映射 SysReq |
| 第3步 SysReq 推导 | BA IPO + SR 推导基础 | 修正后从第4步重新推导 |
| 第4步 BA IPO 推导 | SR 架构定义 + 映射 | 修正后从第5步重新推导 |
| 第5步 SR 推导 | OR 聚合基础 | 修正后从第6步重新推导 |
| 第6步 OR 推导 | 追溯矩阵（影响较小） | 仅修正追溯矩阵 |
| 第7步发现问题 | 取决于问题定位 | 定位到对应步骤后回溯修复 |
