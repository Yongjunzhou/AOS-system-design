# AI支持文档

**文档版本**：v1.0  
**创建日期**：2026-05-14  
**目的**：为系统设计流水线的AI深度参与提供支持文档

---

## 📌 文档概述

本目录包含为AI在系统设计流水线开发中的深度参与而准备的支持文档。这些文档定义了AI可以进行自动化处理的规则、标准和模式。

---

## 📚 支持文档清单

### 1. 业务模式库
**位置**：`07-shared-assets/patterns/business-patterns.md`

**用途**：为AI提供可复用的业务模式，支持AI进行模式识别和匹配

**包含内容**：
- 10个常见业务模式（工单处理、流程管理、计划管理等）
- 每个模式的定义、适用场景、IPO结构、变体、性能指标
- 模式选择决策树

**AI使用场景**：
- 在业务架构设计阶段进行模式识别
- 建议使用已验证的业务模式
- 支持业务组件的去重

---

### 2. 质量检查清单
**位置**：`07-shared-assets/quality-standards/quality-checklist.md`

**用途**：为AI提供质量检查标准，支持AI进行自动化质量验证

**包含内容**：
- 6个项目阶段的质量检查清单
- 每个阶段的具体检查项
- 质量指标和标准

**AI使用场景**：
- 在每个阶段进行自动化质量检查
- 验证映射覆盖率、1:1约束、N:1支持等指标
- 生成质量报告

---

### 3. AI Skill定义
**位置**：`07-shared-assets/ai-skills/skill-definitions.md`

**用途**：定义AI可以进行的自动化处理任务

**包含内容**：
- 9个AI Skill的详细定义
- 每个Skill的输入、处理规则、输出、验证标准
- Skill调用顺序

**AI使用场景**：
- 需求规范化、分解、冲突检测、重复检测
- 业务模式匹配、架构模式匹配
- SR→BA、BA→SysReq、SysReq→PA映射

---

### 4. 映射规则文档
**位置**：`07-shared-assets/mapping-rules/mapping-rules.md`

**用途**：定义映射约束和验证规则，支持AI进行映射验证

**包含内容**：
- 1:1分配约束定义和验证方法
- N:1支持规则（平均2-3条，最多5条）
- 双向追溯规则
- 符合性分析规则
- 映射验证流程和质量指标

**AI使用场景**：
- 验证映射关系的正确性
- 检查1:1约束和N:1支持
- 验证双向追溯的完整性
- 生成映射验证报告

## 🎯 AI支持体系架构

```
系统设计流水线开发
    ↓
需要AI支持的环节：
    ├─ 需求规范化 → AI Skill 1
    ├─ 需求分解 → AI Skill 2
    ├─ 冲突检测 → AI Skill 3
    ├─ 重复检测 → AI Skill 4
    ├─ 业务模式匹配 → AI Skill 5 + 业务模式库
    ├─ SR→BA映射 → AI Skill 7 + 映射规则
    ├─ BA→SysReq映射 → AI Skill 8 + 映射规则
    ├─ 架构模式匹配 → AI Skill 6
    ├─ SysReq→PA映射 → AI Skill 9 + 映射规则
    └─ 质量检查 → 质量检查清单 + 映射规则
```

## 📖 使用指南

### 对于流水线开发者

1. **理解AI能力**：阅读本README和各个支持文档
2. **定义AI任务**：根据AI Skill定义，确定哪些环节可以由AI处理
3. **设置AI规则**：根据映射规则、质量检查清单等，配置AI的处理规则
4. **监控AI质量**：使用质量检查清单验证AI的处理结果

### 对于AI系统

1. **加载支持文档**：加载业务模式库、AI Skill定义、映射规则、质量检查清单
2. **执行自动化处理**：根据AI Skill定义执行相应的处理任务
3. **进行质量检查**：根据质量检查清单和映射规则进行验证
4. **生成报告**：生成处理结果和质量报告
