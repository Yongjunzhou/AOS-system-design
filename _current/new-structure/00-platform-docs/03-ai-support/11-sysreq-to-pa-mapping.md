# SysReq 详细分解及 PA 映射技能
**Skill: SysReq Detailed Decomposition & PA Mapping**

**编号**：11
**所属步骤**：第 5 步 — 系统需求详细定义及产品架构定义
**引用准则**：系统设计准则 v2.3 — 系统需求架构层级结构（0-9 级）、第 5 层产品架构、1:1 分配约束
**引用指南**：瀑布式设计指南 v2.0 第 5 步

---

## 用途

将系统需求功能部分 5 级节点分解到 6-9 级（场景活动），然后将 9 级节点分配到产品架构末级节点。

## 输入

- SysReq-F 架构末级节点（5 级，第 3 步产出）
- SysReq-NFR 架构定义（第 4 步产出，作为约束）
- 相关方需求功能部分详细定义（作为符合性分析参考）

## 处理规则

### 一、SysReq-F 详细分解（5 级 → 9 级）

#### 层级定义

| 层级 | 名称 | 说明 | 分解方式 |
|------|------|------|---------|
| 5 级 | 组件实例 | 承接 BA IPO 的系统功能组件 | 入口：来自 BA 去重后的业务组件 |
| 6 级 | 功能模块 | 组件实例内的功能模块划分 | 按 UI 标签页（Tab）划分，一个 5 级可有 1-3 个功能模块 |
| 7 级 | 系统用例 | 功能模块上提供的具体功能 | 按功能按钮或右键菜单划分 |
| 8 级 | 用例场景 | 系统用例的具体业务场景变化 | 按业务规则、权限、数据状态等条件分解 |
| 9 级 | 场景活动 | 用例场景的具体活动步骤 | 末级节点：映射到 PA 末级节点 |

#### 分解示例

```
SysReq-F-001：订单管理组件（5 级）
├─ 6 级：订单列表模块
│  ├─ 7 级：查询订单
│  │  ├─ 8 级：按条件查询 → PA-001
│  │  └─ 8 级：高级搜索 → PA-001
│  └─ 7 级：导出订单
│     └─ 8 级：导出订单列表 → PA-002
└─ 6 级：订单详情模块
   └─ 7 级：查看订单详情
      └─ 8 级：查看订单完整信息 → PA-003
```

### 二、9 级到 PA 末级的映射

#### 1:1 分配约束
每条 SysReq 9 级（场景活动）只能分配到唯一的 PA 末级节点。

#### PA 末级节点分类
- **前端组件**：UI 界面、交互逻辑
- **后端组件**：业务逻辑、数据处理、API

#### 分配优先级
- **优先**：分配到现有 PA 末级节点（功能相关，技术实现满足 NFR 约束）
- **其次**：创建新的 PA 末级节点

### 三、NFR 约束验证

每个 SysReq-NFR 约束应在对应的 PA 节点中体现：
- 性能约束 → 标注响应时间、吞吐量要求
- 安全约束 → 标注认证、授权、加密要求
- 可用性约束 → 标注容错、灾备要求

## 输出格式

### SysReq 详细定义（6-9 级）

```
SysReq-F-001：[5 级组件实例名称]
├─ 6 级：[功能模块名称]
│  ├─ 7 级：[系统用例名称]
│  │  ├─ 8 级：[用例场景名称]
│  │  │  └─ 9 级：[场景活动]
│  │  │     ├─ 描述：[详细说明]
│  │  │     ├─ 验收标准：[验收条件]
│  │  │     └─ 分配到：[PA-001]
```

### PA 末级节点

```
PA-001：[组件名称]
├─ 定义：[组件定义和职责]
├─ 承接的系统需求：SysReq-F-001.x.x.x
├─ 符合性分析：[如何满足所承接的需求]
├─ 实现细节：[技术栈、关键模块、数据存储]
├─ 受约束的非功能需求：SysReq-NFR-001, SysReq-NFR-002
└─ 性能指标：[响应时间、吞吐量等]
```

## 质量检查

- [ ] 所有 5 级节点完全分解到 6-9 级
- [ ] 每个 9 级场景活动有明确的验收标准
- [ ] 1:1 分配约束满足率 100%（9 级到 PA 末级）
- [ ] 前后端组件划分清晰
- [ ] 每个 PA 组件包含实现细节和性能指标
- [ ] 非功能约束已体现到 PA 节点
