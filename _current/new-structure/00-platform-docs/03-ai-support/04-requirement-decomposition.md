# 需求分解技能
**Skill: Requirement Decomposition**

**编号**：04
**所属步骤**：第 1 步 — 原始需求分析及相关方需求架构定义
**引用准则**：系统设计准则 v2.3 — 第 1 层、1:1 分配约束
**引用指南**：瀑布式设计指南 v2.0 第 1 步

---

## 用途

将复杂需求分解为多个原子性的末级需求，每条末级需求只需映射到相关方需求架构的一个末级节点。

## 输入

- 规范化后的需求列表（03-requirement-normalization.md 的输出）

## 处理规则

### 分解原则
1. 每条分解后的末级需求应是**原子性的、不可再分的**
2. 每条末级需求只需映射到相关方需求架构的**一个**末级节点（1:1 约束的逆推）
3. 分解后不丢失原始需求的语义完整性
4. 末级需求的粒度应与同层其他末级需求一致

### 触发分解的条件

遇到以下情况时需求需要分解：
- 需求描述中包含 "和/并且/同时" 连接多个独立功能
- 需求涉及多个独立的操作或流程
- 需求涉及多个角色或相关方
- 需求需要映射到多个相关方需求架构末级节点

### 分解示例

```
原始：系统应支持订单全生命周期管理
├─ 末级 1.1：系统应支持订单创建
├─ 末级 1.2：系统应支持订单修改
├─ 末级 1.3：系统应支持订单取消
└─ 末级 1.4：系统应支持订单查询
```

## 输出格式

```
原始需求：[原始需求描述]

末级分解：
- 末级需求 1.1：[描述] → [分配到的相关方需求架构末级]
- 末级需求 1.2：[描述] → [分配到的相关方需求架构末级]
- 末级需求 1.3：[描述] → [分配到的相关方需求架构末级]

分解决策记录：
- 条目 1.1：[为什么这样分解]
- 条目 1.2：[为什么这样分解]
```

## 质量检查

- [ ] 每条末级需求是原子性的、不可再分的
- [ ] 所有末级需求之和 = 原始需求语义（无遗漏无冗余）
- [ ] 每条末级需求只需映射到一个相关方需求架构末级节点
- [ ] 分解粒度和同层其他需求一致
