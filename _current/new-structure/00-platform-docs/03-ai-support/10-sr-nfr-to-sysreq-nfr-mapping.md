# SR-NFR → SysReq-NFR 映射技能
**Skill: SR-NFR to SysReq-NFR Mapping**

**编号**：10
**所属步骤**：第 4 步 — 相关方需求非功能部分详细定义及系统需求非功能部分架构定义
**引用准则**：系统设计准则 v2.3 — 非功能需求架构层级结构（0-6 级）
**引用指南**：瀑布式设计指南 v2.0 第 4 步

---

## 用途

将相关方非功能需求（SR-NFR）映射到系统非功能需求（SysReq-NFR），建立平行架构体系。

## 输入

- 相关方需求架构定义中的 SR-NFR 架构末级节点
- 非功能需求分类体系

## 处理规则

### 1. 非功能需求的平行架构
非功能需求**不经过**业务架构层，从 SR-NFR 直接映射到 SysReq-NFR。

### 2. SR-NFR 详细定义分解
对每个 SR-NFR 架构末级节点进行详细分解：
- 每条条目包含：描述、量化指标、验证方法
- 量化指标必须具体可测量（如：P99 < 500ms，可用性 > 99.9%）

### 3. 三大类分类体系
| 大类 | 子类 |
|------|------|
| 质量特性需求 | 性能、可靠、安全、可用、可维护 |
| 环境适应性需求 | 兼容、扩展、移植、集成适应 |
| 可实现性需求 | 可生产、可测试、可交付、可部署、可操作、可运维、可退役 |

### 4. N:1 映射
多条 SR-NFR 详细定义条目可映射到同一个 SysReq-NFR 5 级节点。每条 SR-NFR 条目只能分配到唯一的 SysReq-NFR 5 级节点（1:1 约束）。

## 输出格式

### SR-NFR 详细定义

```
SR-NFR-001.1：[分解条目]
├─ 描述：[详细说明]
├─ 量化指标：[具体的性能指标]
├─ 验证方法：[如何验证]
└─ 分配到：[SysReq-NFR-001]
```

### SysReq-NFR 架构定义

```
SysReq-NFR-001：[系统非功能需求名称]
├─ 定义：[非功能需求的定义]
├─ 类型：[性能/可靠性/安全性/...]
├─ 承接的 SR-NFR：SR-NFR-001.1, SR-NFR-001.2
├─ 符合性分析：[如何满足所承接的需求]
└─ 为下层提供的设计指导：[对产品架构的约束]
```

## 质量检查

- [ ] 所有 SR-NFR 有明确的量化指标
- [ ] 非功能需求分类正确（三大类匹配）
- [ ] N:1 映射关系完整
- [ ] 与功能分支的接口（SysReq-F）无重叠
