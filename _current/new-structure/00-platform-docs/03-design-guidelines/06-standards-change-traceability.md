## 📋 准则变更流程

### 第1步：识别变更（1小时）

1. 确定准则的具体变更内容
2. 分类变更类型（映射规则、层级结构、约束条件、流程、质量指标）
3. 评估变更的影响范围

**输出**：变更影响分析报告

### 第2步：更新指南（2-4小时）

1. 根据变更类型，确定需要更新的指南部分
2. 逐个更新四个指南中的相关内容
3. 更新示例和最佳实践
4. 更新验证清单和质量指标

**输出**：更新后的四个指南

### 第3步：一致性检查（1-2小时）

1. 检查四个指南中的相关内容是否一致
2. 检查示例是否仍然有效
3. 检查验证清单是否完整

**输出**：一致性检查报告

### 第4步：版本更新（30分钟）

1. 更新准则版本号
2. 更新指南版本号
3. 更新指南中的"基于准则版本"标注
4. 添加变更日志

**输出**：版本更新完成

### 第5步：提交和发布（30分钟）

1. 提交所有变更到 git
2. 更新文档索引
3. 通知相关团队

**输出**：变更发布完成

## 🚀 最佳实践

1. **定期检查**
   - 每次准则变更后，立即进行一致性检查
   - 每个季度进行一次全面的一致性审查

2. **版本管理**
   - 准则版本和指南版本保持同步
   - 在指南中明确标注所基于的准则版本

3. **变更追溯**
   - 记录每次变更的内容和影响范围
   - 建立变更日志，便于后续查询

4. **团队沟通**
   - 准则变更时，及时通知使用指南的团队
   - 提供变更说明和迁移指导
