## 📋 核心约束速记

### 约束1：1:1 分配约束
```
每条上层需求 → 唯一一个下层方案条目

检查方法：
✓ 需求是否分配了？
✓ 是否只分配到一个条目？
✓ 是否有多个条目都承接了这条需求？

常见问题：
❌ 一条需求分配到多个条目 → 需求分解不足
❌ 一条需求无法分配 → 方案设计不完整
```

### 约束2：N:1 承接支持
```
每个下层方案条目 ← 多条上层需求

合理范围：
- 最少：1条需求
- 最多：4-5条需求
- 平均：2-3条需求

检查方法：
✓ 承接的需求数量是否合理？
✓ 承接的需求之间是否有逻辑关联？
```

### 约束3：双向追溯
```
需求 ↔ 方案（通过超链接）

检查方法：
✓ 每条需求都标注了分配的方案条目？
✓ 每个方案条目都标注了承接的需求？
✓ 所有超链接都有效？
```

## 🏗️ 五层结构速记

```
第1层：原始需求及分析
  ↓ 分解到末级需求
第2层：相关方需求（原始需求的方案）
  ├─ 功能部分 ↓
  │  第3层：业务架构（相关方需求功能部分的方案）
  │    ↓
  │  第4层：系统需求-功能部分（业务架构的方案）
  │    ↓
  │  第5层：产品架构（系统需求的方案）
  │
  └─ 非功能部分 ↓
     第4层：系统需求-非功能部分（相关方需求非功能部分的方案）
       ↓
     第5层：产品架构（系统需求的方案）
```

### 每层的角色

| 层 | 名称 | 角色 | 输出 |
|----|------|------|------|
| 1 | 原始需求 | 需求 | 需求清单 |
| 2 | 相关方需求 | 方案+需求 | 条目化需求+映射说明 |
| 3 | 业务架构 | 方案+需求 | 条目化方案+映射说明 |
| 4 | 系统需求 | 方案+需求 | 条目化需求+映射说明 |
| 5 | 产品架构 | 方案 | 条目化方案+实现细节 |

## ✅ 操作检查清单（精简版）

### 需求分解
- [ ] 末级需求是原子性的
- [ ] 粒度合理（平均2-3条）
- [ ] 每条都有定义和验收标准
- [ ] 每条都分配到下层方案

### 需求映射
- [ ] 1:1约束满足（每条需求唯一分配）
- [ ] N:1支持满足（每个方案承接2-4条需求）
- [ ] 映射关系符合业务逻辑

### 符合性分析
- [ ] 每个方案都有符合性分析
- [ ] 说明了如何满足所承接的需求
- [ ] 逻辑清晰、可验证

### 双向追溯
- [ ] 需求标注了分配的方案
- [ ] 方案标注了承接的需求
- [ ] 所有超链接有效

### 文档规范
- [ ] 结构清晰一致
- [ ] 包含两部分（条目化+映射说明）
- [ ] 版本号正确更新

## 🚀 场景1：反向工程（快速流程）

```
第1步：代码和图纸分析（1-2天）
  ├─ 分析架构图纸
  ├─ 分析代码结构
  └─ 提取功能清单

第2步：产品架构文档编写（2-3天）
  └─ 编写PA条目

第3步：系统需求反向推导（3-5天）
  └─ 从PA推导SysReq

第4步：业务架构反向推导（3-5天）
  └─ 从SysReq推导BA

第5步：相关方需求反向推导（2-3天）
  └─ 从BA推导SR

第6步：原始需求反向推导（1-2天）
  └─ 从SR推导原始需求

第7步：验证（2-3天）
  ├─ 1:1约束验证
  ├─ N:1支持验证
  ├─ 双向追溯验证
  └─ 符合性分析验证

第8步：文档完善和发布（2-3天）
  └─ 发布最终版本

总时间：16-26天（约3-4周）
```

## 🔍 常见问题速答

### Q1：末级需求应该分解到什么程度？
**A**：
- ✅ 原子性的、不可再分的
- ✅ 能够分配到下层方案的某个条目
- ✅ 粒度合理（平均分解数2-3条）

### Q2：一个方案条目应该承接多少条需求？
**A**：
- 最少：1条
- 最多：4-5条
- 平均：2-3条

### Q3：如何判断映射是否合理？
**A**：
- ✅ 1:1约束满足（每条需求唯一分配）
- ✅ N:1支持满足（每个方案承接2-4条需求）
- ✅ 承接的需求之间有逻辑关联

### Q4：如何编写有效的符合性分析？
**A**：
1. 列出所有承接的需求
2. 说明如何满足每条需求
3. 说明验证方法
4. 说明是否有限制或特殊情况

### Q5：如何建立有效的双向追溯？
**A**：
1. 在方案条目中标注承接的需求
2. 在需求中标注分配的方案条目
3. 使用超链接实现快速导航
4. 定期验证所有超链接的有效性

| 指标 | 优秀 | 良好 | 需要改进 |
|------|------|------|---------|
| 映射覆盖率 | 100% | 95-99% | <95% |

## 🎓 最佳实践速记

### 场景1：反向工程

- ✅ 与代码实现进行对比
- ✅ 进行用户访谈补充需求
- ✅ 定期进行验证和审查

- ✅ 确保原始需求清晰完整
- ✅ 多个角色的协作
- ✅ 每个阶段都进行审查
- ✅ 使用需求管理工具

### 场景3：增量需求
- ✅ 优先处理高优先级需求
- ✅ 分阶段处理，避免一次性大量变更
- ✅ 优先映射到现有方案
- ✅ 及时沟通影响范围
