## 🔄 重新生成步骤

### 第1步：分析原始需求和相关方需求（1天）

**目标**：理解当前的需求结构，为重新设计做准备

**具体操作**：
1. 分析76个末级需求的分布
2. 分析21项相关方需求的分类
3. 识别相关方需求之间的关系
4. 规划相关方需求架构末级节点

**输出**：
- 相关方需求架构末级节点清单（预计5-10个）
- 相关方需求详细设计末级节点清单（预计20-40个）
- 映射关系分析

### 第2步：重新设计相关方需求（2-3天）

**目标**：按照v2.0准则重新组织相关方需求

**具体操作**：
1. 定义相关方需求架构末级节点
   - 按照相关方或功能域进行分类
   - 每个架构末级节点应该能够承接多个原始需求末级
   
2. 定义相关方需求详细设计末级节点
   - 对每个架构末级节点进行分解
   - 每个详细设计末级节点应该只映射到一个业务架构末级节点
   
3. 建立映射关系
   - 原始需求末级 → 相关方需求架构末级（N:1）
   - 相关方需求详细设计末级 → 业务架构末级（1:1约束）

**输出**：
- 更新后的相关方需求文档（v2.0格式）
- 相关方需求架构末级节点清单
- 相关方需求详细设计末级节点清单

### 第3步：设计业务架构（3-5天）

**目标**：创建业务架构文档，定义IPO清单

**具体操作**：
1. 识别所有IPO（业务组件）
   - 分析相关方需求详细设计末级节点
   - 识别每个需求对应的业务流程
   - 定义IPO的Input、Process、Output
   
2. 进行IPO去重
   - 检查是否有重复的IPO
   - 复用 → 改进 → 新增的优先级
   
3. 定义业务架构详细设计末级节点
   - 对每个IPO的I、P、O进行详细说明
   - 确保每个详细设计末级节点只映射到一个系统需求架构末级节点

**输出**：
- 业务架构文档（新建 07-business-architecture.md）
- IPO清单（去重后）
- 业务架构详细设计末级节点清单

### 第4步：重新设计系统需求（2-3天）

**目标**：按照v2.0准则重新组织系统需求

**具体操作**：
1. 定义系统需求架构末级节点
   - 功能需求架构：基于IPO去重清单
   - 非功能需求架构：按照需求类别分类
   
2. 定义系统需求详细设计末级节点
   - 对每个架构末级节点进行分解
   - 每个详细设计末级节点应该只映射到一个产品架构末级节点
   
3. 建立映射关系
   - 业务架构详细设计末级 → 系统需求架构末级（N:1）
   - 系统需求详细设计末级 → 产品架构末级（1:1约束）

**输出**：
- 更新后的系统需求文档（v2.0格式）
- 系统需求架构末级节点清单
- 系统需求详细设计末级节点清单

### 第5步：设计产品架构（3-5天）

**目标**：创建产品架构文档，定义前后端组件

**具体操作**：
1. 识别所有产品组件
   - 分析系统需求详细设计末级节点
   - 识别前后端组件划分
   - 定义每个组件的功能和职责
   
2. 定义产品架构详细设计末级节点
   - 对每个组件的实现细节进行说明
   - 包括技术栈、性能指标、约束条件等
   
3. 建立映射关系
   - 系统需求详细设计末级 → 产品架构末级（N:1）

**输出**：
- 产品架构文档（新建 08-product-architecture.md）
- 产品架构末级节点清单
- 产品架构详细设计末级节点清单

### 第6步：验证映射关系（2-3天）

**目标**：验证所有映射关系符合v2.0准则

**具体操作**：
1. 验证1:1约束
   - 每条相关方需求详细设计末级只映射到一个业务架构末级
   - 每条业务架构详细设计末级只映射到一个系统需求架构末级
   - 每条系统需求详细设计末级只映射到一个产品架构末级
   
2. 验证N:1支持
   - 每个业务架构末级承接多个相关方需求详细设计末级
   - 每个系统需求架构末级承接多个业务架构详细设计末级
   - 每个产品架构末级承接多个系统需求详细设计末级
   
3. 验证映射覆盖率
   - 所有相关方需求详细设计末级都被映射
   - 所有业务架构详细设计末级都被映射
   - 所有系统需求详细设计末级都被映射

**输出**：
- 验证报告
- 问题清单和改进建议

### 第7步：更新需求映射矩阵（1-2天）

**目标**：生成完整的需求映射矩阵

**具体操作**：
1. 更新原始需求 → 相关方需求映射
2. 更新相关方需求 → 业务架构映射
3. 更新业务架构 → 系统需求映射
4. 更新系统需求 → 产品架构映射
5. 生成完整的端到端追溯矩阵

**输出**：
- 更新后的需求映射矩阵（06-requirements-mapping-matrix.md）
- 端到端追溯矩阵

### 第8步：文档完善和发布（1-2天）

**目标**：完善所有文档，发布最终版本

**具体操作**：
1. 补充缺失的内容
2. 修正发现的问题
3. 优化文档结构和表述
4. 更新版本号和日期
5. 提交到git

**输出**：
- 完整的系统设计文档集（5层）
- 所有文档版本号更新为v2.0
- git提交
