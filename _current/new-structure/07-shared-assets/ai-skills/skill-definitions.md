# AI Skill定义

**文档版本**：v1.0  
**创建日期**：2026-05-14  
**目的**：定义系统设计流水线中9个AI Skill的详细规范

---

## 📌 文档概述

本文档定义了系统设计流水线中9个AI Skill的详细规范，包括输入、处理规则、输出、验证标准等。

---

## 🎯 Skill 1：需求规范化

**Skill名称**：Requirement Normalization

**目的**：对原始需求进行规范化处理

**输入**：原始需求文本、术语表、规范化规则

**处理规则**：
1. 格式转换：统一需求格式
2. 术语标准化：使用标准术语
3. 结构规范化：确保必要部分完整
4. 清晰性检查：检查表述清晰度

**输出**：规范化需求、规范化报告、质量评分

**验证标准**：
- 所有需求符合规范化格式
- 术语标准化准确率 > 95%
- 需求清晰、无歧义

---

## 🎯 Skill 2：需求分解

**Skill名称**：Requirement Decomposition

**目的**：对原始需求进行分解

**输入**：规范化需求、分解规则、分解深度标准

**处理规则**：
1. 需求分析：分析复杂度
2. 分解策略：选择分解方式
3. 末级需求定义：确保原子性
4. 粒度检查：平均分解数2-3条

**输出**：末级需求清单、分解树、质量评分

**验证标准**：
- 分解覆盖率 100%
- 末级需求原子性
- 粒度合理

---

## 🎯 Skill 3：冲突检测

**Skill名称**：Conflict Detection

**目的**：检测需求之间的冲突

**输入**：需求清单、冲突检测规则、业务规则

**处理规则**：
1. 需求对比：对比所有需求对
2. 冲突识别：识别冲突需求对
3. 冲突分析：分析冲突类型
4. 冲突报告：生成报告

**输出**：冲突检测报告、冲突清单、解决建议

**验证标准**：
- 冲突检测准确率 > 95%
- 所有冲突被识别
- 冲突分类准确

---

## 🎯 Skill 4：重复检测

**Skill名称**：Duplicate Detection

**目的**：检测需求之间的重复

**输入**：需求清单、重复检测规则、相似度阈值

**处理规则**：
1. 需求对比：对比所有需求对
2. 相似度计算：计算相似度
3. 重复识别：识别重复需求对
4. 重复分析：分析重复类型

**输出**：重复检测报告、重复清单、去重建议

**验证标准**：
- 重复检测准确率 > 95%
- 所有重复被识别
- 相似度计算准确

---

## 🎯 Skill 5：业务模式匹配

**Skill名称**：Business Pattern Matching

**目的**：识别需求中的业务模式

**输入**：相关方需求清单、业务模式库、匹配规则

**处理规则**：
1. 需求分析：分析业务特征
2. 模式识别：识别可能的模式
3. 模式匹配：与模式库匹配
4. 匹配评分：计算匹配度

**输出**：模式匹配报告、匹配清单、模式建议

**验证标准**：
- 模式匹配准确率 > 90%
- 所有可匹配需求被识别
- 匹配度评分准确

## 🎯 Skill 7：SR→BA映射

**Skill名称**：Stakeholder Requirements to Business Architecture Mapping

**目的**：将相关方需求映射到业务架构

**输入**：相关方需求清单、业务模式库、映射规则

**处理规则**：
1. 需求分析：分析每条需求
2. 业务组件识别：识别需要的组件
3. 映射关系建立：建立映射
4. 映射验证：验证正确性

**输出**：业务架构清单、映射矩阵、质量报告

**验证标准**：
- 映射覆盖率 100%
- 1:1约束满足率 100%
- 映射准确率 > 95%
