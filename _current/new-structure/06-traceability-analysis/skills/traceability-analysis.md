# 端到端符合性分析 Skill

## 目标
验证从相关方需求（SR）到业务架构（BA）、系统需求（SysReq）、产品架构（PA）的完整链路，确保需求追溯的完整性和一致性。

## 输入
- 所有层级的设计文档
- 所有映射矩阵

## 输出
- 完整的追溯矩阵
- 符合性分析报告
- 缺陷清单

## 处理步骤

### 步骤1：收集所有设计文档
- 收集SR和SR-NFR
- 收集BA设计
- 收集SysReq设计
- 收集PA设计

### 步骤2：收集所有映射矩阵
- 收集SR→BA映射
- 收集BA→SysReq映射
- 收集SysReq→PA映射
- 收集SR-NFR→SysReq-NFR映射

### 步骤3：生成端到端追溯矩阵
- 合并所有映射矩阵
- 生成完整的追溯矩阵
- 验证追溯矩阵的完整性

### 步骤4：检查追溯完整性
- 检查SR追溯
- 检查BA追溯
- 检查SysReq追溯
- 检查PA追溯

### 步骤5：检查映射关系一致性
- 检查1:1约束
- 检查N:1关系
- 检查无孤立设计
- 检查无遗漏需求

### 步骤6：识别缺陷
- 识别遗漏的需求
- 识别孤立的设计
- 识别冲突的需求
- 识别不一致的映射

### 步骤7：分类和优先级排序
- 分类缺陷
- 优先级排序
- 评估影响

### 步骤8：提出改进建议
- 针对每个缺陷提出改进建议
- 评估改进建议的可行性

### 步骤9：生成报告
- 生成追溯矩阵
- 生成符合性报告
- 生成缺陷清单

## 关键约束

- **完整性**：确保所有需求和设计都被检查
- **一致性**：确保映射关系的一致性
- **可追溯性**：保持需求和设计之间的可追溯性

## 最佳实践

1. **系统性**：按照层级逐层检查
2. **完整性**：确保所有需求和设计都被检查
3. **一致性**：确保映射关系的一致性
4. **可追溯性**：保持需求和设计之间的可追溯性
5. **文档化**：记录所有缺陷和改进建议
