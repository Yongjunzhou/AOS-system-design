# 占位符创建与验证检查清单

**目的**：提供占位符创建和验证的标准检查清单，确保占位符的质量和一致性

**版本**：v1.0  
**最后更新**：2026-05-12

---

## 一、占位符创建检查清单

### 1.1 准备阶段

- [ ] 上游文档已完成并冻结
- [ ] 上游文档的末级节点已明确
- [ ] 下游团队已理解上游文档的结构
- [ ] 下游文档的模板已准备好

### 1.2 快速扫描阶段

- [ ] 已阅读上游文档的总体结构
- [ ] 已识别所有末级节点
- [ ] 已理解末级节点的含义和特征
- [ ] 已记录末级节点的 ID、名称、描述
- [ ] 末级节点清单已完成

### 1.3 占位符创建阶段

- [ ] 为每个上游末级节点创建了对应的占位符
- [ ] 占位符的 ID 符合命名规范（`[层级缩写]-NODE-[序号]`）
- [ ] 占位符的名称清晰、简洁、名词性结构
- [ ] 占位符的映射关系正确
- [ ] 占位符的其他内容用 `[待设计]` 占位
- [ ] 占位符的状态标记为"占位符"

### 1.4 映射表创建阶段

- [ ] 映射表已创建
- [ ] 映射表包含所有占位符
- [ ] 映射表的格式规范
- [ ] 映射表的内容准确

### 1.5 完整性验证阶段

- [ ] 是否为每个上游末级节点创建了对应的占位符？
- [ ] 是否存在遗漏的末级节点？
- [ ] 占位符的 ID 是否唯一？
- [ ] 占位符的名称是否清晰？
- [ ] 映射关系是否正确？
- [ ] 是否存在重复的占位符？

### 1.6 规范性验证阶段

- [ ] 占位符的 ID 是否符合命名规范？
- [ ] 占位符的名称是否符合命名规范？
- [ ] 占位符的格式是否符合标准格式？
- [ ] 占位符的描述是否用 `[待设计]` 占位？
- [ ] 占位符的状态是否标记为"占位符"？

### 1.7 上游团队确认阶段

- [ ] 占位符版本已提交给上游团队
- [ ] 上游团队已确认映射关系的正确性
- [ ] 上游团队已签字确认
- [ ] 占位符结构已冻结

## 二、占位符填充检查清单

### 2.1 准备阶段

- [ ] 占位符结构已冻结
- [ ] 映射关系已确认
- [ ] 上游文档已冻结
- [ ] 下游团队已理解占位符和上游文档

### 2.2 理解阶段

- [ ] 已详细阅读上游文档的每个末级节点
- [ ] 已理解每个末级节点的需求、约束、背景
- [ ] 已理解占位符与上游节点的映射关系
- [ ] 已记录关键信息和疑问
- [ ] 疑问已解决

### 2.3 填充阶段

- [ ] 逐个填充占位符的具体内容
- [ ] 填充的内容与上游文档的映射关系一致
- [ ] 填充的内容完整、清晰、准确
- [ ] 填充过程中及时验证与上游文档的一致性
- [ ] 所有占位符都已填充

### 2.4 下游占位符预留阶段

- [ ] 已识别需要进一步分解的节点
- [ ] 已为这些节点预留下一层级的占位符
- [ ] 已建立下一层级的映射表
- [ ] 下一层级的占位符已冻结

### 2.5 覆盖性验证阶段

- [ ] 下游节点的内容是否完全覆盖上游节点的需求？
- [ ] 是否存在遗漏的需求？
- [ ] 是否存在超出范围的内容？
- [ ] 所有需求都已被承接？

### 2.6 一致性验证阶段

- [ ] 下游节点的内容是否与上游节点的需求一致？
- [ ] 是否存在冲突或不一致？
- [ ] 所有冲突都已解决？

### 2.7 完整性验证阶段

- [ ] 下游节点的所有必填字段是否都已填充？
- [ ] 是否还有 `[待设计]` 的占位符？
- [ ] 所有内容都已填充？

### 2.8 最终验证阶段

- [ ] 下游文档的质量是否满足要求？
- [ ] 下游文档是否可以冻结？
- [ ] 是否需要进行迭代修改？

## 三、映射表验证检查清单

### 3.1 映射表的完整性验证

- [ ] 映射表是否包含所有上游末级节点？
- [ ] 映射表是否包含所有下游节点？
- [ ] 是否存在遗漏的映射关系？
- [ ] 是否存在重复的映射关系？

### 3.2 映射表的一致性验证

- [ ] 映射表中的上游节点 ID 是否与上游文档一致？
- [ ] 映射表中的下游节点 ID 是否与下游文档一致？
- [ ] 映射表中的映射关系是否与实际文档一致？

### 3.3 映射表的规范性验证

- [ ] 映射表的格式是否规范？
- [ ] 映射表的内容是否准确？
- [ ] 映射表的状态是否及时更新？

### 3.4 映射表的验证规则

- [ ] 每个上游末级节点都有对应的下游节点
- [ ] 每个下游节点都有对应的上游节点
- [ ] 映射关系是一对一的（N:1 多对一）
- [ ] 不存在遗漏或重复

## 四、占位符状态检查清单

### 4.1 占位符创建完成后

- [ ] 占位符的状态是否标记为"占位符"？
- [ ] 占位符的 ID 是否唯一？
- [ ] 占位符的名称是否清晰？
- [ ] 占位符的映射关系是否正确？
- [ ] 占位符的描述是否用 `[待设计]` 占位？

### 4.2 占位符填充开始前

- [ ] 占位符的状态是否更新为"设计中"？
- [ ] 占位符的映射关系是否确认？
- [ ] 下游团队是否已准备好填充？

### 4.3 占位符填充完成后

- [ ] 占位符的状态是否更新为"完成"？
- [ ] 占位符的所有内容是否都已填充？
- [ ] 占位符的内容是否通过验证？

## 六、检查清单的使用指南

### 6.1 何时使用

- **占位符创建完成后**：使用"占位符创建检查清单"
- **占位符填充完成后**：使用"占位符填充检查清单"
- **映射表创建或更新后**：使用"映射表验证检查清单"
- **占位符状态变更时**：使用"占位符状态检查清单"

### 6.2 如何使用

1. 逐个检查清单中的项目
2. 对于每个项目，确认是否满足要求
3. 如果不满足，记录问题并进行修改
4. 所有项目都满足后，签字确认

### 6.3 谁来使用

- **占位符创建检查清单**：下游团队
- **占位符填充检查清单**：下游团队
- **映射表验证检查清单**：下游团队 + 上游团队
- **占位符状态检查清单**：下游团队
