# Task 3：BA → SysReq映射 - 工作流

## 工作流

### 步骤1：分析业务架构节点
- 理解每个业务架构节点
- 识别节点的职责
- 确定节点之间的关系

### 步骤2：识别独立信息系统
- 识别一个或多个独立的信息系统
- 确定系统范围

### 步骤3：创建功能需求分支
- 创建功能需求分支
- 与非功能需求分支分离

### 步骤4：定义业务组件类型
- 定义流程、工单、计划、指标等业务组件类型
- 确定类型之间的关系

### 步骤5：定义业务组件小类
- 对业务组件类型进行小类分类
- 确定小类之间的关系

### 步骤6：定义组件实例
- 定义组件实例（SysReq 5级）
- 明确每个组件实例的职责

### 步骤7：定义功能模块
- 定义功能模块（SysReq 6级）
- 以标签页为载体

### 步骤8：定义系统用例
- 定义系统用例（SysReq 7级）
- 以功能按钮或右键菜单为载体

### 步骤9：定义用例场景
- 定义用例场景（SysReq 8级）
- 定义具体的场景

### 步骤10：定义场景活动
- 定义场景活动（SysReq 9级）
- 定义具体的活动步骤

### 步骤11：建立BA → SysReq映射
- 将业务架构节点映射到系统需求节点
- 建立N:1映射关系

### 步骤12：验证映射完整性
- 验证每个BA 5级节点都映射到至少一个SysReq 5级节点
- 验证没有孤立的SysReq节点

### 步骤13：标记占位符
- 为SysReq 9级节点预留PA映射关系
- 标记占位符

### 步骤14：交付检查
- 验证所有需求都已映射
- 验证所有占位符都已标记
- 准备交付下一阶段
