## 一、业务层级 L0~L6

EOS 平台的业务分为七级。L1~L4 为菜单层级（由菜单引擎生成），L4~L6 为业务内容（由场景/部件/元件级引擎生成）：

| 层级 | 名称 | 定义 | EOS 平台体现 | 生成引擎 |
|------|------|------|-------------|---------|
| L0 | EOS 平台 | 全部业务模块集合 | 平台自身 | — |
| L1 | 业务系统 | 一个业务域的所有业务集合 | 一级菜单 | 菜单引擎（横切） |
| L2 | 项目业务（业务模块） | 一个 E2E 项目的所有构件业务场景集合 | 二级菜单 | 菜单引擎（横切） |
| L3 | 构件业务 | 一个构件的所有场景业务集合 | 三级菜单 | 菜单引擎（横切） |
| L4 | 场景业务 | 某角色的 E2E 场景或某构件的业务场景 | **四级菜单（末级）**，打开后多窗口 | 场景引擎（组件级） |
| L5 | 场景部件 | 场景中的一个功能业务 | **一个窗口 = 一个场景部件 = 一份输出文档** | 表单/流程/指标/工单引擎（部件级） |
| L6 | 功能元件 | 原子级业务构件 | 实体对象、事件、定时任务等 | 实体引擎（元件级） |

### 关键关系

- **场景部件 : 输出文档 = 1 : 1**：每个场景部件是一个部件级引擎的一次配置实例，产出一份输出文档
- **场景业务 = STR-F 节点锚点**：场景业务（L4）是端到端业务闭环，由多个场景部件（L5）编排组成
- **实体引擎约束子业务 TAB**：实体引擎配置实体间关系图谱，表单引擎消费该图谱来配置主表单的子业务 TAB 群

## 二、引擎三级 + 横切

| 引擎层级 | 引擎 | 生成 | EOS 平台体现 |
|---------|------|------|-------------|
| 组件级 | 场景引擎 | L4 场景业务 | 编排场景部件为多窗口场景 |
| 部件级 | 表单引擎、流程引擎、指标引擎、工单引擎 | L5 场景部件 | 一个窗口（含主表单 + 子业务 TAB 群） |
| 元件级 | 实体引擎 | L6 功能元件 | 原子构件（实体对象、事件、定时任务等） |
| 横切 | 菜单引擎 | L1~L4 菜单树 | 菜单节点 CRUD + 权限管理 + 末级节点映射到 L4 场景业务 |

### 引擎分工总图

```
实体引擎（元件级）
  │  配置实体 + 实体间关系
  │  产出：L6 功能元件 + 实体关系图谱
  │        实体A → 子业务实体清单 [B, C, D, ...]
  │        该清单是表单引擎配置子业务TAB的约束范围
  │
  ↓  实体关系图谱作为约束输入
  │
表单/流程/指标/工单引擎（部件级）
  │  配置：主表单 + 子业务TAB群（TAB只能从实体关系清单中选取）
  │  产出：L5 场景部件 = 1个窗口 = 1份输出文档
  │
  ↓  多个场景部件被编排
  │
场景引擎（组件级）
  │  编排场景部件的执行序列
  │  产出：L4 场景业务（端到端业务闭环 = STR-F 节点锚点）
  │
  ↓  菜单映射
  │
菜单引擎（横切，不参与业务生成）
  │  L1~L4 菜单树 CRUD + 权限 + 末级节点映射到 L4 场景业务
```

### 编排关系

- 场景业务 = 多个场景部件编排组成（场景引擎调用表单/流程/工单引擎）
- 场景部件 = 主表单 + 子业务 TAB 群（表单/流程/工单引擎调用实体引擎；指标引擎采集各引擎运行数据）
- 子业务 TAB 由实体引擎的实体关系图谱约束——实体 A 关联了哪些实体，表单引擎才能将哪些实体配置为 A 的子业务 TAB

## 三、UI 交互模型

### 3.1 初始态（单窗全宽）

单击场景菜单进入场景业务时，浏览器窗口为**单窗全宽**，顶部多 TAB 对应场景业务下的多个场景部件。

```
┌─────────────────────────────────────────────────────────┐
│ [TAB: 场景部件A] [TAB: 场景部件B] [TAB: 场景部件C]      │
│                                                         │
│  ┌───────────────────────────────────────────────┐      │
│  │ 主表单（列表/树表）                              │      │
│  │ ┌────┬────┬────┬────┐                         │      │
│  │ │行1 │    │    │    │                         │      │
│  │ │行2*│ ← 双击选中行                              │      │
│  │ │行3 │    │    │    │                         │      │
│  │ └────┴────┴────┴────┘                         │      │
│  └───────────────────────────────────────────────┘      │
│                                                         │
│  底部导航：场景业务 > 场景部件A                          │
└─────────────────────────────────────────────────────────┘
```

### 3.2 分屏态（下钻触发）

双击行数据 → 窗口分裂为左右双窗：左窗为原窗（变窄），右窗为新开子业务窗口。

```
┌───────────────────────┬─────────────────────────────────┐
│ 左窗（原窗变窄）        │ 右窗（新开）                     │
│                       │                                 │
│ [TAB: 部件A]          │ [TAB: 子业务X] [TAB: 子业务Y]    │
│ ┌──────────────┐      │ ┌─────────────────────────┐     │
│ │行1           │      │ │ 行2的子业务数据           │     │
│ │行2*          │      │ │ ┌────┬────┬────┬────┐   │     │
│ │行3           │      │ │ │行a │    │    │    │   │     │
│ └──────────────┘      │ │ │行b*│                     │     │
│                       │ │ └────┴────┴────┴────┘   │     │
│                       │ └─────────────────────────┘     │
│                       │                                 │
│ 导航：场景业务 > 部件A │ 导航：场景业务 > 部件A           │
│                       │       > 子业务X                 │
└───────────────────────┴─────────────────────────────────┘
```

### 3.3 持续下钻

在右窗双击行 → 右窗移至左窗，新子业务 TAB 出现在右窗位置。可一直下钻直至系统下钻层级数限制。

```
┌───────────────────────┬─────────────────────────────────┐
│ 左窗：子业务X          │ 右窗：子子业务Z                   │
│ ...                   │ ...                             │
└───────────────────────┴─────────────────────────────────┘
```

### 3.4 面包屑导航

页面底部常驻导航条，形如资源管理器目录路径，显示当前表单的层级路径：

```
场景业务 > 场景部件A > 子业务X > 子子业务Z
```

- 双击导航条中任意节点 → 该节点对应表单成为左窗，右窗为其子业务
- 可用于任意跳转到下钻路径中的任何层级

### 3.5 交互规则总结

| 规则 | 说明 |
|------|------|
| 初始单窗 | 进入场景业务时为单窗全宽，不下钻不分屏 |
| 首次下钻 | 双击数据行触发分屏，左窗变窄、右窗新开 |
| 无限下钻 | 右窗→左窗，新右窗出现，直到系统层级数限制 |
| 面包屑导航 | 底部常驻，显示当前层级路径 |
| 面包屑跳转 | 双击任意节点跳至该层级 |
| 多 TAB | 每个窗含多个 TAB，对应场景部件或子业务 |

## 四、场景部件的两种产生方式

| 维度 | 类型1 · 引擎直产 | 类型2 · 人工外设 |
|------|-----------------|-----------------|
| **处理主体** | 引擎算法 | 人类 + CAX 工具 |
| **人类角色** | 操作引擎界面 | 使用专业工具完成实质转换 |
| **场景部件职责** | 提供操作界面 + 处理算法 | 提供规则/知识/合规/资源，收纳输出附件 |
| **输入文档** | 引擎读取并处理 | 人类取出，导入 CAX 工具 |
| **输出文档** | 引擎生成 | 人类在 CAX 中产出，作为附件回传 |
| **CU 选配剖面** | 完整选配（含校验/行为联动） | 轻量选配（不选校验/行为） |

两种方式均由现有部件级引擎支撑，区别在**配置单元的选配剖面**不同，不需要新引擎类型。

## 五、双链关系

- **平台设计链**：产出引擎软件构件（前端组件、后端服务、数据模型、API、基础设施）。FR-ENG 需求走此链（→ wft01-eng → wft02-eng）
- **业务设计链**：产出配置定义（场景业务 + 场景部件配置）。FR-BIZ 需求走此链（→ wft01-biz → wft02-biz）
- 引擎是配置的能力边界，配置运行在引擎之上
- 业务链推进中发现引擎能力不足 → 需求上升为 FR-ENG → 走平台设计链补齐引擎 → 业务链再扩展
