# Skill: 冲突检测

## 目的
检测新增需求与已有需求之间的冲突，并进行分类和建议。

## 输入
- 新增需求（规范化和分解后）
- 已有需求列表

## 输出
- 冲突列表
- 冲突分类
- 处理建议

## 冲突类型

### 1. 直接冲突
**定义**：新需求与已有需求直接矛盾，无法同时满足

**例子**：
- 已有需求：系统应支持用户删除订单
- 新增需求：系统应禁止用户删除订单

**处理建议**：需要相关方确认，选择其中一个或进行调和

### 2. 间接冲突
**定义**：新需求与已有需求间接矛盾，可能导致系统行为不一致

**例子**：
- 已有需求：订单创建后不能修改
- 新增需求：支持订单修改

**处理建议**：需要相关方确认，调整需求或业务规则

### 3. 可合并冲突
**定义**：新需求与已有需求存在冲突，但可以通过调整合并

**例子**：
- 已有需求：支持按订单号查询
- 新增需求：支持按客户名查询

**处理建议**：合并为"支持按多个条件查询"

## 冲突检测规则

### 1. 语义冲突检测
- [ ] 新需求与已有需求是否存在直接矛盾
- [ ] 新需求与已有需求是否存在间接矛盾
- [ ] 新需求与已有需求是否存在约束冲突

### 2. 业务规则冲突检测
- [ ] 新需求是否违反已有的业务规则
- [ ] 新需求是否与已有的业务流程冲突

### 3. 技术约束冲突检测
- [ ] 新需求是否与已有的技术约束冲突
- [ ] 新需求是否与已有的系统架构冲突
