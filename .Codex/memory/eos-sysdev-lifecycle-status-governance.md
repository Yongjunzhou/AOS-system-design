# EOS 系统设计生命周期状态治理规则

## 结论摘要

本项目在 EOS 系统设计子线中，对 OR 原料、OR 切分文档、OR 基线条目、STR 架构节点、STR 详细节点、BA 节点、SysReq 节点、PA 组件等对象，统一采用“生命周期/当前状态 + 人类决策”驱动下游步骤入口的规则。

`AI建议` 仅用于给人类提供阅读提示和操作建议，不作为自动进入下游步骤的硬条件。

## 规则

1. 下游步骤入口判断只看权威状态表中的 `生命周期` 或 `当前状态`，以及 `人类决策`。
2. `AI建议` 不能作为步骤入口硬条件；它只能表达“建议下一步做什么”。
3. 某个对象的生命周期状态说明，应放在该对象的权威状态表旁边，至少包含：
   - 状态取值
   - 各状态含义
   - 由哪个 skill 设定
   - 典型流转路径
4. 上层规范只定义通用约束；对象级权威状态表对该对象的具体状态语义拥有优先解释权。
5. 文件命名需区分文本化文档与切分文档：
   - 文本化文档：`.textualized.md`
   - 切分文档：`.chunk-{seq}.md`

## 当前统一口径

- OR 原料：`已登记 → 已文本化 → 切分方案待确认 → 已切分 → 已废弃`
- OR 切分文档：`已切分 → 澄清中 → 已澄清 → 待确认规范方案 → 已规范化 → 已废弃`
- OR 基线条目：`待分配 → 待确认方案 → 已分配 → 验证中 → 已验证冻结 → 已废弃`
- STR 架构节点：`待分解分配 → 待确认方案 → 迭代中 → 已分解分配 → 验证中 → 已验证冻结 → 已废弃`
- STR 详细节点：`待BA承接 → 待确认方案 → 迭代中 → 已BA承接 → 验证中 → 已验证冻结 → 已废弃`
- BA 节点：`待架构映射 → 待确认方案 → 迭代中 → 已架构映射 → 验证中 → 已验证冻结 → 已废弃`
- SysReq 节点：`待产品架构 → 待确认方案 → 迭代中 → 已产品架构 → 验证中 → 已验证冻结 → 已废弃`
- PA 组件：`待追溯验证 → 待确认方案 → 迭代中 → 验证中 → 已验证冻结 → 已废弃`

## 关键职责边界

- `ort00` 负责原料文本化，不负责切分决策本身。
- `ort01` 在 OR 原料已文本化且人类同意切分后，先生成切分方案，再在人类确认后执行切分。
- `ort02` / `ort03` 按 OR 切分文档生命周期推进澄清与规范化。
- `wft01` 审阅通过后，将 STR 架构节点推进到 `待分解分配`，并把对应 OR 条目同步为 `已分配`。
- `wft02` 设定 `已分解分配`。
- `wft03` 设定 `已架构映射`。
- `wft05` 设定 `已产品架构` 与 `待追溯验证`。
- `wft06` 负责将进入验证批次的对象推进到 `验证中`，并在通过后推进到 `已验证冻结`。
