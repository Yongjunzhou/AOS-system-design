# AOS 系统设计项目

## 项目定位

本项目同时开发两个产品，遵循统一的五层系统设计方法论：

- **产品L — 系统设计流水线**：将运营体系的系统设计从手工作坊式的文档撰写，转变为结构化、可追溯、AI 辅助的工业化生产过程。产品L的交付物是准则、指南、AI 辅助文档、任务定义四类知识组件。
- **产品M — AOS 企业运营体系**：企业实际运行所依赖的 IT/AI 化运营系统，涵盖市场、研发、生产、售后等业务，同时也包含"设计运营体系本身"这项业务。

第三个并行目标是：通过产品L和产品M的实践，验证和提炼可复用的产品开发方法论。

**当前范围**：
- 流水线：严格限定在流水线开发四阶段中的阶段1（系统设计），即从原始需求到产品架构的完整设计链路
- 运营体系：产品开发全过程，即系统设计、组件开发、系统集成与交付、系统运维

## 方法论概览

本项目遵循**五层系统设计方法论**：

```
OR（原始需求）→ SR（相关方需求）→ BA（业务架构）→ SysReq（系统需求）→ PA（产品架构）
```

三条核心规则：**1:1 分配约束**、**N:1 承接支持**、**双向追溯**。详见[通用设计准则](00-general/10-general-design-standards/01-general-design-standards.md)。

本项目同时包含两个产品，它们共享同一套方法论但 PA 产物形态不同——产品L是知识/方法类产品（交付文档+AI），产品M是软件类产品（交付信息系统）。详见 [`CLAUDE.md`](CLAUDE.md)。
