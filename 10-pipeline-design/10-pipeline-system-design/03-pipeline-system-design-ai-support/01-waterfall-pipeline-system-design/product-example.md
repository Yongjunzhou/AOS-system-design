# 瀑布式系统设计：产品数据范例
**Waterfall System Design: Product Data Example**

**用途**：展示一个完整的瀑布式系统设计过程的输入和输出范例，AI 参照此范例生成自己的设计文档。

**范例项目**：产品B·系统设计流水线（System Design Pipeline）

> **产品类型说明**：系统设计流水线属于**方法论/工具类产品**。其产品架构的分类方式为"规范与准则 / 设计指南 / AI辅助文档 / 任务定义"（而非软件产品的"前端/后端/数据/基础设施"）。本范例完整展示此类型产品的系统设计过程。

---

## 一、第1步产出范例

### 1.1 原始需求收集和初步整理

```
OR-001: 设计准则与规范体系
├─ 需求描述：系统设计流水线应提供一套完整的系统设计准则，定义五层结构（OR/SR/BA/SysReq/PA）
│  三条核心规则和术语体系，作为所有系统设计活动的总纲和依据
├─ 原始碎片清单：
│  ├─ "做系统设计必须先有个总纲，把五层结构和每层的定义说清楚" — 系统架构师 2026-05-01
│  ├─ "各层的规范要独立成文，方便不同设计阶段查阅" — 流水线开发者 2026-05-02
│  └─ "术语应该统一，否则跨层沟通会混乱" — AOS架构师 2026-05-03
├─ 包含的性能指标：准则内容完整覆盖五层结构，规则定义无矛盾
├─ 依赖/冲突关系：无（核心基础需求）
└─ 整理说明：将架构师和流水线开发者的碎片合并，归结为准则体系定义

OR-002: 多场景设计方法指南
├─ 需求描述：流水线应提供覆盖四种设计场景（瀑布式/敏捷式/逆向工程/DevOps）的指南，
│  每种指南包含适用条件判断、完整操作流程、各步骤输入输出说明和质量要求
├─ 原始碎片清单：
│  ├─ "全新研发和迭代优化的设计方法不一样，要分别给指南" — AOS架构师 2026-05-04
│  ├─ "指南要能指导AI操作，不只是给人看的" — 流水线开发者 2026-05-05
│  └─ "最好有个快速参考卡，设计时随手翻" — 系统设计者 2026-05-06
├─ 包含的性能指标：每种指南需包含完整的操作流程（步骤数 ≥ 6）和质量检查清单
├─ 依赖/冲突关系：依赖 OR-001（指南需引用准则定义的方法论框架）
└─ 整理说明：三个角色的碎片合并，涵盖指南数量和多样性需求

OR-003: AI辅助自动化设计支持
├─ 需求描述：流水线应为每种设计场景提供AI辅助文档集，包括场景说明（README）、
│  AI自检清单（checklist）、工作流提示词（workflow-prompts）和产品数据范例（product-example），
│  使AI能按流程自动完成系统设计的每一步
├─ 原始碎片清单：
  ├─ "AI参与设计要有一套标准的提示词，每次调用都按流程走" — 流水线开发者 2026-05-07
  ├─ "AI做完一步要能自检，不能漏掉质量检查" — AOS架构师 2026-05-08
  └─ "最好给个完整范例，AI照着出就不会偏" — 系统设计者 2026-05-09
├─ 包含的性能指标：AI辅助文档覆盖全部四种场景，每种场景包含4份文档
├─ 依赖/冲突关系：依赖 OR-002（AI辅助文档的内容基于设计指南的结构）
└─ 整理说明：流水线开发者关注AI操作流程，AOS架构师关注质量，合并为AI辅助文档需求

OR-004: 设计任务操作规程
├─ 需求描述：流水线应将六步设计法的每一步定义为标准化任务，
│  包含任务描述、输入/输出定义、操作步骤、质量检查清单和验收标准
├─ 包含的性能指标：每个任务定义包含完整的输入/输出/步骤/检查清单
├─ 依赖/冲突关系：依赖 OR-001 和 OR-002
└─ 整理说明：单一来源，直接保留

OR-005: 设计工具体系集成与导航
├─ 需求描述：流水线应将准则/指南/AI辅助/任务定义整合为一套连贯的设计工具体系，
│  设计者能按场景选择合适的方法，并快速在各类文档间切换和定位
├─ 原始碎片清单：
│  ├─ "文档多了之后怎么串起来？要有个导航" — AOS架构师 2026-05-10
│  └─ "跨文档引用要能追溯，不能各说各的" — 流水线开发者 2026-05-11
├─ 包含的性能指标：跨文档引用一致性 100%，场景导航覆盖所有场景
├─ 依赖/冲突关系：依赖 OR-001~OR-004（集成的前提是各组件已定义完成）
└─ 整理说明：来自两个角色的碎片，关注集成和一致性维护

OR-NFR-001: 设计文档质量标准
├─ 需求描述：所有流水线产出的设计文档应格式规范统一、内容完整一致、术语使用准确
├─ 包含的性能指标：
│  - 格式规范: 所有文档使用 Markdown + Mermaid，文件命名遵循 NN-descriptive-name.md
│  - 内容完整: 每份文档包含头部元数据、正文、版本记录
│  - 术语一致: 核心术语在准则中统一定义，所有文档引用同一术语体系
└─ 整理说明：来自流水线开发者的总体质量要求
```
