# 系统设计逆向工程：AI 自检清单
**System Design Reverse Engineering: AI Self-Check List**

**用途**：AI 完成每一步后，使用此清单自行验证产出质量。全部通过后方可提交给流水线开发者审核。

---

## 一、跨步骤通用检查（适用于所有步骤）

### 制品一致性
- [ ] 每个 PA 组件都能精确定位到对应的产物或设计制品
- [ ] 每个 PA 组件的内容与 PA 定义一致（不超出 PA 定义的职责范围）
- [ ] 不存在 PA 中不存在但文档中凭空出现的组件/需求
- [ ] 不存在 PA 中存在但文档中未覆盖的组件

### 反推值标注
- [ ] 所有从下层制品反推的性能指标已标注"反推值 ⚠️ 待确认"
- [ ] 直接取自下层制品的指标标注了来源（如"取自PA定义"）
- [ ] 无法推断的指标标注了"待补充"

### 格式规范性
- [ ] 文档编号格式统一（OR-XXX / SR-F-XXX / BA-XXX / SysReq-F-XXX / PA-XXX）
- [ ] 架构定义文档均包含两部分：第一部分（树形结构）+ 第二部分（映射关系表）
- [ ] 树形结构层级缩进正确，末级节点清晰标注
- [ ] 超链接格式正确，指向有效目标

### 术语一致性
- [ ] "架构末级节点"和"详细定义末级节点"未混用
- [ ] "1:1 分配约束"和"N:1 承接支持"未混淆
- [ ] 功能需求和非功能需求的标识统一（SR-F vs SR-NFR / SysReq-F vs SysReq-NFR）

### 资产优先原则（新增）
- [ ] 反向聚合时，是否优先检查上层是否已有可承接的节点（复用已有架构）？
- [ ] 在自底向上推导过程中，发现的重复节点是否标记了合并建议？
- [ ] 废弃/合并的节点是否标注而非物理删除？
- [ ] 同一节点是否经历 3 次以上 Extend 却未审查 Split？

### 术语一致性
- [ ] "架构末级节点"和"详细定义末级节点"未混用
- [ ] "1:1 分配约束"和"N:1 承接支持"未混淆
- [ ] 功能需求和非功能需求的标识统一（SR-F vs SR-NFR / SysReq-F vs SysReq-NFR）

### 依据B对齐
- [ ] 逆向推导的 AOS 设计文档是面向产品M（AOS运营体系），而非产品L（流水线）
- [ ] AOS 的设计内容使用了运营体系领域术语（业务流程、组织管理、绩效指标等），而非流水线术语
- [ ] AOS 产品架构的分类体系符合产品类型（AOS为IT/AI化运营系统 → 软件产品分类：前端/后端/数据/基础设施）
- [ ] 产品L（流水线）的产品架构示例应根据目标产品类型选择 PA 分类：软件产品为前端/后端/数据/基础设施，方法论/工具产品为规范与准则/设计指南/AI辅助文档/任务定义
- [ ] 涉及引用示例时，示例内容反映相应产品类型的业务场景

## 二、前置准备 · 第1步自检：现有产物梳理与架构分析

> 本步骤仅在 PA 文档不存在或质量严重不足时执行。如果 PA 已存在且可用，跳过本步。

### 产物梳理
- [ ] 项目目录结构已完整扫描
- [ ] 所有产物已按架构层级分类（OR/SR/BA/SysReq/PA）
- [ ] 产物类型已标注（规范定义/设计指南/操作模板/产品数据模板/工具脚本等）
- [ ] 覆盖盲区已标记

### 架构评估
- [ ] 产物覆盖度评估已执行
- [ ] 初版架构评估已完成（完整度/分类合理性）
- [ ] 性能指标已从现有产物特征反推并标注"反推值"
- [ ] 所有待确认项已标注

---

## 三、第2步自检：产品架构文档编写

### 架构定义
- [ ] 所有组件有清晰的定义和职责边界
- [ ] 组件粒度合理（以"可由单人独立完成"且"可独立审查和验收"为双重标准）
- [ ] 分类体系与产品类型匹配
- [ ] 树形结构完整（0级→1级→2级→3级·组件/末级）
- [ ] 映射关系表骨架已建立（预留 SysReq 字段）

### 详细定义
- [ ] 每个 PA 组件有清晰的需求定义和职责描述
- [ ] 每个 PA 组件有内容结构说明（文档格式、章节组织）
- [ ] 每个 PA 组件有版本策略和质量指标
- [ ] 每个 PA 组件有性能指标（反推值 ⚠️ 待确认）

### 产物追溯
- [ ] 每个 PA 组件能对应到具体的产物目录
- [ ] 不存在无法追溯回产物的 PA 组件
- [ ] 所有现有产物已纳入 PA 组件覆盖

---

## 四、第3步自检：系统需求反向推导

### PA→SysReq 追溯验证
- [ ] 分析活动已完成：PA→SysReq 逆向追溯验证已执行
- [ ] 所有 PA 组件都能反推出对应的 SysReq（覆盖率 100%）
- [ ] 功能场景活动链完整
- [ ] 非正常场景已尽可能从 PA 组件的职责边界中识别

### 功能需求架构
- [ ] 0-9 级层级结构正确（从 9 级场景活动向上聚合到 5 级）
- [ ] 5 级节点（组件实例）承接了所有 PA 组件的功能
- [ ] 每个 9 级场景活动分配到一个 PA 组件（1:1 约束满足）
- [ ] 9 级活动的描述与 PA 组件的需求定义一致
- [ ] 5 级节点包含符合性分析、设计指导、性能指标

### 非功能需求架构
- [ ] 非功能需求按 0-6 级平行架构组织
- [ ] 已按三大类分类（质量特性/环境适应性/可实现性）
- [ ] 每个 SysReq-NFR 5 级节点包含类型和验证方法
- [ ] 量化指标已从 PA 组件特征提取或反推（标注反推值/取自配置/待补充）
- [ ] 约束条件明确

### 同步设计
- [ ] 已同步建立业务架构 5 级节点骨架
- [ ] SysReq → BA 骨架映射关系已建立

## 五、第4步自检：业务架构反向推导

### SysReq→BA 追溯验证
- [ ] 分析活动已完成：SysReq→BA 逆向追溯验证已执行
- [ ] 每个 SysReq 5 级节点都能反推出合理的 BA IPO
- [ ] 候选 IPO 已执行去重（复用→改进→新增三级策略）
- [ ] 去重报告已生成（去重前/后数量对比）

### IPO 定义
- [ ] 每个 BA IPO 标注了节点类型（配置/开发）
- [ ] 每个 IPO 包含完整的 I（输入）/ P（处理）/ O（输出）三维定义
- [ ] 每个 IPO 包含符合性分析和设计指导
- [ ] 性能指标已从 SysReq 继承（标注反推值）
- [ ] IPO 命名使用"动词+名词"结构
- [ ] Input/Process/Output 与 SysReq 9 级场景活动的数据流一致

### 同步设计
- [ ] 已同步建立相关方需求架构骨架
- [ ] BA → SR 骨架映射关系已建立

## 六、第5步自检：相关方需求反向推导

### BA→SR 追溯验证
- [ ] 分析活动已完成：BA→SR 逆向追溯验证已执行
- [ ] 每个 BA IPO 都能反推出合理的相关方诉求
- [ ] 不存在"纯技术实现"但无业务语义的 BA IPO
- [ ] 全业务域覆盖（无缺失业务域）

### 功能需求 SR-F
- [ ] 所有 BA IPO 已映射到 SR 架构末级节点（覆盖率 100%）
- [ ] 功能需求（SR-F）和非功能需求（SR-NFR）分别处理
- [ ] 每个架构末级节点包含符合性分析和设计指导
- [ ] 每个架构末级节点包含性能指标（反推值 ⚠️）
- [ ] 详细定义末级满足 1:1 约束

### 非功能需求 SR-NFR
- [ ] 所有 SysReq-NFR 5 级节点已反推为 SR-NFR 架构末级节点
- [ ] SR-NFR 架构末级节点包含需求类型（性能/安全/可用性等）和需求类别（质量特性/环境适应性/可实现性）
- [ ] 非功能需求包含业务语义描述（而不仅是技术指标）
- [ ] 量化指标从 SysReq-NFR 正确继承

## 七、第6步自检：原始需求反向推导

### SR→OR 追溯验证
- [ ] 分析活动已完成：SR→OR 逆向追溯验证已执行
- [ ] 所有 SR 架构末级节点都能反推出合理的原始需求
- [ ] OR 聚合语义完整（聚合后的 OR 使用自然语言描述）

### 原始需求
- [ ] 所有 SR 架构末级节点已聚合为 OR 条目
- [ ] 每条 OR 已分解到末级
- [ ] 每条末级满足 1:1 约束（只需一个 SR 节点承接）
- [ ] 冲突检测和重复检测已完成
- [ ] 所有 `⚠️ 待确认` 标记已标注

### 5 层闭环
- [ ] 正向追溯完整（OR → SR → BA → SysReq → PA，覆盖率 100%）
- [ ] 反向追溯完整（PA → SysReq → BA → SR → OR，覆盖率 100%）
- [ ] 所有方案节点标注了承接来源
- [ ] SR 文档中"承接原始需求末级"字段已补充

## 八、第7步自检：完整性和一致性验证

### 1:1 分配约束
- [ ] 原始需求末级 → SR 架构末级：1:1 满足率 100%
- [ ] SR 详细定义末级 → BA IPO：1:1 满足率 100%
- [ ] BA IPO → SysReq 5 级：1:1 满足率 100%（去重后的 IPO）
- [ ] SysReq 9 级 → PA 末级：1:1 满足率 100%

### N:1 承接支持
- [ ] 每个 SR 架构末级节点承接 2-5 条原始需求末级（合理范围）
- [ ] 每个 BA IPO 承接 2-5 条 SR 详细定义末级（合理范围）
- [ ] 每个 SysReq 5 级节点承接 2-5 个 BA IPO（合理范围）
- [ ] 每个 PA 组件承接 2-5 个 SysReq 9 级节点（合理范围）
- [ ] 承接 1 条的已考虑合并，承接 6+ 条的已检查是否需要拆分

### 逆向合理性检查（专用）
- [ ] 每个 PA 组件与产物目录中的实际制品一致（无编造组件）
- [ ] 所有反推的性能指标有下层制品依据
- [ ] 不存在"空壳"PA 组件（制品中不存在但文档中虚构的组件）
- [ ] 产物目录中的主要制品都已覆盖（无遗漏）
- [ ] 制品与文档的差异已记录在差异清单中

### 符合性分析
- [ ] 所有方案节点都有符合性分析（完整率 100%）
- [ ] 每个符合性分析说明了方案如何满足所承接的需求
- [ ] 符合性分析包含验证方法

### 性能指标跨层一致性
- [ ] 性能指标从 PA → SysReq → BA → SR → OR 逐层传递，无断层
- [ ] PA 组件指标不低于 SysReq 要求（逆向中：SysReq 指标不高于 PA 实际能力）
- [ ] 所有反推指标标注"反推值 ⚠️ 待确认"
