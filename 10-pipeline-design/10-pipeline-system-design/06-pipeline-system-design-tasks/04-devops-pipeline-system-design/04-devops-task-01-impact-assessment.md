# task-01：快速影响范围评估

**对应场景**：04-devops · 第1步
**对应 AI 模板**：`03-pipeline-system-design-ai-support/04-pipeline-system-design-devops/workflow-prompts.md` 第1步

---

## 目标

快速判断问题的影响范围，选择合适的处理模式。

## 输入

| 输入 | 说明 |
|------|------|
| 问题报告或紧急需求 | bug、性能问题、紧急需求描述 |
| 现有完整系统设计文档集 | 基线文档 |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 问题分类 | AI | bug / 功能需求 / 性能优化 |
| 2 | 影响范围快速评估 | AI | 定位受影响的层和节点 |
| 3 | 处理模式选择 | AI | 模式1（快速修复）/ 模式2（局部设计）/ 模式3（完整设计） |
| 4 | 结果审核 | 人类 | 确认问题分类和处理模式 |

## 人类审核要点

- [ ] 优先级判定（P0/P1/P2）— 需确认
- [ ] bug 根本原因诊断
- [ ] 处理模式的最终决定

## 质量门禁

- [ ] 问题分类完成
- [ ] 处理模式已选定
- [ ] 影响范围已定位

## 产出

| 文件 | 说明 |
|------|------|
| 影响范围评估报告 | 本步产出，指导后续修复步骤 |
