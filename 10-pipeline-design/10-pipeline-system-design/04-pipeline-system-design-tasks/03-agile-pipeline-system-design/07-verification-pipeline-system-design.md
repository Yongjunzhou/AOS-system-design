# task-07：完整性和一致性验证

**对应场景**：03-agile · 第7步
**对应 AI 模板**：`03-pipeline-system-design-ai-support/03-agile-pipeline-system-design/workflow-prompts.md` 第7步
**对应设计指南**：[敏捷式系统设计指南 第7步](../../02-pipeline-system-design-guidelines/03-agile-pipeline-system-design-guide.md#第7步完整性和一致性验证)

---

## 目标

验证增量更新后的系统设计的完整性和一致性。

## 上下文继承

```
从哪里来：
  - task-01 ~ task-06 的全部增量产出
  - 本次迭代前的完整基线文档

核心验证任务：
  1. 1:1 分配约束验证（增量部分）
  2. N:1 承接合理性验证（更新后）
  3. 双向追溯验证（增量部分）
  4. 增量一致性检查（有无"更新了上层漏了下层"）
  5. 版本一致性验证
```

---

## 输入

| 输入 | 说明 |
|------|------|
| 全部增量产出 | task-01 ~ task-06 的增量内容 |
| 现有基线文档 | 本次迭代前的完整基线 |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 分析活动：增量验证分析 | AI | 需求说明符合性分析汇总（增量部分逐层验证映射覆盖率）+ 需求性能符合性分析汇总（增量性能指标逐层继承验证） |
| 2 | 1:1 分配约束验证（增量） | AI | 只检查新增/修改的映射关系 |
| 3 | N:1 承接合理性验证 | AI | 更新后各节点的承接数量 |
| 4 | 双向追溯验证（增量） | AI | 增量部分的追溯链完整性 |
| 5 | 增量一致性检查 | AI | 有无"更新了上层漏了下层"的情况 |
| 6 | 版本一致性验证 | AI | 所有文档版本号和时间一致 |
| 7 | 结果审核 | 人类 | 确认验证结论 |

## 人类审核要点

- [ ] 验证报告中的未通过项
- [ ] 增量一致性是否有遗漏
- [ ] 1:1 约束未通过项是否能接受或需要回溯修复
- [ ] 版本一致性是否通过

## 质量门禁

- [ ] 分析活动：增量验证分析已执行
- [ ] 1:1 约束通过率 100%
- [ ] N:1 承接合理性评估完成
- [ ] 双向追溯验证通过（增量部分）
- [ ] 增量一致性检查通过
- [ ] 版本一致性验证通过
- [ ] **运行 conformance-review 进行完整符合性审查**（增量更新后的关键门禁）
- [ ] 按审查修复流程处理问题

## 产出

| 文件 | 说明 |
|------|------|
| 增量验证报告 | 本步产出（附录到验证报告文档） |
