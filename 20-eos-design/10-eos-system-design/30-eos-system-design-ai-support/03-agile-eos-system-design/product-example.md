# EOS 敏捷式系统设计：产品数据范例
**EOS Agile System Design: Product Data Example**

**用途**：展示一个完整的 EOS 敏捷式增量系统设计过程的输入和输出范例，AI 参照此范例生成自己的 EOS 增量设计文档。

**范例项目**：EOS·EOS 企业运营体系（Enterprise Operation System）

**基线说明**：本范例基于瀑布式场景（`01-waterfall/product-example.md` v1.0）的完整 EOS 设计输出作为基线。基线内容包含：
- 14 个 PA 组件（前端 5 + 后端 5 + 数据层 2 + 基础设施 2）
- 产品类型为**IT/AI化运营系统**，PA 分类为"前端/后端/数据/基础设施"

本次增量在基线基础上，新增"**绩效看板与预警管理**"能力，使 EOS 运营体系支持组织绩效数据可视化、异常预警和趋势分析。

## 一、第1步产出范例

### 1.1 新需求收集与分类

```
OR-006: 绩效看板与预警管理
├─ 需求描述：EOS 应提供组织级绩效看板，实时展示各部门的关键绩效指标（KPI）完成情况、
│           支持按时间维度（日/周/月/季）切换查看趋势，并在指标偏离目标时自动触发预警通知
├─ 原始碎片清单：
│  ├─ "管理层需要一个大屏看板，实时看到各部门的KPI完成情况" — 运营总监 2026-05-12
│  ├─ "绩效指标低于目标值时要自动发预警消息给部门负责人" — 绩效管理部 2026-05-13
│  └─ "不光看当前，还要能看趋势，上个月和这个月对比" — 业务部门经理 2026-05-14
├─ 包含的性能指标：看板加载 < 3s，预警响应 < 5s，实时数据刷新周期 < 30s
├─ 需求类型：功能需求
├─ 变更类型：新增
├─ 与现有需求的关系：与 OR-003（绩效管理）相关，可复用现有绩效指标定义和计算引擎
└─ 依赖/冲突关系：依赖 OR-003（需要绩效指标定义作为数据基础）
```

├─ Input：预警规则参数（KPI指标、比较运算符、阈值、通知方式）

│  2. 系统校验规则完整性

├─ Output：预警规则配置（含启用/停用状态）
├─ 承接的SR详细定义末级：SR-F-006.2

+ BA-021: 预警通知处理（5级·架构末级节点）
├─ 节点类型：配置
├─ 定义：预警触发后自动通知责任人，支持预警全生命周期管理

│  2. 生成预警事件（含指标、偏离值、建议措施）
│  3. 根据预警级别选择通知方式（消息/邮件/短信）
│  4. 发送预警通知给部门负责人
│  5. 责任人确认、处理或转办预警事件
│  6. 记录预警处理全流程日志

├─ 符合性分析：通过预警引擎和通知分发满足预警通知处理需求
├─ 性能指标：预警通知 < 5s
├─ 设计指导：系统应提供预警引擎、通知渠道管理、预警处理工作流
