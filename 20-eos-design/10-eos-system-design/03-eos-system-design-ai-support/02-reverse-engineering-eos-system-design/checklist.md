# EOS 系统设计逆向工程：AI 自检清单
**EOS System Design Reverse Engineering: AI Self-Check List**

**用途**：AI 完成每一步后，使用此清单自行验证产出质量。全部通过后方可提交给 EOS 设计人员审核。

---

## 一、跨步骤通用检查（适用于所有步骤）

### PA/产物一致性
- [ ] 每个 EOS PA 组件都能精确定位到对应的产物目录或模块
- [ ] 每个 PA 组件的实现细节（技术栈、关键类、接口）与PA定义一致
- [ ] 不存在PA定义中不存在但文档中凭空出现的组件/需求
- [ ] 不存在PA/产物中存在但文档中未覆盖的主要功能模块

### 反推值标注
- [ ] 所有从 PA 及制品反推的性能指标已标注"反推值 ⚠️ 待确认"
- [ ] 直接取自产品配置的指标标注了来源（如"取自配置"）
- [ ] 无法推断的指标标注了"待补充"

### 格式规范性
- [ ] 文档编号格式统一（OR-XXX / SR-F-XXX / BA-XXX / SysReq-F-XXX / PA-XXX）
- [ ] 架构定义文档均包含两部分：第一部分（树形结构）+ 第二部分（映射关系表）
- [ ] 树形结构层级缩进正确，末级节点清晰标注
- [ ] 超链接格式正确，指向有效目标

### 术语一致性
- [ ] "架构末级节点"和"详细定义末级节点"未混用
- [ ] "1:1 分配约束"和"N:1 承接支持"未混淆
- [ ] 功能需求和非功能需求的标识统一（SR-F vs SR-NFR / SysReq-F vs SysReq-NFR）

### 资产优先原则（新增）
- [ ] 反向聚合时，是否优先检查上层是否已有可承接的节点（复用已有架构）？
- [ ] 在自底向上推导过程中，发现的重复节点是否标记了合并建议？
- [ ] 废弃/合并的节点是否标注而非物理删除？
- [ ] 同一节点是否经历 3 次以上 Extend 却未审查 Split？

### 依据B对齐
- [ ] 逆向推导的 EOS 设计文档是面向EOS（企业运营体系），而非设计线（流水线）
- [ ] EOS 的设计内容使用了运营体系领域术语（业务流程、组织管理、绩效指标等），而非流水线术语
- [ ] EOS 产品架构的分类体系符合产品类型（EOS为IT/AI化运营系统 → 软件产品分类：前端/后端/数据/基础设施）
- [ ] 涉及引用示例时，示例内容反映 EOS 的业务场景

- [ ] 各层文档覆盖盲区已标注
- [ ] 现有文档质量已评估（是否符合准则）
- [ ] 关键技术决策已从现有产物中提取（语言、框架、数据库、中间件）

### 初版架构评估

- [ ] EOS 性能指标已从制品特征反推并标注"反推值"

- [ ] 所有 EOS 组件有清晰的定义和职责边界
- [ ] 组件粒度合理（可独立部署、1-2周开发周期）
- [ ] 4 大组件分类正确（前端/后端/数据/基础设施）
- [ ] 树形结构完整（0级→1级→2级→3级·组件/末级）
- [ ] 映射关系表骨架已建立（预留 SysReq 字段）

### 详细定义
- [ ] 每个 EOS PA 组件有技术栈定义（从产物提取）

- [ ] 每个 PA 组件有接口定义（从制品定义提取）

- [ ] 每个 PA 组件有性能指标（反推值 ⚠️ 待确认）

### 制品追溯
- [ ] 每个 EOS PA 组件的实现细节与制品位置一一对应
- [ ] 不存在无法追溯回制品的 PA 组件
- [ ] 接口定义中的 METHOD 和路径与制品一致

## 八、第7步自检：完整性和一致性验证

### 1:1 分配约束
- [ ] 原始需求末级 → SR 架构末级：1:1 满足率 100%
- [ ] SR 详细定义末级 → BA IPO：1:1 满足率 100%
- [ ] BA IPO → SysReq 5 级：1:1 满足率 100%（去重后的 IPO）
- [ ] SysReq 9 级 → PA 末级：1:1 满足率 100%

### N:1 承接支持
- [ ] 每个 SR 架构末级节点承接 2-5 条原始需求末级（合理范围）
- [ ] 每个 BA IPO 承接 2-5 条 SR 详细定义末级（合理范围）
- [ ] 每个 SysReq 5 级节点承接 2-5 个 BA IPO（合理范围）
- [ ] 每个 PA 组件承接 2-5 个 SysReq 9 级节点（合理范围）
- [ ] 承接 1 条的已考虑合并，承接 6+ 条的已检查是否需要拆分

### 逆向合理性检查（专用）
- [ ] 每个 EOS PA 组件与制品中实际功能模块一致（无编造组件）
- [ ] 所有反推的性能指标有制品依据（缓存配置、连接池、超时设置等）
- [ ] 不存在"空壳"PA 组件（制品不存在但文档中虚构的组件）
- [ ] 制品中的主要 EOS 运营模块都已覆盖（无遗漏）
- [ ] 制品与文档的差异已记录在差异清单中

### 符合性分析
- [ ] 所有方案节点都有符合性分析（完整率 100%）
- [ ] 每个符合性分析说明了方案如何满足所承接的需求
- [ ] 符合性分析包含验证方法

### 性能指标跨层一致性
- [ ] 性能指标从 PA → SysReq → BA → SR → OR 逐层传递，无断层
- [ ] PA 组件指标不低于 SysReq 要求（逆向中：SysReq 指标不高于 PA 实际能力）
- [ ] 所有反推指标标注"反推值 ⚠️ 待确认"
