# 敏捷式流程第8步：文档完善和发布
**Agile Step 8: Document Completion and Release**

**文档版本**：v1.0
**创建日期**：2026-05-28
**对应场景**：03-agile · 第8步
**对应 AI 提示词**：`ai-prompts.md`（同目录）

## 一、目的

完善增量更新后的文档，生成变更日志，发布新版本。

本步在设计链中承担的角色：
- **输入**：task-01 ~ task-06 的全部增量产出，task-07 产出的验证报告
- **最终产出**：更新后的完整五层文档（增量更新版本），变更日志（按层分类的变更明细），版本发布说明

### 本步与三条工程化设计原则的关系

| 原则 | 在本步中的体现 |
|------|--------------|
| **详概分离** | 本步不涉及新的设计工作，仅做文档完善、问题修复和版本管理。基线区与增量区合并为新版本基线 |
| **文档结对** | 本步验证所有文档结对关系的完整性——验证报告中的问题通过回溯修复后，重新检查受影响的结对关系 |
| **基于资产** | 本次迭代的完整设计文档集为发布资产。v2 增量区与 v1 基线区合并，生成新的 v2 基线（冻结·只读），作为下一次迭代的资产 |

### 文档结对关系

本步执行文档发布和版本管理的收尾工作：

| 结对 | 内容 | 工作方式 |
|------|------|---------|
| 增量内容 ↔ 基线文档 | 合并生成新版本 | 增量标记内容整合进基线，v2 增量区→新的 v2 基线区 |
| 验证报告问题 ↔ 修复后的文档 | 闭环处理 | 根据验证报告问题清单逐项修复（回溯到 t01~t06 对应步骤修复） |
| 变更日志 ↔ 各层文档 | 变更明细记录 | 按层分类汇总，生成变更日志 |

## 三、操作流程

本步按五个阶段的顺序执行：问题修复 → 变更汇总 → 生成日志 → 发布准备 → 终验。

核心驱动逻辑：先处理验证报告的问题，再汇总变更并生成发布物。确保所有问题闭环后再发布。

操作流程概览：

```
[阶段一] 审查修复               [阶段二] 变更汇总               [阶段三~四] 发布准备
  按验证报告逐项修复              按层分类汇总变更                 生成变更日志
  回溯到对应步骤                   影响分析摘要                    生成版本发布说明
[阶段五] 终验
  人类确认迭代完成
```

### 阶段一：审查修复

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 按审查修复流程处理问题 | AI | 根据验证报告的问题清单逐项修复；格式性问题直接修复，结构性问题提修复方案标记 ⚠️ |

### 阶段二：变更汇总

|---|------|--------|------|

### 阶段三：生成日志

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 3 | 生成变更日志 | AI | 变更明细、影响分析摘要 |

### 阶段四：发布准备

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 4 | 生成版本发布说明 | AI | 版本号、发布日期、本次迭代范围摘要 |

### 阶段五：终验

### 启动方式

人类向 AI 发出以下指令即可启动：

AI 动作序列：读取本目录下 task-definition.md + ai-prompts.md → 加载 task-01~task-06 的全部增量产出 → 加载 task-07 产出的验证报告 → 加载完整基线文档集 → 按上述五阶段流程执行 → 全部完成后通知人类终验。

## 四、审核要点

- [ ] 变更日志准确性（按层分类、无遗漏）
- [ ] 新版本号确认
- [ ] 验证报告问题闭环处理（所有问题已修复或标记）
- [ ] 所有文档版本号一致更新
- [ ] 超链接有效性（跨文档引用是否正确）

## 五、质量门禁

- [ ] 审查修复流程完成（验证报告问题全部处理）
- [ ] 变更日志已生成（按层分类汇总）
- [ ] 版本发布说明已生成（版本号+日期+范围）
- [ ] 所有文档版本号一致更新
- [ ] 增量内容已正确合并进基线文档
- [ ] 终验通过（人类确认）
- [ ] 运行 rule-conformance-review 进行符合性审查，按审查修复流程处理问题
