# task-07：完整性和一致性验证

**对应场景**：02-reverse-engineering · 第7步
**对应 AI 模板**：`03-eos-system-design-ai-support/02-reverse-engineering-eos-system-design/workflow-prompts.md` 第7步
**版本**：v1.0
**创建日期**：2026-05-21

---

## 目标

验证完整逆向推导后各层之间的追溯链，输出验证报告。核心验证内容包含：1:1 分配约束验证、N:1 承接支持验证、双向追溯验证、符合性分析。

## 上下文继承

```
来源：第1~6步产出
产出：追溯矩阵 + 验证报告（含问题清单和建议）
```

## 输入

| 输入 | 说明 |
|------|------|
| 全部设计文档（OR → SR → BA → SysReq → PA） | task-01 ~ task-06 的全部产出 |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 分析活动：全链路合并验证 | 人类+AI | 在逐层验证之前，先对全部五层进行整体合并验证：汇总各步的分析结果（PA→SysReq、SysReq→BA、BA→SR、SR→OR），形成完整的全链路追溯视图；评估各层之间的语义一致性，识别跨层矛盾。此步骤涵盖：汇总各步推导结果→构建端到端追溯链→识别跨层不一致→评估整体推导合理性 |
| 2 | 1:1 分配约束验证 | AI | 逐层检查每条详细定义末级的映射：OR末级→SR、SR详细→BA IPO、SysReq 9级→PA |
| 3 | N:1 承接支持验证 | AI | 检查各层方案架构末级节点的承接数量合理性（建议范围 2-5 条/节点，1 条考虑合并、6+条建议拆分） |
| 4 | 双向追溯验证 | AI | OR → PA 和 PA → OR 均需验证 |
| 5 | 反向推导一致性验证 | AI | 逆向推导的上下游内容是否一致 |
| 6 | 生成追溯矩阵 | AI | 端到端映射关系表 |
| 7 | 生成验证报告 | AI | 含问题清单和建议 |

## 人类审核要点

- [ ] 验证报告中的未通过项
- [ ] 逆向推导后的设计是否自洽
- [ ] 是否存在"推导矛盾"——上层推导的结果和下层推导的结果不一致
- [ ] 全链路合并验证中发现的跨层矛盾是否已处理
- [ ] N:1 承接数量是否合理——有无过度集中或过度分散的节点

## 质量门禁

- [ ] 1:1 约束通过率 100%
- [ ] 反向推导一致性通过
- [ ] N:1 承接数量合理（建议 2-5 条/节点），无异常集中或分散
- [ ] 双向追溯（OR→PA 和 PA→OR）全部通过
- [ ] 全链路合并验证完成，跨层矛盾已处理
- [ ] 验证报告已生成
- [ ] 制品-文档差异已按四类型处理（D-R020）
- [ ] **运行 rule-conformance-review 进行完整符合性审查**（建立基线前的关键门禁）
- [ ] 已按审查修复流程处理发现的问题
