# task-01：新需求分析和分类

**对应场景**：03-agile · 第1步
**对应 AI 模板**：`30-eos-system-design-ai-support/03-agile-eos-system-design/workflow-prompts.md` 第1步
**版本**：v1.0
**创建日期**：2026-05-21

---

## 目标

分析新需求的性质和影响范围，确定优先级。

## 上下文继承

```
从哪里来：
  - 直接输入：新需求原始表述（新增或变更需求的原始描述）
  - 参考基线：现有完整系统设计文档集（5 层 + 映射矩阵）

预期产出：
  1. 新需求分析报告（分类 + 影响范围评估 + 优先级排序）

本步决策对后续的影响：
  - 影响范围评估结果决定后续增量操作的覆盖层
  - 优先级排序决定处理顺序
  - 碎片归并结果影响 task-02 的分解粒度
```

## 输入

| 输入 | 说明 |
|------|------|
| 新需求原始表述 | 新增或变更需求的原始描述 |
| 现有完整系统设计文档集 | 基线文档（5 层 + 映射矩阵） |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 新需求分类 | AI | 功能/非功能，新增/修改/删除 |
| 2 | 碎片归并 | AI | 多来源碎片合并为完整需求 |
| 3 | 分析活动：影响范围评估 | AI | 需求说明符合性分析（逐层检查受影响节点）+ 需求性能符合性分析（性能指标的影响范围评估） |
| 4 | 优先级排序 | AI | 建议优先级，标记待确认 |
| 5 | 结果审核 | 人类 | 确认优先级 |

## 人类审核要点

- [ ] 优先级判定（P0-P3）— 需人工确认
- [ ] 影响范围评估是否准确
- [ ] 碎片归并是否正确
- [ ] 需求分类（功能/非功能、新增/修改/删除）是否合理
- [ ] 性能指标是否被正确识别并归入非功能分支

## 质量门禁

- [ ] 分析活动：影响范围评估已执行
- [ ] 新需求分析报告完整
- [ ] 影响范围评估完成
- [ ] 功能/非功能需求分流正确（功能走 BA 链路、非功能走 NFR 平行链路）
- [ ] 运行 rule-conformance-review 进行符合性审查
- [ ] 按审查修复流程处理问题
