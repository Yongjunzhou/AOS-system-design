## 一、第1步产出范例

### 1.1 EOS 原始需求收集和初步整理

```
OR-001: 组织架构与角色管理
├─ 需求描述：系统应支持企业组织架构的层级定义、角色权限配置和人员管理，形成统一的组织管理体系
├─ 原始碎片清单：
│  ├─ "组织架构树要支持多级部门，可以拖拽调整层级" — 人力资源部 2026-05-01
│  └─ "角色权限要灵活，不同部门看到的菜单和功能不一样" — IT运维部 2026-05-02
├─ 包含的性能指标：组织树加载 < 2s
├─ 依赖/冲突关系：无
└─ 整理说明：合并HR与IT运维的碎片，归结为组织架构与权限管理需求

OR-002: 业务流程建模与执行
├─ 需求描述：系统应支持业务流程的图形化建模、版本发布、自动执行和实时监控
├─ 原始碎片清单：
│  ├─ "业务流程要能像流程图一样画出来，审批节点要灵活配置" — 业务管理部 2026-05-03
│  ├─ "流程执行到哪一步了要能看到，超时要自动提醒" — 运营管理部 2026-05-04
│  └─ "流程版本更新后，旧版本的实例要继续跑完" — 流程设计者 2026-05-05
├─ 包含的性能指标：流程实例启动 < 1s
├─ 依赖/冲突关系：依赖 OR-001（流程执行需要组织角色信息）
└─ 整理说明：三个角色碎片合并，体现流程全生命周期诉求

OR-003: 绩效指标体系管理
├─ 需求描述：系统应支持企业绩效指标（KPI）的定义、数据采集、自动计算和可视化展示
├─ 原始碎片清单：
│  ├─ "各部门的KPI要能在系统里配置，数据要自动从流程中抓取" — 战略管理部 2026-05-06
│  └─ "绩效看板要实时更新，红黄绿灯标出异常指标" — 管理层 2026-05-07
├─ 包含的性能指标：指标计算延迟 < 5s
├─ 依赖/冲突关系：依赖 OR-002（指标数据来源业务流程）
└─ 整理说明：战略管理部关注指标配置，管理层关注展示，合并为指标管理需求

OR-004: 运营报表与分析
├─ 需求描述：系统应提供多维度运营数据分析能力，支持自定义报表和可视化图表
├─ 原始碎片清单：
│  ├─ "报表要支持拖拽式自定义，图表类型要丰富" — 业务分析部 2026-05-08
│  └─ "分析结果要能导出成Excel和PDF，方便周报使用" — 运营管理部 2026-05-09
├─ 包含的性能指标：报表生成时间 < 5s
├─ 依赖/冲突关系：依赖 OR-003（报表数据来源绩效指标）
└─ 整理说明：两个角色关注点互补，合并为分析与报表需求

OR-005: 系统集成与主数据管理
├─ 需求描述：系统应支持与现有ERP、OA等系统的数据集成，并提供统一的主数据管理能力
├─ 原始碎片清单：
│  ├─ "流程中的数据要能和ERP对接，比如采购审批后自动同步到ERP" — IT运维部 2026-05-10
│  └─ "部门、人员等基础数据最好统一管理，各模块共享" — 数据管理部 2026-05-11
├─ 包含的性能指标：接口响应时间 < 3s
├─ 依赖/冲突关系：无
└─ 整理说明：集成与主数据管理合并为统一的数据与集成需求

OR-NFR-001: 系统性能与可用性要求
├─ 需求描述：系统应保障企业级并发使用，确保关键业务持续可用
├─ 包含的性能指标：
│  ├─ 平均响应时间 < 500ms，P99 < 2s
│  ├─ 支持 500 并发用户
│  ├─ 系统可用性 99.9%（月停机 < 43分钟）
│  └─ 敏感数据加密存储和传输
└─ 整理说明：来自信息技术部的总体非功能要求
```

```
├─ 末级分解：
│  ├─ OR-NFR-001.1: 500并发下响应时间达标 → SR-NFR 承接
│  ├─ OR-NFR-001.2: 系统可用性 99.9% → SR-NFR 承接
│  └─ OR-NFR-001.3: 数据加密存储与传输 → SR-NFR 承接
```

## 二、第2步产出范例

### 2.1 EOS 相关方需求功能部分详细定义

```
架构末级节点：SR-F-001（组织架构管理功能）
详细定义末级节点：
  - ID: SR-F-001.1
    标题: 组织架构树创建与维护
    描述: 管理员创建企业组织架构的多级树形结构，支持部门的新增、修改、合并和撤销
    分配目标: BA-组织树管理 [1:1映射]
    验收标准: 组织树支持无限层级，节点拖拽调整，变更记录可追溯
    对上层的符合性: 直接满足 SR-F-001 对组织架构管理的需求

  - ID: SR-F-001.2
    标题: 角色定义与权限分配
    描述: 管理员定义角色权限模板，为组织节点分配操作权限和数据权限
    分配目标: BA-角色权限管理 [1:1映射]
    验收标准: 角色支持继承，权限分配到功能按钮级别，支持批量分配
    对上层的符合性: 满足 SR-F-001 对权限管理的需求

  - ID: SR-F-001.3
    标题: 人员管理与部门归属
    描述: 管理员维护人员信息，设置人员在组织树中的归属位置
    分配目标: BA-人员管理 [1:1映射]
    验收标准: 人员可多部门兼职，归属变更记录完整
    对上层的符合性: 满足 SR-F-001 对人员管理的需求

架构末级节点：SR-F-002（业务流程管理功能）
详细定义末级节点：
  - ID: SR-F-002.1
    标题: 图形化流程建模
    描述: 设计者使用拖拽方式创建业务流程，配置审批节点、条件分支和自动动作
    分配目标: BA-流程建模 [1:1映射]
    验收标准: 支持 BPMN 标准，节点类型齐全（审批/条件/自动/子流程），拖拽流畅
    对上层的符合性: 满足 SR-F-002 对流程建模的需求

  - ID: SR-F-002.2
    标题: 流程版本管理
    描述: 管理流程模板的版本生命周期，支持发布、下线和版本回滚
    分配目标: BA-流程版本管理 [1:1映射]
    验收标准: 历史版本可追溯，运行中实例不受版本升级影响
    对上层的符合性: 满足 SR-F-002 对版本管理的需求

  - ID: SR-F-002.3
    标题: 流程实例执行与路由
    描述: 根据流程定义自动创建实例，按条件路由到各审批节点
    分配目标: BA-流程执行 [1:1映射]
    验收标准: 实例创建 < 1s，路由逻辑正确，超时自动流转
    对上层的符合性: 满足 SR-F-002 对流程执行的需求

  - ID: SR-F-002.4
    标题: 流程监控与告警
    描述: 实时展示流程实例的运行状态，超时和异常触发告警通知
    分配目标: BA-流程监控 [1:1映射]
    验收标准: 状态刷新 < 3s，告警通知及时（短信/邮件/站内信）
    对上层的符合性: 满足 SR-F-002 对监控的需求

架构末级节点：SR-F-003（绩效指标管理功能）
详细定义末级节点：
  - ID: SR-F-003.1
    标题: 绩效指标定义
    描述: 管理者定义KPI的名称、计算公式、目标值和数据来源
    分配目标: BA-指标定义 [1:1映射]
    验收标准: 支持算术/聚合/条件三种公式类型，数据来源可配置
    对上层的符合性: 满足 SR-F-003 对指标定义的需求

  - ID: SR-F-003.2
    标题: 绩效数据采集
    描述: 系统自动从流程执行数据和外部系统采集绩效指标原始数据
    分配目标: BA-数据采集 [1:1映射]
    验收标准: 采集任务可调度，失败自动重试，采集日志完整
    对上层的符合性: 满足 SR-F-003 对数据采集的需求

  - ID: SR-F-003.3
```
