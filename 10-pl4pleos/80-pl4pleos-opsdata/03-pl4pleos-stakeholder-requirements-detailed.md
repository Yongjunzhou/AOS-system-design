### SR-F-001：建立标准化的设计准则体系 — 详细定义

**相关方**：企业架构团队 | **优先级**：P0

**需求描述**：建立和维护一套标准化的设计准则，涵盖相关方需求的规范化、分解、冲突检测、重复检测、BA分配、SysReq分配、PA分配等全流程。保证系统设计的一致性和可控性，避免不同项目间的设计方法差异。

**约束条件**：准则必须支持两轨模型（功能需求、非功能需求）。

**详细定义末级节点**：

| 末级节点 | 描述 | 分配目标（BA IPO） |
|---------|------|-------------------|
| SR-F-001.1 | 建立≥5条覆盖全流程的核心准则，含详细规范说明 | → BA: 准则文档编制 IPO |
| SR-F-001.2 | 每条准则提供完整示例 | → BA: 准则示例编制 IPO |
| SR-F-001.3 | 准则覆盖功能需求和非功能需求两轨流程 | → BA: 两轨准则验证 IPO |

**验收标准**：
1. 至少包含5条核心准则
2. 每条准则都有详细的规范说明和示例
3. 准则涵盖功能需求和非功能需求两轨流程

**Pipeline 实现**：
- 准则0：占位符规范与两阶段工作流程
- 准则1：需求规范化与初步设计
- 准则2：SR→BA映射
- 准则3：BA→SysReq映射
- 准则4：SysReq→PA映射
- 位置：`10-eos-pipeline-system-design-specification/` 和各 Task 中的 guidelines

### SR-F-002：建立可复用的文档模板库 — 详细定义

**相关方**：企业架构团队 | **优先级**：P0

**需求描述**：开发和维护相关方需求、业务架构、系统需求、产品架构四层的文档模板，支持新项目快速启动。降低文档编制的工作量和错误率，加快项目启动速度。

**约束条件**：模板必须基于设计准则的要求。

**详细定义末级节点**：

| 末级节点 | 描述 | 分配目标（BA IPO） |
|---------|------|-------------------|
| SR-F-002.1 | 创建相关方需求模板（含功能+非功能） | → BA: SR 模板编制 IPO |
| SR-F-002.2 | 创建业务架构模板（含 IPO 标准格式） | → BA: BA 模板编制 IPO |
| SR-F-002.3 | 创建系统需求模板（含功能+非功能） | → BA: SysReq 模板编制 IPO |
| SR-F-002.4 | 创建产品架构模板 | → BA: PA 模板编制 IPO |
| SR-F-002.5 | 每个模板提供详细填写指南 | → BA: 模板指南编制 IPO |

**验收标准**：
1. 提供4套完整的文档模板
2. 每个模板都包含详细的填写指南
3. 模板支持非功能需求独立处理

**Pipeline 实现**：
- SR 模板：`sr-template.md` + `sr-nfr-template.md`
- BA 模板：`ba-template.md`
- SysReq 模板：`sysreq-template.md` + `sysreq-nfr-template.md`
- PA 模板：`pa-template.md`

### SR-F-004：建立功能需求和非功能需求的两轨处理模型 — 详细定义

**相关方**：企业架构团队 | **优先级**：P0

**需求描述**：明确功能需求（SR-F → BA → SysReq-F → PA）和非功能需求（SR-NF → SysReq-NF → PA）的不同处理路径，确保两类需求都能被正确处理和追溯。

**详细定义末级节点**：

| 末级节点 | 描述 | 分配目标（BA IPO） |
|---------|------|-------------------|
| SR-F-004.1 | 定义功能需求流完整路径（SR-F → BA → SysReq-F → PA） | → BA: 功能需求流程定义 IPO |
| SR-F-004.2 | 定义非功能需求流完整路径（SR-NF → SysReq-NF → PA） | → BA: 非功能需求流程定义 IPO |
| SR-F-004.3 | 定义两轨在产品架构层面的集成规则 | → BA: 两轨集成规则定义 IPO |

**验收标准**：
1. 清晰定义两轨流程
2. 明确非功能需求不经过BA层
3. 明确非功能需求作为系统约束的处理方式

**Pipeline 实现**：
- 功能需求流：Task 2（SR→BA）→ Task 3（BA→SysReq）→ Task 5（SysReq→PA）
- 非功能需求流：Task 4（SR-NF→SysReq-NF）→ Task 5（SysReq-NF 约束验证）
- 两轨集成：Task 5 中的非功能约束验证和权衡决策
