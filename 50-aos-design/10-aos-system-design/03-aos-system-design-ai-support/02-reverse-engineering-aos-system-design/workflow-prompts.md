# AOS 系统设计逆向工程：提示词模板集
**AOS System Design Reverse Engineering: Workflow Prompt Templates**

**用途**：AI 按 8 步流程依次执行逆向推导，每个步骤使用对应的提示词模板。AI 将提示词中的 `[占位符]` 替换为实际内容后执行。

## 依据B说明

本提示词模板集是产品L（系统设计流水线）的 **AI辅助文档构件**，其上层产品架构节点为 `03-aos-system-design-ai-support/02-reverse-engineering/`。模板默认演示"软件产品"类型的系统设计逆向推导过程（从 PA 到上游文档），适用范围包括对产品M（AOS）、产品L（流水线）自身或其他软件产品的逆向推导。

**适用场景**：
- 产品架构（PA）已完成，需从 PA 沿五层结构反向追溯生成 SysReq/BA/SR/OR
- PA 文档质量不足，需要从现有产物目录和代码中重建 PA 后再进行逆向推导

**使用时注意**：
- 当 PA 已存在且质量满足要求时，可直接从第3步开始执行，跳过第1-2步
- 当 PA 不存在或质量严重不足时，需先执行前置准备（第1-2步）从产物目录和代码重建 PA
- 所有提示词模板中的 `[占位符]` 在执行时替换为实际内容
- 模板中涉及的术语在生成文档时需确保与目标产品实际业务一致

## 跨步骤上下文管理说明

### 上下文传递方式

8步逆向推导流程中，每步逆向推导的结果是下一步的输入。AI 在执行每一步前，应确保完整继承了上一步的关键决策：

| 继承内容 | 说明 | 保存位置 |
|---------|------|---------|
| 产物分析结果 | 产物总览清单、组件识别、功能清单 | 第1步输出 |
| 架构定义 | PA 组件划分、SysReq 架构、BA IPO | 各步的产品数据文件 |
| 反向推导评级 | 推导结果的 AAA/A/B/C 评级汇总 | 上一步的决策记录 |
| 制品差异记录 | PA与现有产物不一致项 | 差异清单 |
| 待确认项 | 上一步标记的 ⚠️ 项 | 汇总清单 |

### 决策记录机制

每步完成后，AI 将本步的关键决策记录在**本步决策记录模板**中，供后续步骤引用。

### 当需要回溯修正

如果在第 N 步发现问题需要修改第 M 步（M < N）的产出：
1. 评估影响范围（见[AI决策框架 -> 跨步骤影响范围表](../02-ai-decision-framework-aos.md#72-跨步骤影响范围表)）
2. 逆向工程的影响范围更广，因为每步推导依赖前一步的结果
3. 修改后检查下游所有推导链路是否需要同步调整
4. 标注变更记录

## 第1步：现有产物梳理与架构分析

### 上下文继承

**从哪里来**：
  - 直接输入：项目产物目录（现有设计文档、PA定义、产品数据）
  - 参考输入：项目代码库（如有，当PA文档不足时辅助使用）
  - 参考基线：系统设计准则、各层规范

**去哪里**：
  - 本步产出（产物总览清单 + 功能模块清单）→ 第2步（PA 文档编写）的输入

### 目标
全面梳理现有产物（含PA文档、产品数据、已有技术文档及代码），理解产品全貌和架构现状，建立完整的产物全局视图，为后续逆向推导提供基础。

### 输入
- 项目产物目录（所有设计文档、指南、AI辅助文档、产品数据模板）
- 项目代码库（如有，当PA不足时辅助分析）
- 系统设计准则和规范

### 涉及规范
- [产品架构规范 §2 架构定义规范](../01-aos-system-design-standards/06-aos-product-architecture-specification.md#二架构定义规范)

### 决策框架参考

| 决策场景 | 处理方法 | 参考章节 |
|---------|---------|---------|

| 组件识别粒度 | 可独立部署、1-2周开发周期 | 按规范执行 |

## 并行说明

在逆向工程 8 步流程中，以下分支可以**并行执行**：

| 并行组 | 步骤 | 说明 |
|--------|------|------|
| **功能分支** | 第2步 → 第3步 → 第4步 → 第5步 | PA→SysReq→BA→SR 功能分支的连续反向推导 |
| **非功能分支** | 第3c步（SysReq-NFR）→ 第5c步（SR-NFR） | 非功能需求独立于功能分支，从产品配置直接反推 |
| **合并** | 第6步 | 等待功能分支和非功能分支都完成后，合并反推 OR |

因此实际时间线为：
```
第1步 → [第2步 → 第3b+3c步（功能+非功能并行）→ 第4步 → 第5b+5c步（功能+非功能并行）] → 第6步 → 第7步 → 第8步
```
