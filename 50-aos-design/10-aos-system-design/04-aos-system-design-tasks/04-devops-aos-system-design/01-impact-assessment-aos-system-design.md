# task-01：快速影响范围评估

**对应场景**：04-devops · 第1步

**创建日期**：2026-05-21

---

## 目标

快速判断问题的影响范围，选择合适的处理模式。

## 上下文继承

```
从哪里来：
  - 直接输入：问题报告或紧急需求（bug、性能问题、紧急需求描述）
  - 参考基线：现有完整AOS基线文档

预期产出：
  1. 影响范围评估报告（问题分类 + 影响定位 + 处理模式选择）

本步决策对后续的影响：
  - 处理模式选择（模式1/2/3）决定后续执行的 task
  - 影响节点定位为后续修复提供精确目标
```

---

## 输入

| 输入 | 说明 |
|------|------|
| 问题报告或紧急需求 | bug、性能问题、紧急需求描述 |
| 现有完整系统设计文档集 | 基线文档 |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 问题分类 | AI | bug / 功能需求 / 性能优化 |
| 2 | 紧急程度分级 | AI | P0（立即处理）/ P1（今天修复）/ P2（本周修复） |
| 3 | 分析活动：影响范围快速评估 | AI | 需求说明符合性分析（定位受影响的层和节点）+ 需求性能符合性分析（评估性能退化范围） |
| 4 | 处理模式选择 | AI | 模式1（快速修复·仅PA）/ 模式2（局部设计·SysReq+PA）/ 模式3（完整设计·多层） |
| 5 | 结果审核 | 人类 | 确认问题分类和处理模式 |

## 人类审核要点

- [ ] 优先级判定（P0/P1/P2）— 需确认
- [ ] bug 根本原因诊断是否准确
- [ ] 处理模式的最终决定
- [ ] 影响范围评估是否覆盖了所有受影响的层和节点

## 质量门禁

- [ ] 分析活动：影响范围快速评估已执行
- [ ] 问题分类完成
- [ ] 处理模式已选定
- [ ] 影响范围已定位
- [ ] 发布前更新所有修改过的文档版本号
- [ ] 按审查修复流程处理问题

## 产出

| 文件 | 说明 |
|------|------|
| 影响范围评估报告 | 本步产出，指导后续修复步骤 |
