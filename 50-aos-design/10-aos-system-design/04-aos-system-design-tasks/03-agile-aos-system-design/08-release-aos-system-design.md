# task-08：版本发布

**对应设计指南**：[敏捷式系统设计指南 第8步](../../02-aos-system-design-guidelines/03-agile-aos-system-design-guide.md#第8步文档完善和发布)

---

## 目标

打包增量更新为正式发布版本。

## 上下文继承

```
从哪里来：
  - task-01 ~ task-06 的全部增量产出
  - task-07 产出的验证报告

最终产出：
  - 更新后的完整五层文档（增量更新版本）
  - 变更日志（按层分类的变更明细）
  - 版本发布说明
```

## 输入

| 输入 | 说明 |
|------|------|
| 全部增量更新后的文档 | task-01 ~ task-07 的全部产出 |
| 验证报告 | task-07 产出 |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 按审查修复流程处理问题 | AI | 根据验证报告的问题清单逐项修复：格式性问题→直接修复；内容缺失→补充；结构性问题→提方案标注待确认 |
| 2 | 版本号更新 | AI | 所有文档版本号统一更新 |
| 3 | 版本标记 | 人类 | 设置版本基线标记 |
| 4 | 发布说明生成 | AI | 汇总增量变更日志 |
| 5 | 最终审核 | 人类 | 确认可发布状态 |

## 人类审核要点

- [ ] 变更日志是否准确完整
- [ ] 确认新版本号
- [ ] 验证报告的问题清单是否已全部处理
- [ ] 各层文档之间的超链接是否有效

## 质量门禁

- [ ] 按审查修复流程处理的问题已全部闭环
- [ ] 所有更新后的文档已合并到基线
- [ ] 变更日志已生成
- [ ] 版本发布说明已生成
- [ ] 版本号已确认
- [ ] 运行 conformance-review 进行最终符合性审查
- [ ] 按审查修复流程处理问题

## 产出

| 文件 | 说明 |
|------|------|
| 版本发布包 | 新的基线版本 |
