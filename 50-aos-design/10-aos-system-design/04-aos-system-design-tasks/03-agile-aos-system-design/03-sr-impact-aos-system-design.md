## 目标

将新末级需求映射或更新到相关方需求，输出增量定义。

## 上下文继承

```
从哪里来：
  - task-02 产出：新末级需求清单
  - 参考基线：现有 SR 文档（架构定义 + 详细定义）

去哪里：
  - 本步产出 → task-04（BA 影响分析）的 SR 增量定义
  - 同步产出 → 更新的 OR → SR 映射关系表
```

## 输入

| 输入 | 说明 |
|------|------|
| 新末级需求清单 | task-02 产出 |
| 现有 SR 文档（基线） | 基线中的 SR 层 |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 分析活动：架构定义验证 | AI | 需求说明符合性分析（新末级需求是否能由现有 SR 架构末级节点承接）+ 需求性能符合性分析（新需求性能指标是否可在现有 SR 框架内量化） |
| 2 | 映射策略选择 | AI | 选项A：映射到现有 SR 架构末级节点（更新承接关系+符合性分析）；选项B：创建新的 SR 架构末级节点（定义新节点+建立映射） |
| 3 | SR 增量条目定义 | AI | 使用增量标记（`+`/`~`）生成更新内容 |
| 4 | 同步更新映射关系表 | AI | OR → SR 映射 |
| 5 | 结果审核 | 人类 | 确认 SR 更新 |

## 人类审核要点

- [ ] 映射策略选择（复用 vs 新建）— 需确认
- [ ] 增量标记是否准确
- [ ] 功能分支（SR-F）与非功能分支（SR-NFR）处理是否正确
- [ ] SR 详细定义末级是否满足 1:1 约束

## 质量门禁

- [ ] 分析活动：架构定义验证已执行
- [ ] SR 增量定义完成（架构定义 + 详细定义）
- [ ] 功能/非功能分支分离完成
- [ ] 映射关系表已更新
- [ ] 运行 rule-conformance-review 进行符合性审查
- [ ] 按审查修复流程处理问题
