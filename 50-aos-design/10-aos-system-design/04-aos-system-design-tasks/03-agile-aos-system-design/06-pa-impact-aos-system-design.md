# task-06：产品架构影响分析

**对应场景**：03-agile · 第6步
**对应 AI 模板**：`03-aos-system-design-ai-support/03-agile-aos-system-design/workflow-prompts.md` 第6步
**对应设计指南**：[敏捷式系统设计指南 第6步](../../02-aos-system-design-guidelines/03-agile-aos-system-design-guide.md#第6步产品架构影响分析)

---

## 目标

将 SysReq 增量映射到产品架构，更新 PA 构件定义。

## 上下文继承

```
从哪里来：
  - task-05 产出：SysReq 增量定义 + SysReq → PA 骨架映射
  - 参考基线：现有 PA 文档（架构定义 + 详细定义）

去哪里：
  - 本步产出 → task-07（验证）的 PA 增量定义
  - SysReq → PA 映射关系表已更新
```

## 输入

| 输入 | 说明 |
|------|------|
| SysReq 增量定义 | task-05 产出 |
| 现有 PA 文档（基线） | 基线中的 PA 层 |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 分析活动：PA 影响分析 | AI | 需求说明符合性分析（SysReq 增量是否能由现有 PA 组件承接）+ 需求性能符合性分析（修改是否影响性能指标范围） |
| 2 | PA 增量定义 | AI | 更新组件定义、映射关系 |
| 3 | 非功能约束影响的增量评估 | AI | 检查 NFR 变化是否影响 PA |
| 4 | 结果审核 | 人类 | 确认 PA 更新 |

## 人类审核要点

- [ ] 组件划分是否合理
- [ ] 技术决策是否合理
- [ ] 新增组件的 PA 分类体系是否正确
- [ ] 详细定义是否同步更新
- [ ] 映射关系是否正确—SysReq → PA 对应完整

## 质量门禁

- [ ] 分析活动：PA 影响分析已执行
- [ ] PA 增量定义完成（架构定义 + 详细定义）
- [ ] 组件粒度符合"可独立部署和审查"标准
- [ ] 映射关系表已更新
- [ ] 运行 conformance-review 进行符合性审查
- [ ] 按审查修复流程处理问题
