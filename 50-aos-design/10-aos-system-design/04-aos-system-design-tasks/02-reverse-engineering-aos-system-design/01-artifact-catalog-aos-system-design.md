# task-01：现有产物梳理与架构分析

**对应场景**：02-reverse-engineering · 第1步
**对应 AI 模板**：`03-aos-system-design-ai-support/02-reverse-engineering-aos-system-design/workflow-prompts.md` 第1步
**对应设计指南**：[AOS 逆向工程系统设计指南 第1步](../../02-aos-system-design-guidelines/02-reverse-engineering-aos-system-design-guide.md#第1步代码和图纸分析)

---

## 目标

全面梳理产品A（AOS企业运营体系）现有的所有产物，理解当前的目录结构和文件组织方式，建立全局视图，为后续逆向推导提供基础。

## 上下文继承

```
来源：项目目录结构、现有AOS设计文档/指南/AI辅助文档/产品数据
产出：产物总览清单（含覆盖盲区标记）
```

## 输入

| 输入 | 路径 |
|------|------|
| 项目目录结构 | 整个 AOS 仓库根目录 |
| 现有设计文档 | `01-aos-system-design-standards/` |
| 现有设计指南 | `02-aos-system-design-guidelines/` |
| 现有 AI 辅助文档 | `03-aos-system-design-ai-support/` |
| 现有产品数据模板 | `10-aos-system-product-data/` |
| 通用方法论 | `00-general/` |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 生成完整目录树 | AI | 列出仓库全部目录和文件（排除无关目录） |
| 2 | 按架构层级分类现有产物 | AI | 将每个文件归类到 OR/SR/BA/SysReq/PA 中对应该层结构 |
| 3 | 识别产物类型 | AI | 标注每份产物的类型：规范定义/设计指南/操作模板/产品数据/其他 |
| 4 | 分析活动：初版架构评估 | 人类+AI | 两类分析：(1) 覆盖盲区分析——识别哪些产物已在但未正式化，哪些内容缺失；(2) 层级完整性分析——判断各架构层级（OR/SR/BA/SysReq/PA）的覆盖完整度 |
| 5 | 汇总输出 | AI | 按模板生成产物总览清单 |

## 人类审核要点

- [ ] 产物清单是否完整？是否有遗漏的目录或文件？
- [ ] 覆盖盲区的判断是否准确？
- [ ] 分类是否合理——某产物是否应该归到不同的架构层级？
- [ ] 初版架构评估的覆盖盲区和层级完整性判断是否准确？有无需要补充的调研方向？
- [ ] 各架构层级（OR/SR/BA/SysReq/PA）的覆盖程度分级是否合理？

## 质量门禁

- [ ] 产物总览清单已生成
- [ ] 覆盖盲区已标记
- [ ] 各架构层级覆盖完整度已评估
- [ ] 产物分类符合五层架构体系的语义要求
- [ ] 产物与架构层级的对应关系无可疑交叉
- [ ] 人类开发者已审核确认清单完整性
- [ ] 已按审查修复流程处理发现的问题
