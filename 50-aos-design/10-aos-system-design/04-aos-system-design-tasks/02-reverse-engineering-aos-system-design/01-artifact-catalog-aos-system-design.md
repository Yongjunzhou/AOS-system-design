## 目标

全面梳理产品A（AOS企业运营体系）的现有所有产物（含PA文档、产品数据、已有技术文档及代码），理解当前的目录结构和文件组织方式，建立完整的产物全局视图，为后续逆向推导提供基础。

## 上下文继承

```
来源：项目产物目录结构、现有AOS设计文档/指南/AI辅助文档/产品数据（代码库仅当PA不足时辅助使用）
产出：产物总览清单（含覆盖盲区标记） + 功能模块清单
```

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 现有产物目录梳理 | AI | 遍历现有产物目录，将每个文件归类到 OR/SR/BA/SysReq/PA 对应层级，标注类型（规范/指南/模板/产品数据/其他）；识别文档覆盖盲区 |
| 2 | 产物类型与层级分类 | AI | 按五层结构分析各层架构定义的完整度和详细定义的覆盖范围，标注缺失或质量不足的层级和节点 |
| 3 | 功能模块清单提取（辅助分析） | AI | 从现有产物和代码（如有）中提取功能模块清单，按四大分类（前端/后端/数据/基础设施）归类；每个功能模块对应一个或多个产品架构构件 |
| 4 | 分析活动：需求说明符合性分析 + 需求性能符合性分析 | 人类+AI | 两类分析：(1) 需求说明符合性分析——评估现有产物是否覆盖完整五层结构，识别覆盖盲区；(2) 需求性能符合性分析——从产物的实现性能反向推断应满足的性能指标要求，标记待补充项 |
| 5 | 汇总输出 | AI | 按模板生成产物总览清单（含覆盖盲区标记）和功能模块清单 |

## 人类审核要点

- [ ] 产物总览清单是否完整？是否有遗漏的产物或层级？
- [ ] 产物分析中识别的功能模块分类是否准确（前端/后端/数据/基础设施）？
- [ ] 覆盖盲区的判断是否准确？
- [ ] 需求说明符合性分析中发现的覆盖盲区是否已准确识别？
- [ ] 需求性能符合性分析中识别的性能指标来源是否合理？
- [ ] 产物与架构层级的对应关系是否有交叉？

## 质量门禁

- [ ] 功能模块清单已提取，按四大分类归类完成
- [ ] 产物总览清单已生成
- [ ] 覆盖盲区已标记
- [ ] 需求说明符合性分析 + 需求性能符合性分析已完成
- [ ] 各架构层级覆盖完整度已评估
- [ ] 产物分类符合五层架构体系的语义要求
- [ ] 性能指标反推值需标注来源和"反推值"（参见 D-R021 置信度分级）
- [ ] 制品差异已按四类型处理：补充缺失/标注不存在/以PA为准/判断是否活跃（参见 D-R020）
- [ ] 人类开发者已审核确认清单完整性
- [ ] 运行 rule-conformance-review 进行符合性审查
- [ ] 已按审查修复流程处理发现的问题
