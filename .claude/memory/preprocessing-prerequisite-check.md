# 预处理链前置校验设计

由用户于 2026-06-06 在变更记录表实施对话中提出，确认新开对话实施。

## 背景

目前各 skill 对前置条件的检查分散且不一致，导致静默跳过（条件不满足时不告知人类该先运行什么）、信息不透明（看到"0 条记录"时不知缺哪个上游步骤）、异常场景缺失（目录不存在等无显式指引）等问题。

## 依赖链

```
ort00（文本化+切分建议）
  → ort01（切分执行）
    → ort02（澄清建议）
      → ort03（澄清迭代）
        → ort04（规范化建议）
          → ort05（规范化执行）
            → t01（OR→SR 映射·下游）
```

## 校验原则

1. **逐级报错，首次即停**：只报告最早的未满足条件。如 ort05 启动时 ort01 还没跑完，报"请先运行 ort01-chunk-execute"，不继续往后查 ort04
2. **两种粒度**：
   - 全局前置（目录/文件不存在）：硬阻断，直接结束
   - 逐文件前置（部分文件未就绪）：列出哪些文件不符合条件，处理符合条件的
3. **指引完备**：指引信息包含 skill 名称和简短说明

## 实施位置

每个 skill 中 `AI 动作序列` 的 **Step 0**（前置校验），在所有处理之前执行。

## 模式泛化

2026-06-06 该模式已泛化写入通用设计规范 §3.6「任务入口前置校验」+ §3.7「Skill 独立执行原则」，覆盖三原则（逐级报错首次即停、两种粒度、指引完备）+ 典型校验内容 + Step 0 实践说明 + 独立执行五规则。
