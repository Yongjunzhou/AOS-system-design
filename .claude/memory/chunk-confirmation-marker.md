# 切片确认标记机制

## 问题

预处理链中，一个 Skill 输出"待确认"状态后，需要人类确认才能进入下游。但"确认"这个动作的操作机制如果没有明确定义，就会出现"入口条件要求状态=X，但没人设这个状态"的设计缺口。

## 解决方案

三步转迁：**AI 在文档末尾写入操作选项 → 人类删行保留选项 → AI 检测后自动同步状态表**。

```
上游 Skill
  │
  ├─ 写入文档正文（含行内标记）
  └─ 追加确认区（两行操作选项）
       │
       ▼
     人类审阅
       │
       ├─ 保留 [确认切片] → 启动下游 → AI 读到后提升状态为"已确认"→ 执行
       └─ 保留 [返回分析] → 启动下游 → AI 读到后更新状态为"待重切"→ 退回
       │
       ▼
    人类只操作文档，不编辑状态表
    AI 从文档检测意图，自动同步状态表
```

## 关键规则

1. **文档即状态**：文档中的确认标记是人类的意图记录，登记表状态表是 AI 维护的同步视图
2. **删行即确认**：人类通过删除不需要的选项行、保留目标行来表达决策
3. **AI 自动转迁**：AI 在入口处检测保留的选项，自动更新状态表（人类不直接编辑）
4. **两行只留一行**：都保留或都删除视为无效，AI 跳过该文件

## 适用范围

本模式适用于预处理链中需要人类线下确认的 Skill 间转迁。与三字段正交模型（[[three-field-orthogonal-status-model]]）的"人类操作文档、AI 同步状态"原则一致。

与 [[skill-action-output-principle]] 的差异：终结原则规定 Skill 输出末端要给出行动建议（做什么+为什么）；本模式规定跨 Skill 的人类确认如何操作（文档删行）。两者互补。
