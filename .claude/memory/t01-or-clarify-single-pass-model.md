# t01-or-clarify 单次通过模型重构（v0.11）

## 三个可复用的设计模式

### 1. 单次通过模型

AI 一轮完成所有阶段（阶段一→二→三→行动清单）后结束，不再有会话级循环等待人类响应。

- **重入驱动**：`last_size`（文件大小变化）是唯一重入信号。人类修改文档→文件大小变→下次启动自动检测并通读续接
- **适用场景**：预处理链中需要人类线下决策的 Skill（t00a 对应"澄清需与相关方沟通"，t00b 同理）
- **与迭代循环的区别**：迭代循环在一个会话内反复等待人类输入；单次通过结束后人类自主决定下一步（重启当前 Skill、启动下游 Skill、或离线操作后再说）

### 2. 行动清单模式

AI 输出分为"动态行动项"和"静态操作指南"两个层面：

- **行动清单**（动态）：AI 每轮结束时按材料状态分组输出具体的行动项（需操作文档选项/需补充澄清记录/候选纳入规范化/其他），告诉人类"对这些材料做什么"
- **人类操作参考**（静态）：文档中的参考表说明每类操作"如何执行"（操作步骤+适用场景）
- **关键原则**：两者分组一致，可对照使用。AI 不替人类决策，只列出选项和状态

### 3. Skill 间接口契约

预处理链中顺序 Skill 间的两条独立门禁：

- **生命周期=已澄清**（质量门）：AI 判定三项条件满足→文档选项 [确认通过]→人类保留→AI 校验后确认
- **人类决策=已确认**（批次选材门）：人类在登记表中确认纳入本批→下游 Skill 读取此条件

两字段正交独立，t00b 入口条件为两者同时满足。
