#### v2.40：BA 层级体系图重建

**旧图问题**：把三类东西混在一棵树里——真正的架构层级（L4→L5→L6）、Phase B 操作步骤（输出文档分配/资产建议/完备性检查）、横切制品（双视角映射/指标/治理缺口）。

**新图**：以 23 输出产品架构资产层级为骨架，从 EOS(L0)→业务域(L1)→A2/Bn(L2) 起锚，L3 分叉、L5 交汇、L6 分归：

```
EOS（L0）
  └── 业务域（L1）
        ├── A2产品（L2）
        │     └── L3 角色类型
        │           └── L4 角色职责
        │                 └── L5 场景业务          ← 交汇层
        │                       └── L6 场景组件
        └── Bn产品（L2）
              └── L3 构件类型
                    └── L4 WBS / 业务模块
                          └── L5 业务输出文档（=场景部件）← 交汇层
                                └── L6 功能元件
```

横切制品（菜单/指标/双视角映射/治理缺口）从树中移出，改为独立的"随附制品"表。

**同步更新**：L4/L5/L6 描述文字 + IPO 模型表（行标题改为"场景业务（A2 L5）"/"业务输出文档（Bn L5）"）+ BA 节点完整要素（改用新层级路径表述）。
