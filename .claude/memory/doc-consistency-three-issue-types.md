文档一致性审查的三类典型问题，按复用价值排序：

1. **时序/状态矛盾（最易漏检）**：同一事实在多处表述中状态更新时机不一致。检出方法：追踪同一状态变量的所有写操作位置，画时间线比对。例如 pre-t03-norm 的 registry-log 更新时机在操作表/启动序列/字段定义/审查流程四处各说各话。

2. **语义聚类错位**：操作按执行顺序排列但跨语义阶段边界，导致阶段标题与实际内容不匹配。检出方法：不按序号读，按标题聚类检视——每个阶段的标题是否覆盖了全部操作，操作是否回答了该阶段应该回答的问题。

3. **记录完整性缺失**：流程产生了中间/终了决策但未在决策记录模板中设置对应字段。检出方法：遍历所有"人类决策点"和"决策输出"，对照决策记录模板逐一比对是否有对应的字段槽位。

**Why：** 这三类问题在复杂流程文档中反复出现，与文档规模正相关。建立系统化的检出意识比逐字校对更高效。

**How to apply：** 每次审查 Skill 文档时，先针对这三类做定向扫描，再做逐节通读。
