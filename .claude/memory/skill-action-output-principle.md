# Skill 终结必荐后续原则

## 原则陈述

每个 Skill 在 AI 执行完毕后，必须向人类输出结构化的后续行动建议——告知人类"接下来可以做什么"以及"为什么建议这么做"，让人类无需了解 Skill 内部处理逻辑即可决定下一步。

## 核心要素

| 要素 | 说明 |
|------|------|
| **输出时机** | Skill 的 AI 执行部分结束时，即输出产物的最后一步 |
| **输出对象** | 人类操作者 |
| **内容** | 行动项（做什么）+ 原因（为什么） |
| **分组方式** | 按行动类型分组，让人类可批量处理同类操作 |
| **适用范围** | 所有 Skill，贯穿整条流水线 |

## 规范位置

已纳入 **EOS 系统设计规范 §15.4（Skill 终结输出规范）**，作为 Skill 文档规范的一部分。

## 设计意图

- **Skill 边界即人机交接点**：AI 处理完毕不是终点，而是向人类交棒的时机。输出数据本身不足以为人类提供决策支持，必须有显式的后续行动建议
- **降低认知负担**：人类不需要打开 Skill 文档去查"这个输出之后我要做什么"——行动清单直接告诉答案
- **统一交互模式**：整条流水线的 Skill 都遵循同一输出模式，人类形成稳定的操作预期

## 具体实现（t00a 案例）

t01-or-clarify 的 **行动清单模式**（[[t01-or-clarify-single-pass-model#2. 行动清单模式|见行动清单模式]]）是本原则的首次完整实现：

- AI 输出按状态分组的行动清单（需操作文档选项/需补充澄清记录/候选纳入规范化/其他）
- 人类操作参考作为静态指南配合使用
- 后续 Skill 应遵循相同的终结输出结构

## 具体实现（ort00 案例）v2

ort00-chunk-propose 对本原则做了三条扩展，形成更友好的人机交接模型：

### 1. 产出写入持久化

AI 的核心产出（概要、建议）不依赖对话输出，而是写入登记表（或状态表）的对应字段。这样即使对话上下文丢失，人类仍可通过登记表查阅：

| 前置条件 | 写入目标 |
|---------|---------|
| 登记表中有该材料的记录行 | 更新内容摘要、AI 建议等字段 |
| 状态表中有该材料的条目 | 更新处理状态、操作指引等字段 |

### 2. 增值优化建议

AI 在完成主要职责（切片分析）之外，输出轻量的内容质量分析作为增值洞察。优化建议类型包括：结构完整性、模糊项提示、缺失信息、体量提示。这些建议随行动清单直接输出，不写入登记表。

**判断条件**：仅当 Skill 的主要工作涉及对输入材料的通读或结构分析时适用。纯格式转换（如 ort01 切片执行）不适用。

### 3. 末端交互询问

Skill 执行结束时，AI 不直接退出，而是主动询问人类是否需要打开关键文档（登记表/状态表/详情页）查看详情；人类回答"是"则提供文件路径和关注指引，回答"否"则结束。

**设计要点**：
- 此交互是可选交接，不改变执行结果
- 询问对象是登记表等聚合视图（而非单个材料文件），减少交互频次
- 路径后附带"建议关注行"帮助人类快速定位

### 三条扩展的适用范围

这三条扩展继承自 Skill 终结必荐后续原则，适用于预处理链中需要将处理结果持久化到登记表的 Skill。纯设计类 Skill（如 t01 OR→SR 映射）可仅套用原则本体（行动清单模式），无需这三条扩展。
