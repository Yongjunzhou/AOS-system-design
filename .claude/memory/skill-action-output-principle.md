# Skill 终结必荐后续原则

## 原则陈述

每个 Skill 在 AI 执行完毕后，必须向人类输出结构化的后续行动建议——告知人类"接下来可以做什么"以及"为什么建议这么做"，让人类无需了解 Skill 内部处理逻辑即可决定下一步。

## 核心要素

| 要素 | 说明 |
|------|------|
| **输出时机** | Skill 的 AI 执行部分结束时，即输出产物的最后一步 |
| **输出对象** | 人类操作者 |
| **内容** | 行动项（做什么）+ 原因（为什么） |
| **分组方式** | 按行动类型分组，让人类可批量处理同类操作 |
| **适用范围** | 所有 Skill，贯穿整条流水线 |

## 规范位置

已纳入 **EOS 系统设计规范 §15.4（Skill 终结输出规范）**，作为 Skill 文档规范的一部分。

## 设计意图

- **Skill 边界即人机交接点**：AI 处理完毕不是终点，而是向人类交棒的时机。输出数据本身不足以为人类提供决策支持，必须有显式的后续行动建议
- **降低认知负担**：人类不需要打开 Skill 文档去查"这个输出之后我要做什么"——行动清单直接告诉答案
- **统一交互模式**：整条流水线的 Skill 都遵循同一输出模式，人类形成稳定的操作预期

## 具体实现（t00a 案例）

t01-or-clarify 的 **行动清单模式**（[[t01-or-clarify-single-pass-model#2. 行动清单模式|见行动清单模式]]）是本原则的首次完整实现：

- AI 输出按状态分组的行动清单（需操作文档选项/需补充澄清记录/候选纳入规范化/其他）
- 人类操作参考作为静态指南配合使用
- 后续 Skill 应遵循相同的终结输出结构
