2026-06-15 确定 wft02 执行模型：

**执行模型**：废除"两趟"（第一趟输出产品→第二趟角色场景推导），改为两条路径并行从 STR 直接分配。

- 输出产品 BA（流水线视角）：5 级 = 业务定义/A2，{产品类型×模式×域} 三元组分解
- 角色场景 BA（工作台视角）：5 级 = 编排定义/A2，角色履职流程分解

**派生 STR 全对称**：BA 分解中识别的 A1/A2 缺口，由 AI 按 STR 规范生成派生 STR 节点草稿，人类确认后写入 02-*.md。派生 STR 走与原始 STR 完全相同的两条路径——产品视角和场景视角各自分解。不做特例处理。

**覆盖所有业务域**：每个业务域都包括两类产品——输出产品和角色场景。流程与IT域也不例外（输出产品：引擎+业务定义；角色场景：流程配置员、引擎开发者等）。

**收敛方式**：靠多轮 wft02 迭代收敛，无需一次性交叉验证。派生 STR 进入下一轮 wft02 后被各自路径处理。
