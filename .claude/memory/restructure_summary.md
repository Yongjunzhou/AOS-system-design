# AOS 流水线重构方案总结

## 📌 核心确认

基于用户的4个关键确认，重构方案已定型：

1. ✅ **非功能需求直接映射**：SR-NFR → SysReq-NFR，无中间层
2. ✅ **产品架构末级**：前后端组件
3. ✅ **占位符处理**：在各任务阶段直接完成
4. ✅ **任务5和6合并**：功能和非功能需求在Task 5中统一处理

## 🎯 重构目标

| 目标 | 当前状态 | 重构后 |
|------|---------|--------|
| **组织方式** | 资产类型驱动（standards/pipeline/products） | 任务驱动（01-task-*/08-products） |
| **工作单元** | 分散在多个目录 | 每个任务一个独立目录 |
| **易用性** | 新用户需要理解整个系统 | 新用户可以快速找到特定任务 |
| **并行工作** | 困难，前后交联多 | 容易，任务相对独立 |
| **知识积累** | 分散在各处 | 集中在每个任务目录 |

## 🔗 映射关系总览

### 功能需求流

```
SR（功能）
  ↓ [N:1映射]
BA 5级（业务组件）
  ↓ [N:1映射]
SysReq 5级（组件实例）
  ├─ SysReq 6级（功能模块）
  ├─ SysReq 7级（系统用例）
  ├─ SysReq 8级（用例场景）
  └─ SysReq 9级（场景活动）
       ↓ [N:1映射]
PA末级（前后端组件）
```

### 非功能需求流（平行体系）

```
SR-NFR（非功能相关方需求）
  ↓ [N:1映射，无中间层]
SysReq-NFR 5级（非功能需求承接层）
  ↓ [权衡决策]
PA非功能权衡
```

### 关键约束

| 约束 | 说明 |
|------|------|
| **1:1约束** | 每条需求只能映射到唯一的承接点 |
| **N:1关系** | 承接点可以承接多条需求 |
| **末级定义** | BA 5级、SysReq-NFR 5级是末级，不再分解 |
| **平行体系** | 非功能需求独立于功能需求，不经过业务架构 |

## 🔄 工作流程

### 新增项目的6步工作流

1. **Task 1**：收集原始需求 → 规范化 → 生成SR和SR-NFR
2. **Task 2**：SR → BA映射 → 生成BA 0-5级
3. **Task 3**：BA → SysReq映射 → 生成SysReq 0-9级（功能）
4. **Task 4**：SR-NFR → SysReq-NFR映射 → 生成SysReq-NFR 0-5级
5. **Task 5**：SysReq → PA映射 + 非功能权衡 → 生成PA末级
6. **Task 6**：端到端追溯 + 符合性检查 → 生成报告

### 两阶段工作流程

**第一阶段：占位符创建（快速，1-2小时）**
- 在各任务中快速创建占位符
- 建立上下游映射关系
- 冻结占位符结构

**第二阶段：具体设计（深入，2-6天）**
- 填充占位符的具体内容
- 完成详细设计
- 验证设计完整性

## 🤝 并行工作建议

由于任务之间的相对独立性，可以进行以下并行工作：

```
Task 1（需求规范化）
  ├─ Task 2（SR → BA映射）
  │   └─ Task 3（BA → SysReq映射）
  │       └─ Task 5（SysReq → PA映射）
  │           └─ Task 6（符合性分析）
  │
  └─ Task 4（SR-NFR → SysReq-NFR映射）
      └─ Task 5（SysReq → PA映射）
          └─ Task 6（符合性分析）
```

**并行机会**：
- Task 2和Task 4可以并行进行（功能和非功能需求设计）
- Task 3和Task 4可以并行进行（功能和非功能需求架构设计）
- Task 5可以在Task 3和Task 4完成后进行

## ✅ 逻辑验证

### 问题1：非功能需求处理流程

**问题**：非功能需求是否需要中间层？

**答案**：✅ 不需要。非功能需求是平行体系，直接从SR-NFR映射到SysReq-NFR。

**验证**：
- 非功能需求与具体的业务组件无关
- 非功能需求在系统级别进行定义和权衡
- 非功能需求在产品架构层面进行权衡决策

### 问题2：产品架构末级定义

**问题**：产品架构末级节点应该是什么？

**答案**：✅ 前后端组件。

**验证**：
- 前端组件：页面、功能模块、UI组件、前端服务
- 后端组件：API端点、业务服务、数据访问层、第三方集成
- 这是可以独立开发、测试、部署的最小单位

### 问题3：占位符处理

**问题**：占位符应该如何处理？

**答案**：✅ 在各任务阶段直接完成。

**验证**：
- 不需要单独的占位符创建阶段
- 在Task 2中为BA 5级节点预留下游映射关系
- 在Task 3中为SysReq 9级节点预留PA映射关系
- 在Task 4中为SysReq-NFR 5级节点预留PA权衡关系
- 支持两阶段工作流程（占位符创建 + 具体设计）

### 问题4：任务5和6的合并

**问题**：任务5和6应该如何处理？

**答案**：✅ 合并为一个任务。

**验证**：
- 功能需求和非功能需求都映射到产品架构
- 非功能权衡是产品架构设计的重要部分
- 在同一个任务中处理可以确保功能和非功能的协调
