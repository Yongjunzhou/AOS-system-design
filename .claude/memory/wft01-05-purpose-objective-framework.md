## 全链路主要目的与目标

2026-06-26 建立，7 个 Skill 文件 §一 首段统一格式：**取信于谁 + 做什么 + 目标红线**。

| Skill | 取信于 | 目标 |
|-------|--------|------|
| **wft01-eng** | 相关方——「你听懂了我的需求」 | 名称级识别，不脑补。AI 推断标注 [推断]，架构决策提交人类 |
| **wft01-biz** | 相关方——「你听懂了我的需求」 | 名称级识别，不脑补 |
| **wft02-eng** | 业务方——「方案对」 | 模式级呈现，不伪配置。刻画配置类别和关键约束，不逐字段罗列 |
| **wft02-biz** | 业务方——「方案对」 | 模式级呈现，不伪配置 |
| **wft03** | 业务开发人员——「功能覆盖业务」 | 人机划界，独立于实现 |
| **wft04** | 架构师——「指标合理，分层正确」 | 量化可验证，分层不越位 |
| **wft05** | 开发方——「能开工」 | 组件边界完整，能直接认领 |

## 信任接力链

```
相关方 → wft01(听懂) → wft02(方案对) → wft03(覆盖业务) → wft04(指标合理) → wft05(能开工) → 开发方
```

## 设计推导过程

1. 从"核心原则是什么"出发 → 提取5条操作原则（太方法论内部视角）
2. 从"相关方需求的目的是什么"往回看 → 每一环解决一个信任问题
3. 从"做什么、到什么程度、有什么价值"提炼 → 目的+目标一段话
4. 标题从"设计原则"→"主要目的与目标"→"本Skill目的与目标"
