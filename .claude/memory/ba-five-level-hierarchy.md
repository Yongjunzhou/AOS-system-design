## EOS 产品定位

EOS（A3）是**软件产品形态的流水线类产品**（非非流水线类软件），以生产输出产品 B 为存在目的。因此 EOS 的 BA 使用**输出产品架构锚定法**（通用规范 §5.2.2），推导锚点为输出产品 B 的 PA 节点类型。

**Why:** 之前混淆了"产品类型"（流水线类 vs 非流水线类）和"产品形态"（实物/软件/AI Skill/服务）两个正交维度。EOS 虽然是软件形态，但它是流水线类——这个判断决定了 BA 方法的选择。

## 23 号资产的角色

`23-eos-output-architecture.md` 是 EOS 的输出产品架构，记录 EOS 产出什么（产品构成）。它是 BA 输出产品架构锚定法的**直接引用来源**：

- 产品类型（父）= BA 2 级项目类型
- 架构节点类型（子）= BA 3 级架构节点类型端到端业务
- 关联文档（孙）= BA IPO 的 I/O 参考来源（关联文档的 `产出：{IPO}` / `消费：{IPO}` 隐含了 IPO 之间的流转关系）

## BA 五级层级

| 层级 | 名称 | 来源 |
|------|------|------|
| 1级 | 业务域 | 业务职能领域划分（采购/生产/售后/研发/运营等） |
| 2级 | 项目类型 = 产品类型 | 从 23-*.md 产品类型读取 |
| 3级 | 架构节点类型端到端业务 | 从 23-*.md 架构节点类型读取 |
| 4级 | WBS 包 | wft02 分解产生（可选，3级下 IPO 过多时加一层文件夹） |
| **5级** | **业务组件或引擎组件（BA IPO）** | wft02 定义，I/O 关联到 23-*.md 关联文档 |

## 23-*.md 与 04-*.md 的职责边界

- **23-*.md（输出产品架构）**：产品类型 → 架构节点类型 → 关联文档。不含 WBS。关注"产出什么"。
- **04-*.md（业务架构）**：业务域 → 项目类型 → 端到端业务 → WBS → IPO。含 WBS。关注"怎么产出"。

WBS 不放 23-*.md 的理由：
1. WBS 是过程分解（怎么做），不是产品构成（产出什么），属性不同
2. WBS 会随项目规模/团队分工/工期变化，23 作为稳定产品架构参考不宜放动态的过程组织
3. 关联文档的 `产出：{IPO} / 消费：{IPO}` 已隐含了 IPO 间的流转关系，可间接推导 WBS 分段

## 对 wft02 的含义

wft02 的 BA 分解应明确体现五级层级，沿 23 号资产树定位路径：

```
STR 功能说明步骤
  → 1级 业务域
    → 2级 项目类型（23 产品类型）
      → 3级 架构节点类型端到端业务（23 架构节点类型）
        → 4级 WBS 包（本步分解产出，可选）
          → 5级 BA IPO（关联 23 关联文档）
```

当前 wft02（v2.4）按扁平 IPO 清单处理，未体现该层级结构，需修改。
