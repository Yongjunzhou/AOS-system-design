2026-06-01 完成对 `01-general-system-design-specification.md` 的内部风格统一，并将规则写入 `03-specification-template.md` v2.0。

**已实施的具体修改（01-通用设计规范）：**
1. **P0**：§十一 中 8 处 H5 编号错误 `10.x.x` → `11.x.x`
2. **P1**：§十 中 5 处无编号 H5 补充编号（`10.1.1`~`10.3.1`），§十一 1 处（`11.1.1`）
3. **P2**：附录C/D 标题冒号 → 顿号统一
4. **P3**：§1.3 唯一定位行内 `**注**` 改为 `> **注**` 块引用格式
5. **P4**：§八 6 处 `*对应§*` → `*适用步骤：§*` 与 §七 统一

**已确立的风格规则（写入 03-规范模板 v2.0）：**
- H5编号、H6不编号
- 附录标题 `附录[字母]、`
- 表格默认左对齐，无 `:-` `-:`
- `§` 后无空格，中文前保留空格
- 短注行内 `**注**`，长注块引用 `> **注**`
- 代码块无语言标签
- 流程图 ASCII 树，箭头方向统一
- 无序列表用 `-`
- 适用范围标注 `*适用步骤：§*`
- `---` 仅用于 H3 大节之间

**Why:** 消除通用设计规范内部的风格不一致（特别是 §十一 的编号错误直接影响文档可信度）。

**How to apply:** 新写的规范文档应遵循 03-规范模板 v2.0 的风格规则；发现现有文档有同样的编号/格式/语气问题时，按模板 §八（表达风格）的规则修正。

**表达风格内核（已写入模板 §八）：**
- 文档定位：工程规范，不是教科书/培训材料。读者查而非学。
- 人称：零人称祈使句（规则/操作）+ 第三人称陈述句（定义）。严禁"你""我们"。
- 规则硬度：应/须（强制）、不得/不应（禁止），避免"可以/建议"。
- 解释：允许因果解释，禁止比喻/类比/辅导口吻。
- 元评论：文档不谈自身——禁止"本文的核心内容""重点"等旁白。
- 对比示例：仅用于消除歧义的关键判断点，不滥用。
- 可读性（§8.8）：单句单意、慎用"进行"、避免重复表述、去掉教科书铺垫、缩写首次出现标注全称。
