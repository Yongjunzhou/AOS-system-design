# 三字段正交状态模型（v2）

## 模型

| 字段 | 含义 | 维护者 | 值 |
|------|------|--------|-----|
| **生命周期状态** | 流水线进度（事实） | AI 维护 | 已登记→澄清中→已澄清→规范化中→已规范化→分配中→已基线化 + 已废弃 |
| **AI 建议** | AI 对当前状态的判断 | AI 输出 | **需澄清** / **可确认** / 建议终止 |
| **人类决策** | 人类的终结点决定 | AI（从文档同步） | **已确认** / 已终止 |

## 核心原则

1. **"AI 建议，人类确认"**——AI 输出报告附带建议，人类查看文档后决定
2. **文档即状态**——文本化工作副本（`*-clarified.md`）驱动一切，registry-log 是同步视图
3. **中间状态不写字段**——迭代中的人类沟通、记录、暂停，都不需要在三字段上做任何操作
4. **AI 维护字段，人类操作文档**——生命周期状态和人类决策由 AI 根据文档内容自动维护

## 协作模式

```
AI 通读文档
  → 输出模糊标注报告（含 AI建议：需澄清/可确认/建议终止）
  ↓
若可确认/建议终止 → AI 在文档末尾追加操作选项
若需澄清        → 不加选项，等人类追加澄清记录
  ↓
人类打开文档 → 删掉不需要的选项行 → 保存
或：人类追加澄清记录到文档 → 保存
  ↓
AI 下次启动扫描文档
  → 读到保留的选项 → 校验 → 执行
  → 追加正式确认块/废弃说明块
  → 同步到 registry-log（生命周期状态 + AI建议 + 人类决策）
```

## 操作选项机制

AI 在文档末尾追加，人类删行即确认：

**可确认时**：
```
---

【操作选择】请删除不适用的选项，保留你的选择（保留一整行）：

1️⃣ [确认通过] → 所有模糊项已解决，确认进入已澄清状态
2️⃣ [继续迭代] → 仍有未尽项，需要继续澄清
3️⃣ [已终止]   → 放弃该材料
```

**建议终止时**：
```
---

【操作选择】请删除不适用的选项，保留你的选择（保留一整行）：

1️⃣ [确认终止] → 该材料确实应废弃
2️⃣ [继续迭代] → 不应废弃，继续澄清
```

## 与旧模型的差异

| 维度 | 旧模型 | 简化v2 |
|------|--------|--------|
| AI建议值 | 建议待处理 / 建议暂搁置 / 建议终止 | **需澄清** / **可确认** / 建议终止 |
| 人类决策值 | 待推进 / 暂搁置 / 已终止 | **已确认** / 已终止 |
| 人类决策维护者 | 人类手动写字段 | AI 从文档选项同步 |
| 迭代中状态操作 | 人类设暂搁置/待推进 | 不需要——追加记录到文档即可 |
| 断点续接 | 三字段联合+待处理提醒 | AI 自动检测文档变化 |
