# 三字段正交状态模型

## 模型

| 维度 | 含义 | 维护者 | 值 |
|------|------|--------|----|
| **生命周期状态** | 流水线进度（事实） | AI 维护，终端需人类确认 | 已登记→澄清中→已澄清→规范化中→已规范化→分配中→已基线化 + 已废弃 |
| **AI 建议** | AI 当前判断（意见） | AI 填写 | 建议待处理 / 建议暂搁置 / 建议终止 |
| **待处理状态** | 人类决策（决定） | **人类填写** | 待处理 / 暂搁置 / 已终止 |

**核心原则**："AI 建议，人类确认"——AI 在每个决策门禁处输出建议，人类做最终决定。

## 协作模式

```
AI 完成阶段工作
  → 更新生命周期状态
  → 输出 AI 建议
      ↓
人类看到 AI 建议 + 支撑材料
  → 设待处理状态（待处理→推进，暂搁置→暂停，已终止→废弃）
```

## 适用范围

- **通用规范 §7.3 阶段一(e)** 承载原则层定义
- **Pipeline skill 实现**（t00a/t00b 等）承载具体执行流程和决策门交互

## 相关决策

- 旧字段（澄清状态 + AI 处理标记 + 人类决策）信息重叠，统一为三个正交维度
- 暂搁置是隐式状态——材料自然停在当前生命周期状态，督促机制定期扫描
- 生命周期状态路径因条目类型而异（原始需求走完整路径，缺陷/架构变更走简化路径）
- t00 拆分为 t00a（原始需求澄清）和 t00b（原始需求规范化），各含两个决策门
