pre-t04-norm（原始需求规范化）采用全自动执行模式：
- AI 一次性完成条目拆分 → 规范化 → 冲突检测 → 优先级分类 → 输出 → 基线管理 → 审查准备 → 更新 registry-log → 输出线下行动指南
- **无决策门B**：移除了"人类选择放行范围"的中间交互，AI 自动全量处理
- **无对话内确认**：§七 从对话确认改为线下审查指引
- **registry-log 更新时机**：AI 阶段五完成后直接更新为"已规范化"，不再等待人类确认（原 pre-t03-norm 继承）
- **多次执行支持**：每次读取登记表中"生命周期=已澄清 + 人类决策=已确认"的未处理记录，新旧 OR 条目共存于 v2 区

**Why：** 预处理链的定位是"将输入加工为标准规格"，不应插入人类交互轮次。人类在 Skill 执行期间没有需要参与的内容决策（冲突检测中 AI 不做裁决，标记待确认供线下处理即可）。

**How to apply：** 同类型的预处理/规范化 Skill 应遵循此模式：全自动执行 + 产出物中的线下行动指南作为人机交接点。对照 [[skill-action-output-principle]]
