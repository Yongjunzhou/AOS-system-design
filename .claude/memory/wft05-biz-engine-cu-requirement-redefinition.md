## 修改摘要

wft05-biz v3.0 本质重定义：PA 组件在 biz 线上 = 引擎配置单元需求，不是软件开发组件。

### 核心定位

| | biz 线（wft05-biz） | eng 线（wft05-eng） |
|--|-------------------|-------------------|
| 做什么 | 定义要配置哪些引擎配置单元 | 开发这些引擎和配置单元 |
| 产出 | 引擎配置单元需求方案 | 引擎前后端组件实现 |
| 对引擎的态度 | 消费者：配置需求方 | 生产者：开发供应方 |

### 三步边界

- wft02-biz：业务架构对不对
- wft03-biz：系统功能是否完整承接业务
- wft05-biz：业务功能能否被引擎配置体系承接——每条 SR-F 由哪个引擎的哪个配置单元承接

### SR-F 类型 → 目标引擎配置单元映射

| SR-F 类型 | 目标引擎 |
|-----------|---------|
| 菜单/入口/可见性 | 菜单引擎 CU |
| L4 场景业务 | 场景引擎 CU |
| L5 表单部件 | 表单引擎 CU |
| L5 流程部件 | 流程引擎 CU |
| L5 工单部件 | 工单引擎 CU |
| L6 功能元件/实体对象 | 实体引擎 CU |
| 指标/看板/聚合 | 指标引擎 CU |
| 资产目录/数据目录/血缘/质量 | 治理类引擎/通用 CU |

### 关键前提与规则

1. wft05-biz 默认目标引擎及其配置单元已经存在，或者已由 eng 路径另行开发
2. 不设计引擎本体，不拆前后端实现组件
3. 引擎能力缺口不进入 wft03-biz / wft05-biz，走 raw material → ort00~03 → wft01-eng
4. 25 引擎模型升格为核心输入（全景加载）——确定 SR-F 可映射到哪些引擎 CU

### 一句话

biz 链到 05 不是"开发组件"，而是"定义要配置哪些引擎配置单元"；eng 链才是"开发这些引擎和配置单元"。
