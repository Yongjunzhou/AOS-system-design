# pl4eos 系统设计规范的使命

**确认日期**：2026-06-03

## 使命定义

`01-pl4eos-spec-sysdev.md`（即 A2 规范）是 **pl4eos 产品的设计描述**，服务于 **构建 pl4eos 设计线的开发人员**，让他们按照本规范完成后续工作。

文档内容分为两段功能：

| 功能 | 对应章节 | 为设计线开发者提供 |
|------|---------|-----------------|
| **架构产出指引** | §一 + §二 + §四 | 产品语境、设计框架、各层设计指引——告诉他们"要产出什么架构" |
| **操作流程指引** | §三 + §七~§十四 | 通用方法论精要、四种模式操作框架、质量门禁、检查清单——告诉他们"按什么流程去产出" |

两份内容面向同一群人，服务于同一目标。

## A1 与 A2 的对称关系

A1（元流水线）的对应规范使命与此对称——它是元流水线的设计描述，服务于元流水线的设计线开发人员。

但由于 A1 和 A2 产品类型相同（流水线类）、产品形态高度相近（AI Skill），且最终目标是开发 EOS，若 A1 不精简则会与 A2 高度重复。因此 A1 规范需从 1110 行精简为简短的方法论说明 + 工程导航。

精简不是因为"A1 不重要"，而是因为"A1 的设计描述与 A2 高度重复"。
