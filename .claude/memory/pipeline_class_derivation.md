流水线类 E2E项目业务的系统需求推导规则（§2.2.5）：

**核心特征**：推导锚点偏移。一般产品实现类以该业务自身产出的"产品"的系统架构为锚点；流水线类以**目标产品**的产品架构为锚点——目标产品有什么类型的构件，就需要有什么操作步骤来生产它，这些步骤就是流水线的 BA IPO。

**目标产品范围**：可以是任何形态——实物、软件、方法/工具集等。推导逻辑不变，仅①步的目标产品 PA 节点类型因形态而异。

**适用场景**：
- 产品开发流水线（目标产品=实物，如机载设备）
- 平台开发流水线（目标产品=软件，如引擎）
- 方法/工具集开发流水线（目标产品=方法/工具集）

**推导路径**：目标产品 PA 节点类型 → OR → SR → BA IPO(1:1) → BA IPO 去重 → SysReq → PA（流水线自身产品架构）

与 §2.2.4 的关系：§2.2.4 是标准分解（产品实现类），§2.2.5 是锚点偏移分解（流水线类）。
