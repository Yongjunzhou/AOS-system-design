# 需求符合性审查：三层二维验证框架

**三层**（递进审查）：
- **第1层**：需求文档分解质量审查——子需求对父需求的语义覆盖完整性、不超出性、逻辑一致性、负需求继承、指标分解符合性
- **第2层**：承接关系验证——需求文档↔目标文档之间的承接存在性、正确性、完整性
- **第3层**：已承接需求符合性审查——目标文档节点是否真正满足需求

**二维**（贯穿三层）：
- **语义符合性**（需求说明符合性）——方案在内容/语义上是否满足需求
- **性能指标符合性**（需求性能符合性）——方案的指标是否继承了需求的指标要求

**Why:** 这是需求符合性审查（requirements-conformance-review SKILL）的核心架构。第3层依赖第2层的产出（必须先知道谁承接了谁，才能审查满足度）。两个维度独立提取、独立审查、在报告中分别呈现。
