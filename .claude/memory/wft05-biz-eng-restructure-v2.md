## 修改摘要

wft05-biz 分流锚点从"IPO类型"重构为"SR-F来源对象"，定位收敛为"EOS业务运行与治理侧PA"。wft05-eng 新增25对齐延续和依赖链承接。

### wft05-biz v1.0 → v2.0 核心变化

| 旧（v1.0） | 新（v2.0） | 核心变化 |
|---|---|---|
| 1+7 配置方案组件 | **四源分流PA组件** | A2场景→交互类 / Bn资产→资产管理类 / 数据治理→治理类 / 双视角追溯→追溯校验类 |
| 配对引擎 Input 配置模板 | **引擎能力依赖仅标注** | 不生成引擎PA组件，标注FR-ENG/OR/STR-E/BA-A1依赖链路 |
| 无数据治理分流 | **数据治理独立分流** | 治理角色（业务架构管理员/数据治理员）的SR-F独立分流 |
| 定位模糊 | **业务运行与治理侧PA** | 明确不是引擎本体架构，引擎由eng路径处理 |

### wft05-eng v1.0 → v1.1 核心变化

| 新增项 | 说明 |
|--------|------|
| 25 对齐延续 | Q1引用/Q2扩展/Q3推断，延续wft03-eng已建立的对齐关系 |
| 依赖链承接 | PA组件间引用关系与wft03-eng 3b的CU间依赖链一致 |
| PA层收敛细化 | 未收敛项标注[PA层未收敛]+原因，不阻断PA方案推进 |
| 分类线索+对齐线索 | 按需加载新增两类线索 |
| 定位说明 | 引擎本体侧PA，与wft05-biz业务运行与治理侧PA边界明确 |

### 四源分流判定规则

1. 治理角色优先（无论流1/2/3）→ 数据治理分流
2. 流3（A2-Bn映射）→ 双视角追溯分流
3. 流1（A2角色场景）→ A2场景分流
4. 流2（Bn产品视角）→ Bn资产分流
