# 三字段正交模型简化与文档选项机制

## 背景

t00a（原始需求澄清）的设计讨论中，发现原三字段模型存在值语义模糊、人类操作负担重的问题：

| 原值 | 问题 |
|------|------|
| AI建议=**建议待处理** | 什么都说不出来——报告出了、有模糊、已满足三类场景都用同一个值 |
| AI建议=**建议暂搁置** | 读起来像"放着别动"，实际意思是"有模糊项需要人类去沟通" |
| 人类决策=**待推进** | 读起来像"还没做完"，实际意思是"确认通过/可以推进" |
| 人类决策=**暂搁置** | 读起来像"丢一边"，实际意思是"暂停等外部输入" |

## 简化后的三字段

| 字段 | 维护者 | 值 | 语义 |
|------|--------|-----|------|
| **生命周期状态** | AI | 已登记→澄清中→已澄清/已废弃 | 流程位置，AI 首次输出报告时自动 已登记→澄清中 |
| **AI 建议** | AI | **需澄清** / **可确认** / 建议终止 | 人类一看就知道：要我去问 / 来看一眼确认 / 废弃 |
| **人类决策** | AI（同步） | **已确认** / 已终止 | 人类不在迭代中操作——终结点 AI 自动同步 |

## 文档选项交互机制

替代原"决策门"概念——AI 在文本化原始需求文档末尾直接追加操作选项，人类打开文档删行即完成确认：

**场景1：AI建议=可确认**
```
---

【操作选择】请删除不适用的选项，保留你的选择（保留一整行）：

1️⃣ [确认通过] → 所有模糊项已解决，确认进入已澄清状态
2️⃣ [继续迭代] → 仍有未尽项，需要继续澄清
3️⃣ [已终止]   → 放弃该材料
```

**场景2：AI建议=建议终止**
```
---

【操作选择】请删除不适用的选项，保留你的选择（保留一整行）：

1️⃣ [确认终止] → 该材料确实应废弃
2️⃣ [继续迭代] → 不应废弃，继续澄清
```

**场景3：AI建议=需澄清**
不加选项——人类直接追加澄清记录到文档即可，AI 后续通读自动发现。

AI 下次启动时扫描文档，读到保留的选项 → 校验 → 执行（追加正式确认块/废弃说明块 → 同步更新 registry-log 中的人类决策值）。

## 核心原则

1. **文档即状态**——文本化原始需求文档（`*-clarified.md`）驱动一切，registry-log 是同步视图而非操作入口
2. **中间状态不写字段**——迭代中的人类沟通、记录、暂停，都不需要在三字段上做任何操作
3. **AI 维护字段，人类操作文档**——生命周期状态和人类决策由 AI 根据文档内容自动维护

## 适用范围

此模型不仅适用于 t00a，也适用于所有需要人机协作迭代的步骤。后续 t00b/t01 等 skill 设计时应采用相同的三字段语义：AI 建议用动作指示（需澄清/可确认），人类决策保留为 AI 同步的终结点标记，中间迭代过程由文档内容驱动。

## 推演过程参考

此结论来自 t00a 设计讨论中的推演式对话——从"两个决策门合并"开始，经"三字段值分析→人类交互简化→文档选项机制推导"逐步收敛。此方法论（推演→定稿→重构）可作为后续 skill 设计的参考模式。
