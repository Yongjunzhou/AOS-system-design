ORT 预处理链（ort00~ort03）Skill 设计审查中确立的关键模式：

**AI建议 前视原则：** AI建议应推荐下一步操作（"可以切分""可以澄清"），而非报告已完成的操作（不应写"文本化完成"）。每个 Skill 的 AI建议指引人类决策进入下一个 Skill。

**人类决策 步骤特异：** 人类决策取值与具体步骤绑定（ort01 用"同意切分"，ort02 用"同意澄清"），而非通用"已确认"。避免一个字段在不同上下文中语义冲突。

**1.1/1.2 表职责分离：**
- 1.1 OR材料进展：一个原始材料一行，追踪全生命周期（已登记→已切分→已澄清→已规范化），含 AI建议（跨步骤）和 人类决策
- 1.2 OR切分进展：一个原始材料一行，追踪切分过程状态（待分析→待确认切分方案→切分已执行/不切分已确认），含 AI建议（切分阶段）和 操作指引
- 切分子文件（-sub-*.md）通过 1.2 追踪，不在 1.1 中为子文件建行

**人类决策语义等价映射：** AI 需将人类自然语言映射到标准字段值（如"同意""可以""好"→"同意切分"），在 Skill 文档中维护映射表。

**关注行三类结构：** Step 5 人机交互按三类列建议：选材入当前Skill / 选材入下游Skill / 方案反馈。每类标注数据源位置、筛选条件、人类操作。
