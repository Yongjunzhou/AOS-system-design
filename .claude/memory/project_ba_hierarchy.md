## 业务架构层级结构（0-6级）

| 层级 | 名称 | 说明 |
|------|------|------|
| 0级 | 架构根节点 | 整个系统的顶层根节点 |
| 1级 | 业务域 | 按业务领域划分 |
| 2级 | 端到端项目 | 跨越多个业务域的项目 |
| 3级 | 项目业务对象架构节点 | 端到端业务的架构节点 |
| 4级 | 工作包（WBS） | 对象架构节点下的工作分解结构 |
| **5级** | **组件** | **★ 承接相关方需求的关键层级**<br>- 业务组件（业务逻辑承载）<br>- 引擎组件（技术能力承载） |
| 6级 | 操作序列 | 组件进一步分解的具体操作<br>- 流程定义 → 流程节点<br>- 计划定义 → 计划执行步骤<br>- 流程引擎 → 流程配置操作步骤 |

## 关键设计点

- **5级组件是需求承接的关键层级**：相关方需求（SR）最终映射到 5 级的业务组件或引擎组件
- **N:1 映射约束**：多条下层需求可映射到唯一的上层承接点，但每条需求只能分配到唯一的承接点
- **6级操作序列**：是组件的具体实现细节，不同类型组件有不同的分解方式（流程、计划、引擎配置等）

## 应用场景

在 SR → BA 映射（准则 2）和 BA → SysReq 映射（准则 3）中，需要确保：
1. 需求映射到正确的层级（通常是 5 级组件）
2. 满足 N:1 多对一映射的约束
3. 上层节点的内容完整覆盖分配到它的所有下层需求
