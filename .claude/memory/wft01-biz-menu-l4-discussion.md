### v1.24 Step 3 最终结构

```
§3.1  STR-F 架构说明
  ├── 场景（@engine-场景引擎 + 场景级 QTC）
  │     └── 菜单（@engine-菜单引擎 + 菜单级 QTC）
  │           └── 输出文档（具体业务引擎 + 文档级 QTC + 生命周期/支持信息/交互信息）
  └── STR-F 节点完整要素

§3.2  要素提取规则（Phase B/C 共用）
  ├── 菜单识别规则（4条）
  ├── 输出文档标注
  └── 要素标注字段（菜单级/文档级）

Phase A: OR 轻量提取与路由  → Q1/Q2/Q3 → 分组
Phase B: 修订已有 STR-F      → 提取(§3.2) → 边界检查 → 四关系比对 → 变更声明
Phase C: 新建 STR-F          → 提取(§3.2) → 三步判断法 → 组装 → 覆盖自查
```

### 三层引擎 + 三级 QTC 映射

| 层级 | 引擎 | QTC |
|------|------|-----|
| 场景（L1） | @engine-场景引擎 | 场景级 QTC = 各菜单 QTC 汇总 + 场景特有 |
| 菜单（L2） | @engine-菜单引擎 | 菜单级 QTC = 菜单下文档 QTC 汇总 + 菜单特有 |
| 输出文档（L3） | 表单/流程/台账...引擎 | 文档级 QTC（质量/及时性/完整性） |

### 核心设计决策

- 交互信息不独立成引擎——开放类型清单：通知/待办/回执/状态/告警/异步消息(MQ)/同步调用(API)
- 不把配置项升格为引擎
- 引擎直接生成业务，无"业务定义"中间层
- 功能菜单分组 = 菜单集合体，不是独立结构层
- 场景业务 → @engine-场景引擎，功能业务 → @engine-菜单引擎，均为确定映射
