# 语义段落聚类的实际判定规则

在规则符合性审查的 Step 0a-3 语义段落组装中，三条标准按以下优先级运作：

1. **同一通读路径**（第一优先）——若干规则在审查时是否在同一次文档遍历中、读同一个位置即可检查？是则倾向合并
2. **同一判断焦点**（第二优先）——若干规则是否在问同一个审查问题？焦点不同但通读路径相同，可合并但需独立编号
3. **同一原文段落**（最低优先）——仅做参考，不应作为聚类依据

新范式（同一判断焦点）与旧习惯（通读路径）之间的拉锯是实际执行中的主要矛盾。独立编号（P0X-RXX）是缓解矛盾的桥梁——即便判断焦点不同，只要通读路径相同，可以放在同一段落但赋予独立编号，审查时逐条判断。

**与[[rule_verifiability_driven_extraction]]的关系**：该记忆定义了"为什么"（范式转换），本记忆定义了"具体怎么判断"（三个标准的优先级和取舍）。

**Why:** 通过实际运行新版SKILL的语义提取（2026-05-23）发现，用户指出P01中混入了不同判断焦点的规则（R01/R02 焦点=类型定义，R03 焦点=配对存在性，R04 焦点=双重属性），但通读路径完全相同（都在§1.3/§1.5.1）。这暴露了三个聚类标准之间的优先级冲突。

**How to apply:** 执行语义段落组装时，问"这些规则在审查时是在同一次通读中检查的吗？"如果答案是"是"，可以合并在同一段落；然后用独立编号确保每条规则被独立判断，段落级核心约束仅作聚焦之用，不作判据。
