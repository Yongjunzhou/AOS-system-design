## E2E 项目定义（2026-06-13 确认）

企业有三类相关方，每类驱动对应的 E2E 项目类型和业务域：

### 1. 上级或监管方
对企业提出经营目标和需要符合的监管要求。企业负责完成经营目标，将监管要求转换成业务变革项目或年度运营项目，取得相关资质或通过年度审核。

产生 E2E 项目：经营规划、质量管理、合规管理等方向的 E2E 项目

### 2. 客户
对企业提出产品研发需求、产品采购需求或产品售后服务需求。企业通过研发业务输出符合客户需求的产品或解决方案，或通过产品生产和交付向用户提供满足其合同的产品，或通过售后服务业务为用户提供产品售后服务。

产生 E2E 项目：市场销售、产品研发、产品生产、售后服务、采购管理等方向的 E2E 项目

### 3. 内部员工（项目间需求）
各项目成员在开展业务运行时，需要向其他项目提出各类需求，如人力、资金、设备、IT工具、空间等。相应的项目通过运行提供满足其他项目需求的人财物等。

产生 E2E 项目：HR 管理、财务管理、设备管理、行政服务、部门管理、流程与 IT 等方向的 E2E 项目

## 对 23.md 结构的影响

表格以 业务域 为顶层行，E2E 项目为 2 级行，构件类型为 3 级行。一个业务域下通常有 5 个以上 E2E 项目，规模大的业务域可拆分子业务域组织。

**Why:** 此前方法论中 E2E 项目的定义不完整，容易把"构件类型"误认为"E2E 项目"（如将采购订单管理作为构件类型挂到采购产品下，但采购域实际有 8+ 个 E2E 项目，采购订单管理只是其中之一）。该定义写入项目记忆，作为判断节点层级归属的依据。
