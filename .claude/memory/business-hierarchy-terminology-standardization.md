# EOS 业务层级术语标准化（2026-06-27）

## 变更背景

将 EOS 平台的业务层级从隐式/混淆状态标准化为明确的 L0~L6 七级，引擎层次从旧四级（L1/L2/L3/L4 组件/部件/单元/元件）重构为三级生成 + 一横切。

## 新术语体系

### 业务层级 L0~L6

| 层级 | 名称 | EOS 平台体现 | 生成引擎 |
|------|------|-------------|---------|
| L0 | EOS 平台 | 平台自身 | — |
| L1 | 业务系统 | 一级菜单 | 菜单引擎 |
| L2 | 项目业务（业务模块） | 二级菜单 | 菜单引擎 |
| L3 | 构件业务 | 三级菜单 | 菜单引擎 |
| L4 | 场景业务 | 四级菜单（末级），打开后多窗口 | 场景引擎（组件级） |
| L5 | 场景部件 | 一个窗口 | 表单/流程/指标/工单引擎（部件级） |
| L6 | 功能元件 | 原子构件（实体对象、事件、定时任务） | 实体引擎（元件级） |

### 引擎三级 + 横切

| 引擎层级 | 引擎 | 生成 |
|---------|------|------|
| 组件级 | 场景引擎 | L4 场景业务 |
| 部件级 | 表单引擎、流程引擎、指标引擎、工单引擎 | L5 场景部件 |
| 元件级 | 实体引擎 | L6 功能元件 |
| 横切 | 菜单引擎 | L1~L4 菜单树（CRUD + 权限 + 末级映射到 L4） |

### 废弃术语

- 功能菜单 → 取消概念
- 功能单元 → 场景部件
- 场景定义 → 场景业务
- 功能业务 → 场景部件
- 对象及其功能 → 功能元件
- L3 单元级 → 并入部件级

### 菜单引擎新定义

菜单引擎不再生成业务，职责为：① L1~L4 四级菜单树节点 CRUD；② 权限管理；③ 末级节点映射到 L4 场景业务。若未映射，场景业务在平台上为不可达的哑资产。
