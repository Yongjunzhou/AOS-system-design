# 新增"产品架构设计"独立章节计划

## 一、背景

PA（产品架构）是五层设计链的终点（纯方案层），承接 SysReq-F 和 SysReq-NFR 两路输入，下游直接对接组件开发阶段。但目前 PA 相关内容散落在 §一（产品类型）、§四（PA 节点识别）、§六（PA 设计步骤）中，缺少统一的完整论述。BA 已有独立章节 §五，PA 作为同等重要的层位，也应独立成章。

## 二、新章位置

**插入 §五（业务架构设计）之后、§六（系统设计过程）之前。**

§五 结束于 L1265，§六 开始于 L1266，需要在两者之间插入新章。

编号方案：作为 §六，后续 §六~§十三 依次迁为 §七~§十四。

## 三、新章结构草案

```
### 第X章、产品架构设计

#### X.1 产品架构在五层设计链中的角色
→ PA 是设计链终点，是 SysReq 的方案方（纯方案层）
→ 承接两条输入：SysReq-F（功能）+ SysReq-NFR（非功能）
→ 输出：PA 架构定义子文档（作为组件开发的输入）

#### X.2 产品类型与 PA 架构形态      ← 来源：§1.6（完整三节）
→ 三种基本产品类型：知识/方法类、软件类、实物类
→ 混合型产品的处理（管道线类、嵌入式等）
→ 各产品类型的 PA 节点分类体系

#### X.3 PA 架构末级节点的判定        ← 来源：§4.7 + §4.7.2
→ 三独立原则（可独立分配/可独立替换/可独立验证）
→ 按产品类型的节点分类模板
  - 软件类：技术层/业务域
  - 实物类：物理组件/子系统
  - 知识类：交付物形态

#### X.4 PA 架构定义子文档             ← 来源：§2.2.1 PA 部分
→ PA 节点四要素（节点定义/上层映射/符合性分析/下游指导）
→ 映射关系：承接 SysReq 详细定义末级（N:1）

#### X.5 PA 节点识别方法               ← 来源：§4.7.3
→ 资产优先法：复用 → 改进 → 外购 → 新制
→ 新制时三独立判定（分配/替换/验证）
→ PA 与 SysReq 边界划分

#### X.6 从 SysReq 到 PA 的设计过程     ← 来源：§6.7
→ 第5步的设计操作要点
→ 结对设计的应用（骨架→填充两阶段）
```
