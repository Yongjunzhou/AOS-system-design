## AI读取文档说明

本文件是 STR 架构类树形资产文档。AI 读取本文件时应按需读取，不应默认读取全部正文。

默认读取顺序：

1. 读取文档元信息和本说明，确认文档用途、状态和读取规则。
2. 读取 `0. STR树画像`，建立 STR-F、STR-NFR、角色分片和节点规模的整体感知。
3. 读取 `1. STR状态与处理队列`，确定本轮目标 STR 节点、活动锁、当前待办 ID 和下游选择记录。
4. 读取 `2. STR节点索引 / 分片索引`，定位目标 STR 正文块、候选同角色节点块和必要追溯关系。
5. 仅读取目标 STR 正文块、追溯关系指向的 OR 块，以及本轮需要写回的反馈区或建议区。

扩大读取范围仅在以下情况允许：

- STR 节点索引缺失或块 ID 无法定位。
- 追溯关系断裂，无法判断 OR 到 STR 的承接关系。
- 候选 STR 节点存在边界冲突、角色归属冲突或十链信息载体归属冲突。
- 需要执行全局一致性检查或批量迁移。
- 人类明确要求通读全文。

禁止事项：

- 不得在常规 `wft01a/wft01b/wft02a` 运行中默认全量读取正文节点块。
- 不得将旧版正文、废弃节点或历史归档作为当前 STR 资产参与判断。
- 不得只凭对话上下文推进状态；人类反馈必须先写入节点反馈区、状态队列或转交记录。

## 3. 规则区 / 口径区

### 3.1 STR-F 节点口径

STR-F 架构末级节点指代场景定义。每个 STR-F 节点对应一个角色的端到端职责闭环，与 `wft02` 中该角色的场景定义形成 1:1 映射。

### 3.2 十链信息载体口径

十类信息链是 `wft01a` 的分析框架，不是场景定义的内容结构。场景定义正文应输出扁平的信息载体清单和场景指标清单，每个载体标注归属链。

### 3.3 依赖资产

| 资产 | 用途 |
|------|------|
| `01-eos-original-requirements.md` | OR 来源与承接关系 |
| `21-eos-stakeholder-roles.md` | 已确认角色归属 |
| `22-eos-nfr-taxonomy.md` | NFR 分类与指标归类 |
| `23-eos-output-architecture.md` | 业务域、产品类型、构件类型候选 |
