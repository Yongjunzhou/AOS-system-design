## AI读取文档说明

本文件是引擎模型资产文档。AI 读取本文件时应先读取资产全貌、处理队列和模型索引，再按引擎模型 ID 读取目标正文块。

默认读取顺序：

1. 读取文档元信息和本说明，确认本文档承载引擎模型资产。
2. 读取 `0. 引擎模型资产全貌`，了解引擎类型、实现策略和分片方式。
3. 读取 `1. 引擎模型状态与处理队列`，确认待新增、待扩展、待替换或待废弃模型。
4. 读取 `2. 引擎模型索引 / 分片索引`，按模型 ID、引擎类型、实现策略或依赖关系定位目标块。
5. 仅读取目标模型正文块、必要依赖模型块和相关追溯关系。

扩大读取范围仅在以下情况允许：

- 多个引擎模型存在职责边界冲突或替代关系。
- 需要判断业务定义是否已有可复用引擎支持。
- 依赖链断裂或接口边界无法判断。
- 人类明确要求通读引擎模型资产。

禁止事项：

- 不得在常规 Skill 运行中默认读取全部引擎模型正文块。
- 不得将候选模型直接作为 PA 或开发强依赖。
- 不得在本文档复制完整技术实现方案；这里只记录架构级模型资产。

## 3. 规则区 / 口径区

1. 引擎模型描述平台能力边界，不替代产品架构组件或代码实现设计。
2. 引擎模型应能追溯到 `23` 构件类型、`24` 业务定义或 `04` BA A1 方案。
3. 同一引擎模型可被多个业务定义复用；复用关系记录在索引和追溯区。
4. 外购、自研、自研+外购是架构级实现策略，不代表采购或开发已经完成。
5. 对外接口只记录架构级能力和输入输出摘要，详细 API 规格应在后续产品本体或开发文档中定义。
