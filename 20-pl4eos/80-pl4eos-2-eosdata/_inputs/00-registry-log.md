## 生命周期状态总览

各条目类型的生命周期状态流转路径：

### 原始需求（完整路径）

```
已登记 ──→ 澄清中 ──→ 已澄清 ──→ 规范化中 ──→ 已规范化 ──→ 分配中 ──→ 已基线化
  │          │          │           │             │            │
  └────────────────← 已废弃 →────────────────────────────────────┘
```

| 状态 | 含义 | 转迁方式 |
|------|------|---------|
| 已登记 | 录入登记表，未开始处理 | AI 自动（初始状态） |
| 澄清中 | t00a 迭代澄清中 | AI 自动（首次标注后） |
| 已澄清 | t00a 完成，人类确认通过 | 决策门B 人类确认 |
| 规范化中 | t00b 规范化处理中 | AI 自动 |
| 已规范化 | t00b 完成，OR 条目就绪 | AI 自动 |
| 分配中 | t01 映射分配中 | AI 自动 |
| 已基线化 | 全流程完成 | 人类确认（终端） |
| 已废弃 | 被放弃 | 人类确认（终端） |

### 缺陷 / 架构变更（简化路径）

```
已登记 ──→ 处理中 ──→ 待确认 ──→ 已基线化
  │          │                    │
  └────→ 已废弃
```

| 状态 | 含义 | 转迁方式 |
|------|------|---------|
| 已登记 | 录入登记表 | AI 自动（初始状态） |
| 处理中 | AI 执行修复/处理 | AI 自动 |
| 待确认 | AI 完成，等待人类确认 | AI 自动 |
| 已基线化 | 处理完成，人类确认 | 人类确认（终端） |
| 已废弃 | 被放弃 | 人类确认（终端） |

### 三个正交维度的协作方式

```
AI 完成阶段工作
  → 更新生命周期状态（材料走到哪一步了）
  → 输出 AI 建议（"建议待处理"或"建议暂搁置——仍有模糊项未解决"）
      ↓
人类看到 AI 建议 + 支撑材料
  → 设待处理状态（"待处理"则推进生命周期，"暂搁置"则暂停，"已终止"则废弃）
```

## 操作流程

### 原始需求

1. **AI 登记**：人类将原始材料放入 `_inputs/10-source-materials/`，告知 AI 登记。AI 创建独立文件并追加到此表（生命周期状态="已登记"）
2. **t00a-clarify — 首次标注**：AI 执行文本化+首次模糊标注后，将生命周期状态更新为"澄清中"，AI 建议="建议待处理"，等待人类通过**决策门A**审查标注
3. **人类沟通与澄清**：人类审查标注后与相关方沟通，将澄清结果记录到文本化文档中
4. **t00a-clarify — 迭代重检**：AI 通读更新后的文档，检查是否达到已澄清标准
   - 仍有模糊 → 保持"澄清中"，输出新增标注报告，AI 建议="建议暂搁置"，返回步骤3
   - 三项标准全满足 → 输出"建议已澄清"，AI 建议="建议待处理"，等待人类通过**决策门B**确认
5. **人类确认**（决策门B）：人类审查 AI 的已澄清建议，确认通过后：
   - 设置待处理状态="待处理"
   - AI 将生命周期状态更新为"已澄清"
   - 在文本化文档末尾追加人类确认块
6. **t00b-norm**：AI 筛选"生命周期状态=已澄清"且"待处理状态=待处理"的记录执行原始需求规范化
7. **完成**：AI 将生命周期状态更新为"已规范化"→"分配中"→"已基线化"（按实际推进）

**废弃路径**：人类可在任意步骤要求废弃某条原始需求记录。AI 将澄清状态更新为"已废弃"，并在文档中追加废弃说明。

**督促机制**：AI 每次执行步骤2时，在摘要中列出"澄清中 ≥ 3 轮"的记录，提示人类注意长期未解决的项。

### 缺陷/问题

1. **AI 登记**：AI 发现缺陷或收到反馈后，创建 `_inputs/20-issues/ISSUE-YYYYMMDD-NNN.md`（含问题描述+影响评估摘要），追加到此表（生命周期状态="已登记"）
2. **AI 建议**：AI 评估缺陷影响范围和紧急程度，输出 AI 建议（"建议待处理"/"建议暂搁置"/"建议终止"）
3. **人类决策**：人类浏览 AI 评估摘要，设置待处理状态（"待处理"→继续；"暂搁置"→暂停；"已终止"→废弃）
4. **AI 处理**：AI 筛选"待处理状态=待处理"且条目类型=缺陷的记录，更新生命周期状态="处理中"，按分类和定位自动选择处理模式执行 t02/t03/t04
5. **完成**：AI 完成修复后更新生命周期状态="待确认"，AI 建议="建议待处理"；人类确认后更新生命周期状态="已基线化"，将 issue 文件移入 `_inputs/30-archived/`

### 架构变更

1. **AI 检测或人类报告**：Source 1 构件（`20-eos-design/10-eos-system-design/`）发生变化后，AI 评估变更影响，创建变更详情文件到 `_inputs/20-issues/CHANGE-YYYYMMDD-NNN.md`，追加到此表（条目类型="架构变更"，生命周期状态="已登记"）
2. **AI 建议**：AI 评估变更影响范围和紧急程度，输出 AI 建议（"建议待处理"/"建议暂搁置"/"建议终止"）
3. **人类决策**：浏览 AI 的影响评估，设置待处理状态（"待处理"→继续；"暂搁置"→暂停；"已终止"→废弃）
4. **AI 处理**：筛选"待处理状态=待处理"的记录，更新生命周期状态="处理中"，按变更类型执行对应操作（更新 PA → 评估上游影响 → 补充推导）
5. **完成**：AI 完成处理后更新生命周期状态="待确认"，AI 建议="建议待处理"；人类确认后更新生命周期状态="已基线化"，将变更详情文件移入 `_inputs/30-archived/`
