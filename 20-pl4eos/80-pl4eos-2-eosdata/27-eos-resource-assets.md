## AI读取文档说明

本文件是资源资产类树形资产文档。资源条目数量会随 EOS 设计和产品数据开发逐步增长，AI 读取本文件时应按需读取，不应默认读取全部正文。

默认读取顺序：

1. 读取文档元信息和本说明，确认文档用途、状态和读取规则。
2. 读取 `0. 资源资产全貌`，建立资源类型、状态和分片的整体感知。
3. 读取 `1. 资源处理队列`，确定本轮是否存在资源新增、扩展、归并、废弃或待确认引用。
4. 读取 `2. 资源索引 / 分片索引`，按资源类型、适用业务域、资源 ID 或状态定位目标正文块。
5. 仅读取目标资源正文块、追溯关系指向的相关块，以及本轮需要写回的反馈区或建议区。

扩大读取范围仅在以下情况允许：

- 资源索引缺失或块 ID 无法定位。
- 资源引用关系断裂，无法判断资源被哪个 BA 计划定义、业务定义或产品数据活动使用。
- 候选资源存在同名、同义、边界冲突或归属争议。
- 需要执行全局一致性检查或批量迁移。
- 人类明确要求通读全文。

禁止事项：

- 不得在常规 Skill 运行中默认全量读取资源正文块。
- 不得将废弃资源或旧版正文作为当前可引用资源参与判断。
- 不得只凭对话上下文推进资源状态；人类反馈必须先写入反馈区、资源处理队列或转交记录。

## 3. 规则区 / 口径区

### 3.1 资源资产定义

资源资产是支撑 EOS 产品数据开发管理定义、计划定义、业务定义配置生成或运行承接的可复用能力、工具、模板或指南。

资源资产不等同于业务定义、业务引擎或文档资产：

| 类型 | 承载文档 | 边界 |
|------|----------|------|
| 业务定义 | `24-eos-business-definitions.md` | 描述业务操作、编排定义或计划定义本身 |
| 引擎模型 | `25-eos-engine-models.md` | 描述生成或运行业务定义所需的平台引擎能力 |
| 文档资产 | `26-eos-document-definitions.md` | 描述输入、输出、规则、状态、协同、指标等信息载体 |
| 资源资产 | `27-eos-resource-assets.md` | 描述完成产品数据开发或专业作业所需的工具、模板、能力和指南 |

### 3.2 资源类型

| 资源类型 | 定义 | 示例 |
|----------|------|------|
| CAX工具 | 支撑专业工程作业的工具资源 | CAD、CAE、CAM、PLM、仿真工具、工艺设计工具 |
| AI工具 | 支撑 AI 辅助生成、抽取、审查或自动化处理的工具资源 | 文档生成工具、需求抽取工具、审查助手、分类器 |
| 模板 | 支撑标准化输入、输出、计划或配置的结构化样板 | 计划模板、评审表模板、配置模板、报告模板 |
| 专业能力 | 支撑判断、开发、审查或验证的人类或组织能力 | 工艺设计能力、结构设计能力、质量审查能力 |
| 作业指南 | 支撑作业执行、审查和交付的规则性指南 | 设计指南、建模指南、审查规程、发布指南 |

### 3.3 写回规则

1. `wft02a` 可读取本文件形成资源引用或资源资产建议，不直接正式写回资源资产。
2. `wft02b` 可根据已确认 BA 主结构和人类确认结果写回资源引用或资源建议。
3. 已确认资源可被计划定义、产品数据开发管理定义或业务定义初步方案正式引用。
4. 待确认或疑似资源只能作为待反馈项，不得作为稳定资源依赖。
5. 若资源缺口改变业务编排机制、状态机制、规则机制、协同机制、指标机制或资源机制，应形成 STR 缺口反馈，不应只在本文件中新增资源。
