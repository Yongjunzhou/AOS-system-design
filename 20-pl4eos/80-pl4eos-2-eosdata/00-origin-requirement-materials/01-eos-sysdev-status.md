# EOS 系统设计状态文档

**用途**：设计链全流程状态追踪的共享接口。AI 通过此表感知上游产出、通知下游，人类通过此表做批量决策（选材、确认进入下游）。本表是材料级和条目级状态追踪的**权威源**——各产品文档头部的内嵌状态表由本表对应子表派生。

**同步契约**：AI 每次写入时必须**先更新本表，再从本表推导写入各产品文档的内嵌状态表**。若检测到两者不一致，以本表为准重建内嵌状态表。

---

## 一、OR 阶段

### 表A — OR原料处理建议及进展

| 记录ID | 内容摘要 | 源文件 | 文本化文档 | 生命周期 | AI建议 | 人类决策 | 操作指引 | 最后更新 |
|--------|---------|--------|-----------|---------|-------|---------|---------|---------|

**粒度**：一行/原始材料（PDF/邮件/纪要等），追踪从入库到切分完成的整段旅程。

**生命周期取值**：已登记 → 已文本化 → 已切分 → 已废弃

**控制流**：人类设 `人类决策=同意切分` → ort01 选材入切分。人类设 `人类决策=已终止` → 废弃处理。

**字段说明**：

| 字段 | 写入者 | 说明 |
|------|--------|------|
| 记录ID | AI（首次登记） | 自动生成的唯一标识，如 ORT-001 |
| 内容摘要 | 各步更新 | 该材料当前处理阶段的一句话摘要 |
| 源文件 | ort00 | 原始文件名（含扩展名） |
| 文本化文档 | ort00（首次）/ ort01（更新） | ort00 写入文本化 `.md` 文件名；ort01 切分后更新为各切分文件 `.md` 名 |
| 生命周期 | 各步转迁 | 已登记/已文本化/已切分/已废弃 |
| AI建议 | 各步更新 | AI 对当前阶段的判断（如"可以切分"） |
| 人类决策 | 人类 | 空（未处理）/ 同意切分（选入切分）/ 已终止（废弃） |
| 操作指引 | 各步更新 | 人类下一步应执行的操作 |
| 最后更新 | 各步更新 | 格式 YYYY-MM-DD |

### 表B — OR切分处理建议及进展

| 源文件 | 子文件 | 内容摘要 | 生命周期 | AI建议 | 人类决策 | last_size | iteration_count | unresolved_count | 操作指引 | 最后更新 |
|--------|--------|---------|---------|-------|---------|----------|----------------|-----------------|---------|---------|

**粒度**：一行/切分子文件（`-sub-{seq}.md`），追踪从切分创建到规范化完成的子文件预处理全生命周期。

**生命周期取值**：

```
已切分 → 澄清中 → 已澄清 → 待确认规范方案 → 已规范化 → 已废弃
```

| 阶段 | 说明 | 管理步骤 |
|------|------|---------|
| 已切分 | 切分执行完成 | ort01 完成 |
| 澄清中 | AI 已做首轮标注或正在迭代，等待人类反馈 | ort02 迭代中 |
| 已澄清 | 三项条件全满足，澄清完成 | ort02 完成 |
| 待确认规范方案 | AI 已给出规范化方案，等待人类确认 | ort03 等待反馈 |
| 已规范化 | 规范化完成，OR 条目已写入基线 | ort03 完成 |
| 已废弃 | 子文件不再处理 | 任何步骤 |

**控制流**：
- 人类设 `人类决策=同意澄清` → ort02 选材入澄清
- 人类设 `人类决策=同意规范化` → ort03 选材入规范化
- 人类设 `人类决策=已终止` → 废弃处理

**单对字段覆盖机制**：生命周期转迁时，AI 更新 `AI建议` 为针对新阶段的前视建议，同时 `人类决策` **自动清空**，等待人类针对新建议重新决策。

**字段说明**：

| 字段 | 写入者 | 说明 |
|------|--------|------|
| 源文件 | ort01 | 所属原始材料文件名 |
| 子文件 | ort01 | 切分子文件路径（`-sub-{seq}.md`） |
| 内容摘要 | 各步更新 | 该子文件核心内容一句话摘要 |
| 生命周期 | 各步转迁 | 见上方生命周期取值 |
| AI建议 | 各步更新 | 对当前阶段的前视建议（如"可以澄清""可以规范化"） |
| 人类决策 | 人类 | 空（未处理）/ 同意澄清 / 同意规范化 / 已终止 |
| last_size | ort02/ort03 | 最近处理时的正文区字节数，用于增量检测 |
| iteration_count | ort02 | 澄清迭代轮次计数 |
| unresolved_count | ort02 | 当前未解决的模糊项数量 |
| 操作指引 | 各步更新 | 人类下一步应执行的操作 |
| 最后更新 | 各步更新 | 格式 YYYY-MM-DD |

### 表C — OR基线处理建议及进展

| 源文件 | OR编号 | 内容摘要 | 标题 | 类型 | 优先级 | 当前状态 | 分批 | AI建议 | 操作指引 | 最后更新 |
|--------|--------|---------|------|------|--------|---------|------|--------|---------|---------|

**粒度**：一行/OR条目，追踪规范化产出的每条 OR 基线条目从生成到进入 t01 分解分配的全过程。

**状态取值**：待分解映射 → 待确认方案 → 迭代中 → 已确认 → 验证中 → 已验证冻结 → 已废弃

| 状态 | 含义 | 管理步骤 |
|------|------|---------|
| 待分解映射 | 初写入（本步设定初始值） | 本步写入 → t01 启动时自动扫描 |
| 待确认方案 | t01 已给出分解映射方案，等人类确认 | t01（人类审阅） |
| 迭代中 | 人类已反馈，AI 需修改方案 | t01（AI 按反馈调整） |
| 已确认 | 分解映射方案已通过 | t06（等待全链走通） |
| 验证中 | 至少一条子路径已在下游流转 | 自动进入（无需操作） |
| 已验证冻结 | 全链验证通过 | t06 批量推进 |
| 已废弃 | OR 条目被废弃 | 本步/下游均可触发 |

**分批取值**：**本批**→选入当前批次进入 t01 分解分配 | **—**→暂不处理

**控制流**：人类设 `分批=本批` → AI 在下一轮 t01 中读取处理。
