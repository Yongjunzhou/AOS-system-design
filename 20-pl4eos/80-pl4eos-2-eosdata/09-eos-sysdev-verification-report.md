## AI读取文档说明

本文件是验证运行数据文档，验证任务、结果和问题数量会随设计迭代增长。AI 读取本文件时应先读取验证全貌、任务队列和结果索引，再按任务 ID 或问题 ID 读取目标正文块。

默认读取顺序：

1. 读取文档元信息和本说明，确认验证范围和当前文档状态。
2. 读取 `0. 验证报告全貌`，了解验证类型、覆盖范围和结论摘要。
3. 读取 `1. 验证任务状态与处理队列`，确认本轮验证任务、活动锁和当前待办 ID。
4. 读取 `2. 验证结果索引 / 问题索引`，按验证任务 ID、问题 ID 或源节点定位正文块。
5. 仅读取目标验证块、相关问题块和必要追溯关系，不默认读取全部验证结果。

扩大读取范围仅在以下情况允许：

- 验证任务需要执行全局覆盖率、1:1 分配约束或文档规范化检查。
- 问题根因无法通过单个任务块或追溯关系定位。
- 验证结论需要跨多个问题块合并判断。
- 人类明确要求通读验证报告。

禁止事项：

- 不得在常规 Skill 运行中默认读取全部验证结果块。
- 不得将 `01~08` 的正文复制到本文档。
- 不得只凭对话上下文关闭问题；关闭必须记录验证证据、处理结果和人类决策。

## 3. 规则区 / 口径区

### 3.1 验证规则

| 规则ID | 规则名称 | 判定口径 | 问题输出 |
|--------|----------|----------|----------|
| VR-ALLOC-1TO1 | 1:1 分配约束 | 上层详细末级只能分配到唯一一个下层架构末级 | 重复分配、未分配 |
| VR-SUPPORT-NTO1 | N:1 承接支持 | 下游可承接多个上游，但需解释合理性 | 过度聚合、过度分解 |
| VR-TRACE-FWD | 正向追溯 | OR → STR → BA → SysReq → PA 链路可通达 | 正向断链 |
| VR-TRACE-BWD | 反向追溯 | PA → SysReq → BA → STR → OR 链路可通达 | 来源缺失 |
| VR-DOC-STRUCT | 文档结构规范 | 文档具备 AI 读取说明、索引、正文块、追溯和反馈区 | 结构缺失 |
| VR-ASSET-REF | 资产引用规范 | 过程文档引用 `21~27` 资产时目标资产存在且状态可用 | 资产引用断裂 |

### 3.2 问题严重度

| 严重度 | 含义 | 处理要求 |
|--------|------|----------|
| P1 | 阻断下游 Skill 或导致设计链路不可验证 | 必须优先修复或人类裁定 |
| P2 | 影响局部设计质量或资产一致性 | 应在本轮或下一轮处理 |
| P3 | 表述、索引或轻量规范问题 | 可排入后续治理 |
