# OR → STR 架构 · Skill B · 文档归属与资产写回
**Skill ID**: wft01b · **Phase 2: OR → STR Settlement**

> **概述**：对 Skill A 已确认的 STR 场景定义执行场景定义归属基准确认、广义文档节点归属与资产写回。Skill B 先明确本轮 STR 场景定义自身所属业务域、产品类型和用户角色，再按十条信息链顺序逐链处理，每次只处理当前一条信息链上的广义文档 / 信息载体。信息链仅作为分析和分组维度，不作为产品归属单位；AI 必须对当前链上的每个广义文档节点分别判定其所属业务域、产品类型和构件类型，并记录归属依据。若 23 资产无合适承接项，则 AI 给出新增 / 扩展建议，经人类反馈确认后再写回资产。`wft01` 无 Phase 2 下半，全部信息链完成后进入 `wft02` 前准备态。
>
> **输入 → 输出**：已确认的 STR 场景定义 → 当前信息链的广义文档节点归属方案局部写入 `02-*.md` + `23/26/22` 资产按块写回 + 信息链归属状态推进；全部信息链完成后同步推进 `01/02` 状态、索引、追溯引用
>
> **适用场景**：瀑布式系统设计第1步 · Phase 2。
>
> **设计模式参考**：[91-doc-paired-skill-spec.md](91-doc-paired-skill-spec.md)
>
> **版本**：v1.21 | **修订**：2026-06-20

## 一、职责定位

Skill B 负责：

1. 接收已确认的 STR 场景定义方案
2. 明确本轮 STR 场景定义自身所属业务域、产品类型和用户角色，并作为后续十链节点归属的场景归属基准
3. 按十条信息链顺序，针对当前信息链中的每个广义文档 / 信息载体节点，分别判定业务域、产品类型、构件类型
4. 按节点所属信息链类型，记录其与执行业务链节点、项目管理构件、资源提供方、公共协同 IPO 输入或被度量对象的关系
5. 基于 23 输出产品架构资产执行文档归属匹配；无合适承接项时提出新增 / 扩展建议
6. 维护 26 广义文档资产定义与归属
7. 维护 23 输出产品架构资产引用
8. 维护 22 非功能需求资产引用或候选线索
9. 推进当前信息链归属状态；全部信息链完成后推进 OR / STR 状态，使节点进入下游 `wft02` 的准备态

Skill B 不负责：

1. 重做 OR→STR 主结构设计
2. 修改 Skill A 已确认的节点边界与承接关系
3. 改变 Skill A 已确认的场景边界、角色、触发和主承接关系
4. 生成下游详细定义

## 二、输入与路由

### 标准输入包

1. 已确认的 STR 场景定义方案
2. 当前 23 / 26 / 22 资产内容
3. 文档归属反馈（如存在待确认归属项）
4. 树画像 / 资产全貌、状态队列、节点索引与本轮正文块定位信息
5. 信息链归属处理状态

### 进入条件

本轮处理的 STR 节点满足以下条件时可进入本 Skill：

1. 当前状态为 `待文档归属`

说明：`待文档归属` 表示该 STR 场景定义已经由 Skill A 完成方案确认，可以进入逐文档归属判定。若存量节点仍停留在 `待确认方案` 且反馈区已有 `[同意]`，AI 应先按 Skill A 规则将节点状态推进为 `待文档归属`，再进入本 Skill。

### 活动 STR 归属节点锁定

`wft01b` 同一时间只能允许一个 STR 场景定义处于文档归属处理状态。一个 STR 节点进入 `wft01b` 后，在其十条信息链未全部写回、节点未推进到 `待分解分配` 前，不得再让新的 STR 节点进入 `wft01b`。

活动 STR 归属节点判定：

1. `02-*.md` 中存在某个 STR 节点处于 `待文档归属`，且 `### 信息链归属处理状态` 仍有 `待归属`、`待反馈`、`可写回` 或 `回退SkillA`。
2. `02-*.md` 中存在该 STR 节点的文档归属待办锁定记录，且当前待办 ID 尚未完成。
3. `02-*.md` 状态队列中该 STR 节点已被标记为进入 `wft01b`，但尚未记录 `wft01b` 处理完成。

活动 STR 归属节点完成判定：

1. 该 STR 节点十条信息链均为 `已写回`，且节点状态已推进到 `待分解分配`。
2. 该 STR 节点被人类明确撤销、废弃或回退 `wft01a`，不再由当前 `wft01b` 继续处理。

锁定规则：

1. Step Start 必须先检查是否存在活动 STR 归属节点，再检查新的 `待文档归属` STR 节点；但若该活动节点已由人类选择进入下游 `wft02a`，本轮只执行活动锁释放 / 转交记录。
2. 若存在活动 STR 归属节点，AI 只能处理该节点的当前信息链、当前文档归属反馈、资产写回或回退；不得选择新的 STR 节点。
3. 若存在多个活动 STR 归属节点，说明状态队列不一致，AI 必须停止新增处理，输出状态冲突说明，并要求人类先选择保留一个活动 STR 归属节点。
4. 只有不存在活动 STR 归属节点时，AI 才能从 `待文档归属` 队列中选择一个新的 STR 节点进入本 Skill。
5. 新 STR 节点一旦进入本 Skill，AI 必须在 `02-*.md` 状态队列、STR 节点块或本轮运行摘要中记录活动锁定关系。
6. 活动锁定关系至少包含：`活动STR归属节点ID`、`当前信息链ID`、`当前待办ID`、`锁定状态`、`锁定原因`、`锁定开始时间或轮次`、`释放条件`。
7. `锁定状态` 建议取值为：`锁定中`、`待反馈`、`待写回`、`待下一链`、`待进入wft02a`、`已释放`、`已转交wft01a`、`已转交wft02a`、`冲突待处理`。
8. 若仅有一个 STR 节点满足 `待文档归属` 且不存在活动 STR 归属节点，AI 可直接进入本 Skill；该行为视为本轮选择该 STR 节点进入 `wft01b`，必须同步写入活动锁定关系。

活动锁定记录建议格式：

```text
活动STR归属节点ID：
当前信息链ID：
当前待办ID：
锁定状态：锁定中 / 待反馈 / 待写回 / 待下一链 / 待进入wft02a / 已释放 / 已转交wft01a / 已转交wft02a / 冲突待处理
锁定原因：
锁定开始：
释放条件：
```

### 不可修改项

进入 Skill B 后，AI 不能再修改以下内容；如需修改，必须回退 `wft01a`：

1. STR 节点清单
2. 节点类型（STR-F / STR-NFR）
3. 节点边界
4. OR→STR 宿主分配
5. 复用 / 改进 / 新增 决策
6. 主承接关系
7. STR-F 场景定义自身的业务域、用户角色、触发和职责边界
8. STR-F 场景定义中的信息载体清单与场景指标清单

## 三、操作流程

### Step 布局总览

| Step | 名称 | 产出 | 写资产 |
|------|------|------|--------|
| Start | 前置校验与路由 | 就绪 / 结束 | — |
| 1 | 资产读取 | 已读取资产内容 | — |
| 2 | 当前链文档清单读取 | 当前链待归属文档清单 | — |
| 3 | 当前链文档归属判定 | 当前链广义文档节点归属方案 | `02-*.md` |
| 4 | 当前链文档归属反馈处理 | 确认后的当前链归属方案 / 回退 | `02-*.md` |
| 5 | 当前链资产写回 | 当前链 23 / 26 / 22 资产更新与写回摘要 | `23-*.md` / `26-*.md` / `22-*.md` / `02-*.md` |
| 6 | 当前链状态推进 | 信息链归属状态更新；全部完成时 OR / STR 状态更新 | `01-*.md` / `02-*.md` |
| End | 运行结束输出 | 人类下一步操作提示 | 独立成章，见 §五 |

### Step Start · 前置校验与路由

通过 `02-*.md` 的树画像 / 状态队列 / 节点索引定位本轮 STR 节点正文块，读取该节点状态、`### 信息链归属处理状态`、`### 文档归属方案` 和 `### 文档归属反馈区`，按以下规则进入后续步骤：

| 条件 | 路由 |
|------|------|
| STR 节点 = `待分解分配` 且人类已回复 `选择进入wft02a`，但活动锁尚未释放或转交 | 只执行活动锁释放 / 转交记录，不进入文档归属处理 |
| 存在一个活动 STR 归属节点 | 只处理该 STR 节点当前链的反馈、写回、推进或回退，不得进入新 STR 节点 |
| 存在多个活动 STR 归属节点 | 停止执行，输出状态冲突说明，要求人类选择保留一个活动 STR 归属节点 |
| STR 节点 = `待文档归属` 且人类已回复 `选择进入wft01b` 或 `选择：<STR节点ID>`，且不存在活动 STR 归属节点 | 执行 Step 1 → Step 2 → Step 3，并写入活动 STR 归属节点锁定关系 |
| 存在多个 STR 节点 = `待文档归属`，且不存在活动 STR 归属节点 | 输出 `等待选择进入wft01b`，当前待办 ID 为 AI 推荐的一个 STR 节点 ID |
| 仅存在一个 STR 节点 = `待文档归属`，且不存在活动 STR 归属节点 | 执行 Step 1 → Step 2 → Step 3 |
| 信息链归属处理状态全部为 `已写回` | 执行 Step 6，推进到 `待分解分配` |
| 节点状态不是 `待文档归属` | 输出不可进入说明后结束 |
| 当前链状态为 `可写回` | 执行 Step 5 → Step 6 |
| 当前链无 `### 文档归属方案` | 执行 Step 1 → Step 2 → Step 3 |
| 当前链有 `### 文档归属方案`，且反馈区有 `[同意] / [修改] / [驳回]` | 执行 Step 4 |
| 当前链文档归属方案中所有条目均为 `可写回` | 执行 Step 5 → Step 6 |
| 当前链有 `### 文档归属方案`，且 `### 文档归属反馈区` 为空 | 输出 `等待建议方案反馈`，当前待办 ID 为一个文档归属建议 ID 或 23 资产建议 ID |

说明：

1. 当前链是 `### 信息链归属处理状态` 中顺序最靠前、处理状态不是 `已写回` 的信息链。
2. 重新运行 Skill B 时，若当前链已存在文档归属方案，AI 不得覆盖该方案或清空反馈区。
3. 若需要重做当前链文档归属判定，只能由 Step 4 的 `[修改] / [驳回]` 分支触发。
4. 若检测到文档归属方案和反馈区内容冲突，应优先保留人类反馈，并输出需要人工确认的冲突说明。
5. 路由优先依据当前链状态；“当前链文档归属方案中所有条目均为 `可写回`”仅用于兼容旧方案或状态缺失场景。
6. 只要存在活动 STR 归属节点未完成，AI 不得响应其他 STR 节点的 `待文档归属` 入口，即使这些节点也满足进入条件。
7. 若多个 STR 节点同时满足 `待文档归属`，AI 不得自动选择其中一个进入本 Skill，必须先输出 `等待选择进入wft01b`。
8. 当前 STR 节点十链全部写回并推进到 `待分解分配` 时，AI 必须记录活动锁进入 `待进入wft02a` 状态。
9. 若人类已对某个待文档归属 STR 节点回复 `选择进入wft01b` 或 `选择：<STR节点ID>`，下一次 `wft01b` 应先将选择记录转写到 `02-*.md` 状态队列或本轮运行摘要，再让该 STR 节点进入当前链处理；不得再次输出同一个 STR 节点选择提示。
10. 若本轮 Step End 的当前待办 ID 是下一条信息链 ID，且活动 STR 归属节点尚未完成，则下一次 `wft01b` 必须先检查人类是否已对该信息链回复 `选择进入wft01b`；未选择时继续输出 `等待选择进入wft01b`，已选择时进入该链处理。
11. 人类对该 STR 节点回复 `选择进入wft02a` 后，下一次 `wft01b` 只允许记录活动锁释放或转交，不得重新执行文档归属处理。

信息链归属处理状态取值：

| 状态 | 含义 | Skill B 行为 |
|------|------|--------------|
| `待归属` | 当前链尚未形成文档归属方案 | 执行 Step 2 → Step 3 |
| `待反馈` | 当前链存在 `23-Q2 / 23-Q3` 或其他待确认归属项 | 等待人类反馈；有反馈时执行 Step 4 |
| `可写回` | 当前链归属方案已确认，可以写回资产 | 执行 Step 5 |
| `已写回` | 当前链资产写回完成 | 选择下一条未写回信息链 |
| `回退SkillA` | 当前链发现结构问题，需要修改场景定义 | 停止 Skill B，按回退规则运行 `wft01a` |

### Step 1 · 资产读取

1. 加载 `02-*.md` STR 树画像 / 状态队列 / 节点索引，读取本轮 STR 节点正文块、场景定义归属基准、信息载体清单、场景指标、信息链归属处理状态、当前链文档归属方案和反馈区
2. 加载 `23-eos-output-architecture.md` 资产全貌层，按信息载体名称、业务域、典型文档、产品类型和构件类型筛选候选分片；只读取候选业务域 / 产品类型 / 构件类型块
3. 加载 `26-eos-document-definitions.md` 资产全貌层，按信息载体名称、文档性质和 26 方案级判断筛选候选文档定义块
4. 加载 `22-eos-nfr-taxonomy.md` 资产全貌层和与本轮场景指标、风险合规链、规则知识链相关的 NFR 分片
5. 加载 `01-*.md` 树画像 / 状态队列 / 节点索引，读取本轮 STR 来源 OR 块和状态队列相关行
6. 按追溯关系读取必要的上游 / 下游相关块，不默认全量读取大文档正文

读取要求：

1. 读取系统设计过程文档和资产文档时，必须遵守 [文档编写约定 §十四](../../../../00-generalspec/00-doc-conventions.md)：先读树画像 / 资产全貌、状态队列和节点索引，再读本轮正文块和必要相关块。
2. 23 号资产的产品类型 + 构件类型全貌层是文档归属判断的必读入口；末级明细、完整 I/O 和历史记录只按候选分片读取。
3. 只有在索引缺失、关系断裂或执行全局一致性检查时，才允许说明原因后扩大读取范围。

局部细节选择规则：

1. **场景线索**：根据本轮 STR 节点 ID，在 `02-*.md` 节点索引中定位场景定义正文块，读取已确认的场景定义归属基准、信息载体清单、场景指标、信息链归属处理状态、当前链文档归属方案和反馈区；这是本轮最小必读范围。
2. **文档线索**：以每个信息载体为处理对象，根据载体名称、归属链、文档性质、产生方式、一句话职责、来源 OR 和 26 方案级判断，筛选 `26` 文档定义候选块。
3. **归属线索**：根据业务域、信息载体服务的业务对象、操作角色、典型文档、产品类型和构件类型，在 `23` 资产全貌中筛选候选业务域 / 产品类型 / 构件类型分片。
4. **指标线索**：根据场景指标的数据来源、指标类别和口径，筛选 `22` 中与效率、质量、成本、风险、协同、体验等相关的 NFR 分片；仅当指标需要写入或引用 22 资产时读取候选正文块。
5. **匹配线索**：对每个待归属文档，先用 `23` 的产品类型 + 构件类型全貌层判断是否存在直接承接项；只有出现同名 / 同义 / 相近构件、边界冲突或新增建议时，才读取候选构件的详细定义块。
6. **反馈线索**：若本轮为文档归属反馈处理，只读取文档归属反馈区指向的信息载体、23 资产建议和候选资产块；不得重新覆盖已形成的当前链文档归属方案。
7. **写回线索**：进入资产写回前，只读取将被写入的 `23/26/22` 本轮资产块及其索引行，用于去重、补充引用和维护状态；不得因写回而默认读取资产全文。
8. **扩大读取触发**：只有当候选归属无法判定、同名资产冲突、产品类型 / 构件类型边界不清、索引缺失、本轮写回块无法定位或需要做全局一致性检查时，才可扩大读取范围，并在输出中说明扩大读取的原因。

### Step 2 · 当前链文档清单读取

1. 从 `### 信息链归属处理状态` 选择当前链。
2. 从 STR-F 场景定义读取场景定义归属基准：业务域、产品类型、用户角色。
3. 从 STR-F 场景定义读取当前链的信息载体；若当前链为 `场景指标链`，读取场景指标。
4. 对当前链每个信息载体提取：载体名称、归属链、文档性质、产生方式、一句话职责、来源 OR、26 方案级判断。
5. 对当前链场景指标提取：指标名称、指标类别、指标含义、主要数据来源。
6. 形成本轮场景定义归属基准、当前链待归属文档清单和指标线索清单。
7. 若发现当前链信息载体或场景指标本身需要新增、删除、改名或改变边界，立即触发结构问题回退 `wft01a`。

### Step 3 · 当前链文档归属判定

Step 3 的归属单位是“广义文档节点”，不是信息链。信息链只用于分组和说明节点来源；每个已确认进入场景定义的广义文档节点，都必须分别判定业务域、产品类型和构件类型，并记录归属依据。

#### 3.1 十类节点归属规则

对当前链每个待归属广义文档节点，按其所属信息链类型执行以下判定：

| 节点类型 | 归属锚点 | 归属规则 |
|------|---------|---------|
| 执行业务链节点 | 被执行、创建、处理或流转的产品数据 | 归属到该产品数据所在产品类型和构件类型；归属锚点是业务对象，不是页面、列表、表单等显示形态 |
| 管理业务链节点 | 被管理的业务对象、业务过程或管理控制对象 | 归属到执行业务链主产品类型下的项目管理构件，并进一步归入对应管理业务 WBS；跨多个构件时拆分为多个 WBS 引用，或定义跨构件管理 WBS 并列出覆盖范围 |
| 个人绩效链节点 | 当前场景内个人对业务/项目运行的贡献、达成、质量和效率 | 归属到执行业务链主产品类型下的项目管理构件，作为个人绩效 WBS；企业统一绩效考核场景另行记录向统一绩效产品的汇总/引用关系 |
| 组织绩效链节点 | 当前场景内组织、团队、部门或项目对业务/项目运行的贡献、达成、质量和效率 | 归属到执行业务链主产品类型下的项目管理构件，作为组织绩效 WBS；企业统一绩效考核场景另行记录向统一绩效产品的汇总/引用关系 |
| 规则知识链节点 | 所附属的执行业务链节点 | 作为执行业务链节点的质量支撑型附属文档处理，产品类型和构件类型继承所附属执行节点；内容包括准则、指南、范例、模板和知识说明 |
| 资源需求链节点 | 支持维资源提供方 | 由执行业务链节点发起，按资源类型归属到支持维对应资源提供类产品类型和构件类型，并由后者负责资源提供；执行业务链节点记录为需求发起方和场景引用方 |
| 风险合规链节点 | 所附属的执行业务链节点 | 与规则知识链同类处理，作为执行业务链节点的附属规则性文档；产品类型和构件类型继承所附属执行节点，内容为事先、事中、事后的风险与合规处理规则 |
| 状态事件链节点 | 所记录的执行业务链节点 | 作为执行业务链节点的系统过程数据和运行过程记录型附属文档，产品类型和构件类型继承所附属执行节点；若表达跨多个业务对象的状态汇总、监控或分析，则转入管理业务链处理 |
| 协同交互链节点 | 公共协同或目标承接方 | 通常作为系统公共协同 IPO 的输入，优先归属到系统公共协同类产品类型及构件类型；若协同事项明确进入某专业业务产品或外部协同/接口产品，则按目标承接方归属 |
| 场景指标链节点 | 被度量对象 | 独立于个人绩效和组织绩效，按每个指标被度量对象分别确定产品类型和构件类型；评价整个角色场景端到端运行质量的指标，归属到角色场景产品类型下的场景定义构件 |

## 五、运行结束输出（Step End）

Step End 的作用是把本轮文档归属结果翻译成人类下一次运行 `wft01b` 前的最小待办。AI 不只输出“完成 / 未完成”，还必须根据当前运行结果，为人类确定一个且仅一个当前动作。

推荐交互方式：

1. 人类优先通过对话回复。
2. AI 先将对话反馈或选择转写到 `### 文档归属反馈区`、信息链归属处理状态、状态队列或人类决策字段，再继续处理。
3. 对话是反馈输入通道，文档中的反馈区 / 状态队列是权威留痕。
4. AI 不得只凭对话上下文推进状态或写回资产，必须先完成规范转写。

### 单 ID 最小待办原则

1. 每次 Step End 只提示人类处理一个 ID。
2. 该 ID 可以是一个文档归属建议 ID、一个 23 资产建议 ID、一个当前链 ID、一个需要回退 `wft01a` 的 STR 节点 ID、一个待进入 `wft02a` 的 STR 节点 ID。
3. AI 必须同时给出剩余待办 ID 数量，帮助人类知道当前队列规模。
4. 若上一次 Step End 已提示某个当前待办 ID，且该 ID 尚未被人类确认完成、明确切换或变为不可操作，下次运行必须继续提示同一个 ID。
5. 人类对当前 ID 的回复可以是 `同意`、`修改：……`、`驳回：……`，或 `选择：……`。
6. 只有当前 ID 已完成、被人类明确切换，或因状态变化不再可操作时，AI 才能按阻断优先级、状态时间和推荐顺序自动选择下一个当前 ID，并说明切换原因。
7. 当前 ID 处理完成后，下一次运行 `wft01b` 再提示下一个 ID。
8. AI 可以附带其他待办摘要，但不得要求人类同时审阅或选择多个 ID。

### 当前待办 ID 连续性规则

1. Step End 输出的 `当前待办ID` 默认形成“待办锁定”。
2. AI 应在 `02-*.md` 对应 STR 节点的 `### 文档归属反馈区`、信息链归属处理状态、状态队列、待进入 `wft02a` 选择记录或本轮运行摘要中保留该 ID 的待办状态。
3. 若下一次运行时该 ID 仍处于待反馈、待确认、待补充或待选择状态，AI 必须继续提示该 ID，不得因为出现新的待办而切换。
4. 若人类回复 `换一个 / 跳过 / 选择：<另一个ID>`，AI 可以切换当前待办 ID，但必须先记录切换原因。
5. 若当前 ID 已经完成，AI 才从剩余待办中选择下一个 ID。
6. 若当前 ID 因上游状态变化不再可操作，AI 必须说明原 ID 不再可操作的原因，再选择新的当前待办 ID。

### Step End 待办类型

| 待办类型 | 何时出现 | 当前 ID | 人类只需做什么 | 下次运行 |
|---------|----------|---------|----------------|----------|
| `反馈建议方案` | 当前链存在 `23-Q2 / 23-Q3` 或其他待确认文档归属 / 资产建议 | 一个文档归属建议 ID 或 23 资产建议 ID | 对该 ID 回复 `同意 / 修改 / 驳回` | `wft01b` |
| `选择进入wft01b` | 当前链已写回，但仍有后续信息链未写回 | 一个信息链 ID | 对该 ID 回复 `选择进入wft01b` | `wft01b` |
| `选择进入wft01a` | 当前链发现结构问题，需要修改场景定义、信息载体、指标或 OR 承接关系 | 一个 STR 节点 ID | 对该 ID 回复 `选择进入wft01a`，或补充结构反馈 | `wft01a` |
| `选择进入wft02a` | 十条信息链均已写回，STR 节点已推进到 `待分解分配` | 一个 STR 节点 ID | 对该 ID 回复 `选择进入wft02a` | `wft02a` |

### 输出结果

本 Skill 结束时，只允许输出以下四类结果之一，且每类结果必须带一个当前待办 ID：

| 输出结果 | 对应待办类型 | 当前 ID | 人类下一步 |
|---------|--------------|---------|------------|
| `等待建议方案反馈` | `反馈建议方案` | 一个文档归属建议 ID 或 23 资产建议 ID | 对该 ID 回复 `同意 / 修改 / 驳回` |
| `等待选择进入wft01b` | `选择进入wft01b` | 一个信息链 ID | 对该 ID 回复 `选择进入wft01b` |
| `等待选择进入wft01a` | `选择进入wft01a` | 一个 STR 节点 ID | 对该 ID 回复 `选择进入wft01a`，或补充结构反馈 |
| `等待选择进入wft02a` | `选择进入wft02a` | 一个 STR 节点 ID | 对该 ID 回复 `选择进入wft02a` |

### 最小待办输出格式

AI 在 Step End 应按以下格式输出：

```text
当前结果：等待建议方案反馈 / 等待选择进入wft01b / 等待选择进入wft01a / 等待选择进入wft02a
当前待办ID：
是否延续上次待办ID：
剩余待办ID数量：
待办类型：
为什么先处理这个ID：
人类只需回复：
回复会写入：
下次运行：
其他待办摘要：
```

要求：

1. `当前待办ID` 必须是单个 ID，不得写成多个 ID 列表。
2. `是否延续上次待办ID` 必须写 `是 / 否`；若为 `否`，必须说明当前 ID 已完成、被人类切换或不再可操作的原因。
3. `剩余待办ID数量` 必须是数字；若无法精确统计，应写 `至少 N 个` 并说明统计口径。
4. `人类只需回复` 必须是一句话，且只包含当前 ID 的动作。
5. `其他待办摘要` 只能给数量和类型，不得要求人类同时处理。
6. 若人类直接在对话中回复，AI 应先转写到 `02-*.md` 对应 STR 节点的 `### 文档归属反馈区`、信息链归属处理状态、状态队列或待进入下游选择记录，再执行下一步。

### 结果一：等待建议方案反馈

适用：当前链存在 `23-Q2 / 23-Q3`，需要人类确认文档归属或 23 资产建议。

当前待办 ID：

1. 优先选择当前链中最早进入待确认状态的文档归属建议 ID。
2. 若多个建议同级阻断，优先顺序为：结构风险较高的归属建议、23 资产新增建议、23 资产扩展建议。
3. 若当前待办 ID 未完成，下次运行必须继续提示同一 ID。

人类当前应做什么：

1. 对当前文档归属建议 ID 或 23 资产建议 ID 回复 `同意`、`修改：……` 或 `驳回：……`。
2. AI 收到回复后，先转写为 `### 文档归属反馈区` 的规范记录，再在下一次 `wft01b` 中处理。

对话反馈示例：

```text
修改：同意 DOC-采购申请单 的归属；但 DOC-采购预算占用信息 建议归入采购管理域，并新增预算控制构件建议。
```

文档反馈区备选格式：

```text
[操作] 同意 / 修改 / 驳回

[文档归属反馈]
-

[23资产建议反馈]
-

[其他说明]
-
```

重点查看位置：

1. `02-*.md` 对应 STR 节点当前链的 `### 文档归属方案`
2. `02-*.md` 对应 STR 节点的 `### 文档归属反馈区`
3. 当前待办 ID 对应的文档归属建议或 23 资产建议
4. `23-eos-output-architecture.md` 中相关业务域、产品类型、构件类型

### 结果二：等待选择进入wft01b

适用：当前链已写回，但仍有后续信息链未写回。

口径说明：

1. `待文档归属` 是 `wft01a` 输出的正式入口许可，表示该 STR 节点允许进入 `wft01b`。
2. `选择进入wft01b` 有两种含义：当多个 STR 节点均为 `待文档归属` 时，表示选择本轮 STR 节点；当同一 STR 节点当前链已写回但仍有后续链时，表示选择继续处理下一条信息链。
3. 只有不存在活动 STR 归属节点时，AI 才能提示人类在多个 STR 节点中选择进入 `wft01b`。
4. 若已经存在活动 STR 归属节点，即使其他 STR 节点也是 `待文档归属`，也不得提示选择新 STR 节点。
5. 若多个 STR 节点同时具备入口许可，AI 必须先输出本结果，不能自动进入 Step 1。
6. 若只有一个 STR 节点具备入口许可且不存在活动 STR 归属节点，AI 可以直接进入 Step 1。

当前待办 ID：

1. 必须是下一条待处理信息链 ID。
2. 若当前 STR 归属节点未完成，不得提示其他 STR 节点进入 `wft01b`。

人类只需做什么：

1. 对当前信息链 ID 回复 `选择进入wft01b`。
2. AI 收到回复后，先转写到信息链归属处理状态或本轮选择记录，再在下一次 `wft01b` 中处理该链。

重点查看位置：

1. `02-*.md` 对应 STR 节点的 `### 信息链归属处理状态`
2. 当前链的写回摘要
3. 下一条信息链的信息载体或场景指标

### 结果三：等待选择进入wft01a

适用：发现文档清单、场景边界、OR 承接等结构问题，需要回退 `wft01a`。

当前待办 ID：

1. 必须是需要回退的 STR 节点 ID。
2. 不得同时要求人类审阅当前链文档归属建议。

人类只需做什么：

1. 对当前 STR 节点 ID 回复 `选择进入wft01a`，或回复 `修改：……` 补充结构反馈。
2. AI 收到回复后，先转写到该 STR 节点反馈区，并将节点状态回退为 `迭代中`，再提示重新运行 `wft01a`。
3. AI 必须记录活动 STR 归属锁转交或释放，不得继续处理其他 STR 节点。

重点查看位置：

1. `02-*.md` 对应 STR 节点反馈区
2. 回退记录中标注的结构问题
3. 受影响的信息载体、场景指标、场景边界或 OR 承接关系

### 结果四：等待选择进入wft02a

适用：十条信息链均已写回，STR 节点已推进到 `待分解分配`。

锁定说明：

1. 当 STR 节点已推进到 `待分解分配`，但人类尚未选择进入 `wft02a` 时，活动 STR 归属节点仍保持锁定。
2. 人类对当前 STR 节点 ID 回复 `选择进入wft02a` 后，AI 应先记录活动锁释放或转交状态，再由 `wft02a` 接手处理。
3. 活动锁释放前，`wft01b` 不得进入新的 STR 节点。
4. 若下一次运行 `wft01b` 时检测到人类已回复 `选择进入wft02a`，AI 只执行锁释放 / 转交记录，不得重新进入文档归属判定、反馈处理或资产写回。

释放 / 转交记录至少包含：

1. 活动 STR 归属节点 ID
2. 当前信息链状态：十链均已写回
3. 被选择进入 `wft02a` 的 STR 节点 ID
4. 锁定状态：`已释放` 或 `已转交wft02a`
5. 释放 / 转交时间或轮次
6. 下一步 Skill：`wft02a`

当前待办 ID：

1. 必须是已推进到 `待分解分配` 的 STR 节点 ID。
2. 若还有未完成的当前链归属待办，不得输出本结果。

人类只需做什么：

1. 对当前 STR 节点 ID 回复 `选择进入wft02a`。
2. AI 收到回复后，先转写到 `02-*.md` 状态队列或本轮选择记录，再提示运行 `wft02a`。
3. 该 `选择进入wft02a` 记录是下游 `wft02a` 可识别的入口选择记录；`wft02a` 不应要求人类对同一 STR 节点重复补写 `同意分解分配`。

重点查看位置：

1. `02-*.md` 状态队列和节点索引中该 STR 节点状态是否已为 `待分解分配`
2. `26-eos-document-definitions.md` 中新增或更新的广义文档定义
3. `23-eos-output-architecture.md` 中新增或更新的产品类型 / 构件类型引用
4. `22-eos-nfr-taxonomy.md` 中与场景指标相关的引用或候选线索
