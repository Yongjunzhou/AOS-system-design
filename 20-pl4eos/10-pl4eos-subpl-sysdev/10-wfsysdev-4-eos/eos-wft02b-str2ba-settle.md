# STR → BA 架构 · Skill B · 资产写回
**Skill ID**: wft02b · **Phase 2: STR → BA Settlement**

> **概述**：对 Skill A 已确认的 BA 主结构方案执行正式资产写回和 STR 缺口反馈处理。Skill B 接收 A1 业务配置平台、A2 xx业务运行系统、Bn 平台输出产品三类 BA 方案，维护 `23/24/25/26/27` 资产引用（仅写架构级关系与本体定义，不含初步方案活动描述）；若存在已升级的 BA 完备性建议、STR 回补建议、业务引擎扩展或新增需求，应生成符合 STR 规范的反馈内容并提交人类确认；普通 BA 完备性缺口观察不得写回，只保留观察记录。在 Phase 2 下半将 STR 详细定义正式写回 `03-*.md`。
>
> **输入 → 输出**：已确认的 BA 主结构方案 → 23/24/25/26/27 资产更新 + STR 缺口反馈建议处理 + `03-*.md` STR 详细定义 + `02/04` 状态推进
>
> **适用场景**：瀑布式系统设计第2步 · Phase 2。
>
> **设计模式参考**：[91-doc-paired-skill-spec.md](91-doc-paired-skill-spec.md)
>
> **版本**：v2.13 | **修订**：2026-06-21

## 一、职责定位

Skill B 负责：

1. 接收已确认的 BA 主结构方案
2. 对 A1 / A2 / Bn 三类产品路径执行资产匹配、正式写回和反向引用维护
3. 维护 23 输出产品架构资产、24 业务定义资产、25 引擎模型与配置 IPO 类型资产（仅注册与引用声明）、26 文档定义资产、27 资源资产引用
4. 处理业务定义扩展 / 新增建议
5. 对已升级的 BA 完备性建议、STR 回补建议和业务引擎能力边界变化生成 STR 缺口反馈内容，提交人类确认
6. 将 STR 详细定义正式写回 `03-*.md`
7. 推进 STR / BA 状态，决定回退、重入或节点完成

Skill B 不负责：

1. 重做 STR→BA 主结构设计
2. 修改 Skill A 已确认的 A1 / A2 / Bn 层级边界与主承接关系
3. 修改 wft01a 已确认的 STR 场景边界、信息载体清单或场景指标
4. 将引擎能力的功能定义写入 25（由 `wft03b` 在 SR 层功能定义确认后负责）
5. 将初步方案活动描述写入资产（仅写架构级引用和本体定义）
6. 直接放行未经确认的业务引擎扩展 / 新增需求

## 二、输入与路由

### 标准输入包

1. 已确认 BA 主结构方案
2. A1 / A2 / Bn 产品路径方案
3. 十链消费结果
4. 业务定义资产匹配结果
5. BA 完备性缺口观察、升级判定、STR 回补建议、业务引擎支撑判定与 STR 缺口反馈建议
6. 当前 23/24/25/26/27 资产内容
7. 当前 `03-*.md` STR 详细定义文档
8. 当前 `02/04` 树画像 / 状态队列 / 节点索引与正文块定位信息
9. 人类反馈（如存在资产建议反馈、STR 缺口反馈或详细定义反馈）

### 进入条件

本轮处理的 BA 节点满足以下条件时可进入本 Skill：

1. 当前状态为 `待资产写回`，且已有 `选择进入wft02b` 记录，或已有 `wft02a` 活动锁释放 / 转交记录
2. 或本 Skill 已进入 `待STR反馈确认` / `待确认细节` / `确认中` 重入状态

说明：`待资产写回` 表示 BA 主结构方案已经由 Skill A 完成方案确认，但不等于已经完成 `wft02a` 到 `wft02b` 的交接。若人类尚未回复 `选择进入wft02b`，且未记录 `wft02a` 活动锁释放 / 转交，AI 不得直接进入资产写回处理。若存量节点仍停留在 `待确认方案` 且反馈区已有 `[同意]`，AI 应先按 Skill A 规则将节点状态推进为 `待资产写回`，再提示人类选择进入 `wft02b`。

### 活动 BA 节点锁定

`wft02b` 同一时间只能允许一个 BA 节点处于资产写回与详细定义处理状态。一个 BA 节点进入 `wft02b` 后，在其资产写回、STR 缺口处理和 `03` 详细定义确认未完成前，不得再让新的 BA 节点进入 `wft02b`。

活动 BA 节点判定：

1. `04-*.md` 中存在 BA 节点状态为 `待资产写回`、`待STR反馈确认`、`待确认细节`、`确认中` 或其他未完成状态，且该节点已被选择进入 `wft02b`。
2. `04-*.md` 中存在该 BA 节点的待办锁定记录，且当前待办 ID 尚未完成。
3. `04-*.md` 状态队列中该 BA 节点已被标记为进入 `wft02b`，但尚未记录 `wft02b` 处理完成。

活动 BA 节点完成判定：

1. 该 BA 节点已完成资产写回、STR 缺口处理和 `03` 详细定义确认，并推进到 `待架构映射`。
2. 该 BA 节点被人类明确撤销、废弃或回退 `wft02a`，不再由当前 `wft02b` 继续处理。

锁定规则：

1. Step Start 必须先检查是否存在活动 BA 节点，再检查新的 `待资产写回` BA 节点。
2. 若存在活动 BA 节点，AI 只能处理该 BA 节点的资产建议反馈、STR 缺口反馈、详细定义反馈、写回、状态推进或回退；不得选择新的 BA 节点。
3. 若存在多个活动 BA 节点，说明状态队列不一致，AI 必须停止新增处理，输出状态冲突说明，并要求人类先选择保留一个活动 BA 节点。
4. 只有不存在活动 BA 节点时，AI 才能从已具备入口许可的 BA 节点中选择一个新的 BA 节点进入本 Skill。
5. 新 BA 节点一旦进入本 Skill，AI 必须在 `04-*.md` 状态队列、BA 节点块或本轮运行摘要中记录活动锁定关系。
6. 活动锁定关系至少包含：`活动BA节点ID`、`来源STR节点ID`、`当前待办ID`、`锁定状态`、`锁定原因`、`锁定开始时间或轮次`、`释放条件`。
7. `锁定状态` 建议取值为：`锁定中`、`待反馈`、`待写回`、`待确认细节`、`已释放`、`已转交wft02a`、`冲突待处理`。
8. 若仅有一个 BA 节点满足入口许可且不存在活动 BA 节点，AI 可直接进入本 Skill；该行为视为本轮承接上游已完成的 `选择进入wft02b`，必须同步写入活动锁定关系。

活动锁定记录建议格式：

```text
活动BA节点ID：
来源STR节点ID：
当前待办ID：
锁定状态：锁定中 / 待反馈 / 待写回 / 待确认细节 / 已释放 / 已转交wft02a / 冲突待处理
锁定原因：
锁定开始：
释放条件：
```

### 冻结原则

进入 Skill B 后，下列内容视为冻结输入；如需修改，必须回退 `wft02a`：

1. BA 节点清单
2. A1 / A2 / Bn 产品路径
3. IPO 类型
4. 节点边界
5. 复用 / 扩展 / 新增 决策
6. 主承接关系
7. 十链消费方式
8. Skill A 已确认的业务定义初步方案

### Step Start · 前置校验与路由

通过 `04-*.md` 的树画像 / 状态队列 / 节点索引定位本轮 BA 节点正文块，读取该节点状态、资产写回方案、资产建议反馈区、STR 缺口反馈建议和详细定义反馈状态，按以下规则进入后续步骤：

入口许可 BA 节点包括两类：

1. `04-*.md` 状态队列、待写回选择记录或本轮运行摘要中已有 `选择进入wft02b` 的 BA 节点或建议方案包。
2. `04-*.md` 或本轮运行摘要中已有 `wft02a` 活动锁释放 / 转交记录，且目标为该 BA 节点或建议方案包。

| 条件 | 路由 |
|------|------|
| 存在一个活动 BA 节点 | 只处理该活动 BA 节点的反馈、写回、详细定义、状态推进或回退，不得进入新 BA 节点 |
| 存在多个活动 BA 节点 | 停止执行，输出状态冲突说明，要求人类选择保留一个活动 BA 节点 |
| BA 节点 = `待资产写回` 且人类已回复 `选择进入wft02b` 或 `选择：<建议方案包ID>`，且不存在活动 BA 节点 | 执行 Step 1 → Step 2 → Step 3，并写入活动 BA 节点锁定关系 |
| 存在多个 BA 节点 = `待资产写回`，且不存在活动 BA 节点 | 输出 `等待选择进入wft02b`，当前待办 ID 为 AI 推荐的一个建议方案包 ID |
| 仅存在一个 BA 节点 = `待资产写回`，但没有 `选择进入wft02b` 或 `wft02a` 锁释放 / 转交记录，且不存在活动 BA 节点 | 输出 `等待选择进入wft02b`，当前待办 ID 为该建议方案包 ID |
| 仅存在一个入口许可 BA 节点，且不存在活动 BA 节点 | 执行 Step 1 → Step 2 → Step 3，并写入活动 BA 节点锁定关系 |
| BA 节点状态不是 `待资产写回` / `待STR反馈确认` / `待确认细节` / `确认中` | 输出不可进入说明后结束 |
| 存在资产建议反馈区且有 `[同意] / [修改] / [驳回]` | 执行 Step 4 |
| 存在资产建议反馈区且为空 | 输出 `等待建议方案反馈`，当前待办 ID 为一个资产写回建议 ID |
| 资产写回方案均为 `可写回` 且未写回 | 执行 Step 5 |
| STR 缺口反馈区有 `[同意] / [修改] / [驳回]` | 执行 Step 6 |
| BA 节点状态为 `待STR反馈确认` 且 STR 缺口反馈区为空 | 输出 `等待建议方案反馈`，当前待办 ID 为一个 STR 缺口反馈建议 ID |
| 存在 STR 缺口反馈建议且未处理 | 执行 Step 6 |
| 资产已写回且无待处理 STR 缺口反馈，`03-*.md` 未生成详细定义 | 执行 Step 7 |
| `03-*.md` 详细定义状态为 `待确认细节` 且反馈区为空 | 输出 `等待建议方案反馈`，当前待办 ID 为一个 `03` 详细定义 ID |
| `03-*.md` 详细定义反馈区有 `[同意] / [修改] / [驳回]` | 执行 Step 8 |
| 详细定义已确认，BA 节点尚未推进 | 执行 Step 8 |
| 首次进入且无资产写回方案 | 执行 Step 1 → Step 2 → Step 3 |

说明：

1. 重新运行 Skill B 时，若当前阶段已有方案且反馈区为空，AI 不得覆盖既有方案。
2. 若需要重做 BA 主结构，只能回退 `wft02a`。
3. 若需要修改 STR 场景边界、信息载体清单或场景指标，应停止本 Skill，并按结构问题回退到上游。
4. 资产建议反馈、STR 缺口反馈、详细定义反馈分别处理，不得混用反馈区。
5. 只要存在活动 BA 节点未完成，AI 不得响应其他 BA 节点的 `待资产写回` 入口，即使这些节点也满足进入条件。
6. 若多个 BA 节点同时满足 `待资产写回`，AI 不得自动选择其中一个进入本 Skill，必须先输出 `等待选择进入wft02b`。
7. 若 BA 节点已由 `wft02a` 完成 `选择进入wft02b` 的转交记录，下一次 `wft02b` 可将该记录视为本 Skill 的进入选择；AI 应先将选择记录转写到 `04-*.md` 状态队列或本轮运行摘要，再让该 BA 节点进入资产写回处理，不得再次输出同一个选择提示。
8. 当前 BA 节点完成并推进到 `待架构映射` 时，AI 必须记录活动锁释放。
9. 当前 BA 节点因结构问题回退 `wft02a` 时，AI 必须记录活动锁转交或释放，并只提示当前一个 BA 节点 ID。

读取要求：

5. `待架构映射`：本轮 STR→BA 已完成，可选入 `wft03`。

8. 主承接关系不成立

```text
[操作] 修改

[结构反馈]

- 需要 Skill A 重新判断的内容：

[其他说明]
-
```

### 回退上游的结构问题

1. STR 场景定义边界需要修改

3. 在 `02-*.md` 对应 STR 节点反馈区写入 `[操作] 修改` 记录
4. 输出需要回退的上游 Skill：`wft01a` 或 `wft01b`

## 五、运行结束输出（Step End）

Step End 的作用是把本轮沉淀结果翻译成人类下一次运行 `wft02b` 前的最小待办。AI 不只输出“完成 / 未完成”，还必须根据当前运行结果，为人类确定一个且仅一个当前动作。

推荐交互方式：

1. 人类优先通过对话回复。
2. AI 先将对话反馈或选择转写到对应反馈区、状态队列或人类决策字段，再继续处理。
3. 对话是反馈输入通道，文档中的反馈区 / 状态队列是权威留痕。
4. AI 不得只凭对话上下文推进状态或写回资产，必须先完成规范转写。

### 单 ID 最小待办原则

1. 每次 Step End 只提示人类处理一个 ID。
2. 该 ID 可以是一个资产写回建议 ID、一个 STR 缺口反馈建议 ID、一个 `03` 详细定义 ID、一个待进入 `wft02b` 的建议方案包 ID、一个需要回退 `wft02a` 的 BA 节点 ID。
3. AI 必须同时给出剩余待办 ID 数量，帮助人类知道当前队列规模。
4. 若上一次 Step End 已提示某个当前待办 ID，且该 ID 尚未被人类确认完成、明确切换或变为不可操作，下次运行必须继续提示同一个 ID。
5. 人类对当前 ID 的回复可以是 `同意`、`修改：……`、`驳回：……`，或 `选择：……`。
6. 只有当前 ID 已完成、被人类明确切换，或因状态变化不再可操作时，AI 才能按阻断优先级、状态时间和推荐顺序自动选择下一个当前 ID，并说明切换原因。
7. 当前 ID 处理完成后，下一次运行 `wft02b` 再提示下一个 ID。
8. AI 可以附带其他待办摘要，但不得要求人类同时审阅或选择多个 ID。

### 当前待办 ID 连续性规则

1. Step End 输出的 `当前待办ID` 默认形成“待办锁定”。
2. AI 应在 `04-*.md` 对应资产建议反馈区、STR 缺口反馈区、`03-*.md` 详细定义反馈区、状态队列或本轮运行摘要中保留该 ID 的待办状态。
3. 若下一次运行时该 ID 仍处于待反馈、待确认、待补充或待选择状态，AI 必须继续提示该 ID，不得因为出现新的待办而切换。
4. 若人类回复 `换一个 / 跳过 / 选择：<另一个ID>`，AI 可以切换当前待办 ID，但必须先记录切换原因。
5. 若当前 ID 已经完成，AI 才从剩余待办中选择下一个 ID。
6. 若当前 ID 因上游状态变化不再可操作，AI 必须说明原 ID 不再可操作的原因，再选择新的当前待办 ID。

### Step End 待办类型

| 待办类型 | 何时出现 | 当前 ID | 人类只需做什么 | 下次运行 |
|---------|----------|---------|----------------|----------|
| `反馈建议方案` | 存在资产写回建议、STR 缺口反馈建议或 `03` 详细定义建议，等待人类确认 | 一个建议方案 ID | 对该 ID 回复 `同意 / 修改 / 驳回` | `wft02b` |
| `选择进入wft02b` | 存在多个已确认 BA 建议方案包、待资产写回包或待细化包，需要确定本轮 Skill B 处理对象 | 一个建议方案包 ID | 对该 ID 回复 `选择进入wft02b`，或指定另一个包 ID | `wft02b` |
| `选择进入wft02a` | 发现 BA 主结构、产品路径、业务定义边界或十链消费方式需要重判 | 一个 BA 节点 ID | 对该 ID 回复 `选择进入wft02a`，或补充结构反馈 | `wft02a` |
| `已提示回退上游` | 已确认 STR 缺口反馈，或发现 STR 场景/文档归属结构问题，不应继续在 `wft02b` 内推进 | 一个 STR 缺口反馈 ID 或 STR 节点 ID | 对该 ID 回复 `同意 / 修改 / 驳回`，或按提示运行上游 Skill | `wft01a` / `wft01b` |
| `当前节点完成` | 当前 BA 节点的资产写回、STR 缺口处理和 `03` 详细定义确认均完成 | 下一步建议 ID，或本轮完成 BA 节点 ID | 对该 ID 回复 `选择进入wft02b` / `选择进入wft02a`，或无需回复 | `wft02b` / `wft02a` |

### 输出结果

本 Skill 结束时，只允许输出以下五类结果之一，且每类结果必须带一个当前待办 ID：

| 输出结果 | 对应待办类型 | 当前 ID | 人类下一步 |
|---------|--------------|---------|------------|
| `等待建议方案反馈` | `反馈建议方案` | 一个资产写回建议、STR 缺口反馈建议或 `03` 详细定义建议 ID | 对该 ID 回复 `同意 / 修改 / 驳回` |
| `等待选择进入wft02b` | `选择进入wft02b` | 一个建议方案包 ID | 对该 ID 回复 `选择进入wft02b`，或指定另一个包 ID |
| `等待选择进入wft02a` | `选择进入wft02a` | 一个 BA 节点 ID | 对该 ID 回复 `选择进入wft02a`，或补充结构反馈 |
| `已提示回退上游` | `反馈建议方案` | 一个 STR 节点或 STR 缺口反馈 ID | 对该 ID 回复 `同意 / 修改 / 驳回`，或按提示运行上游 |
| `当前节点完成` | `选择进入wft02b` / `选择进入wft02a` / 完成记录 | 一个下一步建议 ID，或本轮完成 BA 节点 ID | 对该 ID 回复 `选择进入wft02b` / `选择进入wft02a`，或无需回复 |

### 最小待办输出格式

AI 在 Step End 应按以下格式输出：

```text
当前结果：等待建议方案反馈 / 等待选择进入wft02b / 等待选择进入wft02a / 已提示回退上游 / 当前节点完成
当前待办ID：
是否延续上次待办ID：
剩余待办ID数量：
待办类型：
为什么先处理这个ID：
人类只需回复：
回复会写入：
下次运行：
其他待办摘要：
```

要求：

1. `当前待办ID` 必须是单个 ID，不得写成多个 ID 列表。
2. `是否延续上次待办ID` 必须写 `是 / 否`；若为 `否`，必须说明当前 ID 已完成、被人类切换或不再可操作的原因。
3. `剩余待办ID数量` 必须是数字；若无法精确统计，应写 `至少 N 个` 并说明统计口径。
4. `人类只需回复` 必须是一句话，且只包含当前 ID 的动作。
5. `其他待办摘要` 只能给数量和类型，不得要求人类同时处理。
6. 若人类直接在对话中回复，AI 应先转写到 `04-*.md` 对应资产建议反馈区、STR 缺口反馈区、`03-*.md` 详细定义反馈区或状态队列，再执行下一步。

### 结果一：等待建议方案反馈

适用：本轮存在一个需要人类确认的资产写回建议、STR 缺口反馈建议或 `03` 详细定义建议。

当前待办 ID：

1. 优先选择阻断当前写回推进的建议 ID。
2. 若多个建议同级阻断，优先顺序为：STR 缺口反馈建议、资产写回建议、`03` 详细定义建议。
3. 若同类中存在多个 ID，选择最早进入等待反馈状态且无反馈的一个。

人类只需做什么：

1. 对当前建议 ID 回复 `同意`、`修改：……` 或 `驳回：……`。
2. AI 收到回复后，先转写到该 ID 对应反馈区，再在下一次 `wft02b` 中处理。

回复会写入：

1. 资产写回建议：`04-*.md` 对应 BA 节点的 `### 资产建议反馈区`
2. STR 缺口反馈建议：`04-*.md` 对应 BA 节点的 `### STR缺口反馈区`，必要时同步 `02-*.md` 对应 STR 节点反馈区
3. `03` 详细定义建议：`03-*.md` 对应详细定义块的 `### 反馈区`

### 结果二：等待选择进入wft02b

适用：存在多个已确认 BA 建议方案包、待资产写回包或待细化包，需要确定下一次 `wft02b` 处理对象。

口径说明：

1. `待资产写回` 是 `wft02a` 输出的方案确认状态，表示该 BA 节点具备进入 `wft02b` 的业务前提，但还不等于已经完成交接。
2. `选择进入wft02b` 有两种含义：从 `wft02a` 过来时，表示该 BA 建议方案包已被人类选择并转交给 `wft02b`；在 `wft02b` 内部存在多个入口许可 BA 节点时，表示本轮处理对象的选择动作。
3. 只有不存在活动 BA 节点时，AI 才能提示 `等待选择进入wft02b`。
4. 若已经存在活动 BA 节点，即使其他 BA 节点也是 `待资产写回`，也不得提示选择新 BA 节点。
5. 若多个 BA 节点同时具备入口许可，AI 必须先输出本结果，不能自动进入 Step 1。
6. 若只有一个 BA 节点具备入口许可且不存在活动 BA 节点，AI 可以直接进入 Step 1。
7. 若只有一个 BA 节点处于 `待资产写回`，但尚未记录 `选择进入wft02b` 或 `wft02a` 锁释放 / 转交，AI 应输出本结果，请人类完成交接选择，不得自动进入 Step 1。

当前待办 ID：

1. 优先选择状态最靠近阻断点的建议方案包 ID。
2. 若多个包同级，选择最早进入当前状态的一个。

人类只需做什么：

1. 对当前建议方案包 ID 回复 `选择进入wft02b`，或回复 `选择：<另一个包ID>`。
2. AI 收到回复后，先转写到 `04-*.md` 状态队列或待处理选择记录，再在下一次 `wft02b` 中处理。

### 结果三：等待选择进入wft02a

适用：发现 BA 主结构、产品路径、IPO 类型、业务定义边界、十链消费方式或主承接关系需要重判。

当前待办 ID：

1. 必须是需要回退的 BA 节点 ID。
2. 不得同时要求人类审阅资产写回建议或详细定义。

人类只需做什么：

1. 对当前 BA 节点 ID 回复 `选择进入wft02a`，或回复 `修改：……` 补充结构反馈。
2. AI 收到回复后，先转写到该 BA 节点反馈区，并将节点状态回退为 `迭代中`，再提示重新运行 `wft02a`。
3. AI 必须记录活动 BA 锁转交或释放，不得继续处理其他 BA 节点。

### 结果四：已提示回退上游

适用：已确认 STR 缺口反馈，或发现 STR 场景 / 文档归属结构问题，不应继续在 `wft02b` 内推进。

当前待办 ID：

1. 优先使用 STR 缺口反馈 ID。
2. 若问题来自 STR 场景 / 文档归属结构，使用对应 STR 节点 ID。

人类只需做什么：

1. 对当前 ID 回复 `同意`、`修改：……` 或 `驳回：……`，或按 AI 指定的上游 Skill 处理该 ID。
2. AI 不得要求人类同时处理多个上游回退 ID。

### 结果五：当前节点完成

适用：当前 BA 节点的资产写回、STR 缺口处理和 `03` 详细定义确认均完成。

当前待办 ID：

1. 若还有待进入 `wft02b` 的包，提示一个建议方案包 ID，并输出 `等待选择进入wft02b`。
2. 若没有待进入 `wft02b` 的包，但还有待 `wft02a` 处理的 STR 节点或 BA 结构反馈，提示一个对应 ID，并输出 `等待选择进入wft02a`。
3. 若确实没有 `wft02a/wft02b` 待办，当前 ID 可为本轮完成 BA 节点 ID，剩余待办 ID 数量为 `0`，并说明该节点可进入 `wft03`。

锁定说明：

1. 当前 BA 节点推进到 `待架构映射` 后，AI 必须先记录活动 BA 锁释放，再提示下一个待办。
2. 活动锁释放前，`wft02b` 不得进入新的 BA 节点。
3. 释放记录至少包含：活动 BA 节点 ID、来源 STR 节点 ID、最终状态、释放时间或轮次、下一步建议。
