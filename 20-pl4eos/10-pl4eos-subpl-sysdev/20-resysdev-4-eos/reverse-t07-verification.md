# 逆向推导验证
**Skill ID**: reverse-t07 · **Step 7: Verification**

> **概述**：验证完整逆向推导后各层之间的追溯链，输出验证报告。需验证双链各自的独立追溯链路完整性，以及跨链接口的正确性。
>
> **输入 → 输出**：全部设计文档 + 决策记录 → 追溯矩阵 + 验证报告
>
> **适用场景**：逆向工程第7步。验证 PA→SysReq→BA→SR→OR 全链路追溯完整性。
>
> **版本**：v1.0 | **修订**：2026-05-28

---

## 一、设计角色

本步在设计链中承担的角色：
- **输入**：第1~6步产出的全部设计文档，第1~6步的决策记录
- **产出**：追溯矩阵（`08-*.md`），验证报告（`09-*.md`）

### 与三条工程化设计原则的关系

| 原则 | 在逆向工程中的体现（本步） |
|------|--------------------------|
| **详概分离** | 本步验证两个层面：架构层映射验证（概）和详细层映射验证（详）。追溯矩阵为全量端到端映射（详），验证报告为问题汇总和评估（概） |
| **文档结对** | 验证所有跨层文档结对的正确性。按双链分别验证各结对关系，并增加跨链结对验证——平台链 PA 组件与业务链 SysReq 之间的接口对应关系 |
| **基于资产** | 各步的决策记录是本步的核心资产。逆向推导特有的"制品差异"是验证重点 |

## 三、操作流程

七阶段：全链路合并验证 → 1:1 约束验证 → N:1 支持验证 → 双向追溯 → 逆向一致性验证 → 生成追溯矩阵 → 生成验证报告。

核心驱动逻辑：从最底层（PA）开始，逐层向上验证逆向推导的自洽性。

操作流程概览：

```
[阶段一] 全链路合并验证     [阶段二] 1:1约束验证      [阶段三] N:1支持验证
[阶段四] 双向追溯          [阶段五] 逆向一致性       [阶段六] 生成追溯矩阵
[阶段七] 生成验证报告
```
