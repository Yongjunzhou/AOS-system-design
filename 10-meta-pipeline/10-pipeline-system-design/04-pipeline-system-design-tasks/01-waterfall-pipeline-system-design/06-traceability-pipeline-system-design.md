# task-06：双向追溯验证

**对应场景**：01-waterfall · 第6步
**对应 AI 模板**：`03-pipeline-system-design-ai-support/01-waterfall-pipeline-system-design/workflow-prompts.md` 第6步
**对应设计指南**：[瀑布式系统设计指南 第6步](../../02-pipeline-system-design-guidelines/01-waterfall-pipeline-system-design-guide.md#第6步双向追溯验证)

---

## 目标

验证从 OR 到 PA 的完整追溯链路，输出追溯矩阵和验证报告。

## 上下文继承

```
从哪里来：
  - 第1~5步产出的全部设计文档
  - 第1~5步的决策记录
  - 涉及规范：设计准则 §3.2（三条核心规则）、§5（验收标准）、§6（检查清单）

核心验证任务：
  1. 分析活动合并验证：汇总第1~5步中所有分析活动结果
  2. 1:1 分配约束验证（全部 4 层）
  3. N:1 承接合理性验证
  4. 双向追溯验证（正向 + 反向）
  5. 符合性分析完整率验证
  6. 生成追溯矩阵和验证报告
```

## 输入

| 输入 | 说明 |
|------|------|
| 全部设计文档 | task-01 ~ task-05 的全部产出 |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 分析活动：合并验证 | AI | 汇总第1~5步分析活动：需求说明符合性分析（覆盖度/一致性/追溯）+ 需求性能符合性分析（性能继承/可实现性/权衡决策追溯） |
| 2 | 1:1 分配约束验证 | AI | 逐层检查：OR→SR→BA→SysReq→PA |
| 3 | N:1 承接支持验证 | AI | 检查每个节点的承接数量（2-5条合理）和合理性 |
| 4 | 双向追溯验证 | AI | 正向 OR→PA 和反向 PA→OR |
| 5 | 符合性分析完整率验证 | AI | 所有方案节点（SR/BA/SysReq/PA）都有符合性分析 |
| 6 | 生成追溯矩阵 | AI | 端到端映射关系表 |
| 7 | 生成验证报告 | AI | 含问题清单和建议措施，标注回溯修复的影响范围 |
| 8 | 结果审核 | 人类 | 确认验证结论，批准问题修复方案 |

## 人类审核要点

- [ ] 验证报告中的未通过项和待确认项
- [ ] 追溯链路是否存在断点
- [ ] 符合性分析完整率是否 100%
- [ ] 性能指标跨层是否一致（产品架构不低于 SysReq 要求）
- [ ] 回溯修复方案的影响范围是否评估完整

## 质量门禁

- [ ] 分析活动合并验证完成
- [ ] 1:1 约束通过率 100%
- [ ] N:1 承接合理性通过（2-5 条/节点）
- [ ] 双向追溯完整率 100%（正向 + 反向）
- [ ] 符合性分析完整率 100%（所有方案节点都有符合性分析）
- [ ] 映射覆盖率 100%（无遗漏、无冗余）
- [ ] 性能指标跨层一致
- [ ] 超链接有效率 100%
- [ ] 运行 rule-conformance-review 进行符合性审查，按审查修复流程处理问题
