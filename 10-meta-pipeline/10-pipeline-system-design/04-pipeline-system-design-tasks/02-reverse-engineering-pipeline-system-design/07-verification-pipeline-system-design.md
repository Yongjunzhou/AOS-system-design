# task-07：验证

**对应场景**：02-reverse-engineering · 第7步
**对应 AI 模板**：`03-pipeline-system-design-ai-support/02-reverse-engineering-pipeline-system-design/workflow-prompts.md` 第7步
**对应设计指南**：[逆向工程系统设计指南 第7步](../../02-pipeline-system-design-guidelines/02-reverse-engineering-pipeline-system-design-guide.md#第7步完整性和一致性验证)

---

## 目标

验证完整逆向推导后各层之间的追溯链，输出验证报告。

## 上下文继承

```
从哪里来：
  - 第1~6步产出的全部设计文档
  - 第1~6步的决策记录

核心验证任务：
  1. 1:1 分配约束验证（全部 4 层）
  2. N:1 承接合理性和逆向合理性验证
  3. 双向追溯验证（强调自底向上方向）
  4. 符合性分析完整率验证
  5. 生成验证报告
```

---

## 输入

| 输入 | 说明 |
|------|------|
| 全部设计文档（OR → SR → BA → SysReq → PA） | task-01 ~ task-06 的全部产出 |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 分析活动：全链路合并验证 | AI | 需求说明符合性分析汇总（逐层验证映射覆盖率 100%）+ 需求性能符合性分析汇总（性能指标逐层继承验证）+ 指标追溯（端到端性能指标可逐层追溯） |
| 2 | 1:1 分配约束验证 | AI | 逐层检查每条详细定义末级的映射 |
| 3 | N:1 承接支持验证 | AI | 检查各层节点的承接数量合理性 |
| 4 | 双向追溯验证 | AI | OR → PA 和 PA → OR |
| 5 | 反向推导一致性验证 | AI | 逆向推导的上下游内容是否一致 |
| 6 | 生成追溯矩阵 | AI | 端到端映射关系表 |
| 7 | 生成验证报告 | AI | 含问题清单和建议 |

## 人类审核要点

- [ ] 验证报告中的未通过项
- [ ] 逆向推导后的设计是否自洽
- [ ] 是否存在"推导矛盾"——上层推导的结果和下层推导的结果不一致
- [ ] 1:1 约束未通过项是否能接受或需要回溯修复
- [ ] 逆向推导的"空壳组件"是否有合理解释
- [ ] 符合性分析完整率是否达标

## 质量门禁

- [ ] 全链路合并验证已执行
- [ ] 1:1 约束通过率 100%
- [ ] N:1 承接合理性评估完成
- [ ] 双向追溯验证通过（正向 + 反向）
- [ ] 反向推导一致性通过
- [ ] 符合性分析完整率达标
- [ ] 验证报告已生成
- [ ] **运行 rule-conformance-review 进行完整符合性审查**（这是建立基线前的关键门禁）
- [ ] 按审查修复流程处理问题
