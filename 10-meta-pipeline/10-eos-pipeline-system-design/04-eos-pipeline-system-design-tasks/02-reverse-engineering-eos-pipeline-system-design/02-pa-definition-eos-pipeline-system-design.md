# task-02：产品架构定义

## 目标

根据现有产物的目录结构和文件组织方式，定义设计线的产品架构（PA），包括树形结构、组件定义和映射关系。

## 上下文继承

```
从哪里来：
  - 第1步产出：产物总览清单（产物分类 + 架构层级映射）
  - 第1步决策记录

去哪里：
  - 本步产出 → 第3步（SysReq 反向推导）的 PA 组件输入
  - 映射关系表骨架 → 第3步完成后填充 SysReq 映射
```

---

## 输入

| 输入 | 说明 |
|------|------|
| 产物总览清单 | task-01 产出 |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 识别产品类型 | AI | 确定 PA 分类体系（方法论/工具产品 → 规范与准则/设计指南/AI辅助文档/任务定义） |
| 2 | 按产物分组定义 PA 组件 | AI | 每组产物对应一个 PA 架构末级节点；遵循 §2.4 产品架构分解原则（综合信息模型内聚性、NFR Profile 相容性、信息处理模式一致性三个因素）判断组件合并/分离 |
| 3 | 建立 PA 树形结构 | AI | 按分类体系组织树形层级 |
| 4 | 编写组件定义和符合性分析 | AI | 每个组件包含定义、组件类型、承接关系 |
| 5 | 分析活动：PA 架构评估 | AI | 需求说明符合性分析（组件粒度、职责边界、所有产物是否已覆盖）+ 需求性能符合性分析（从产物实现特征反推组件性能指标） |
| 6 | 结果审核 | 人类 | 确认 PA 组件划分的合理性 |

## 人类审核要点

- [ ] 组件粒度是否合理——每个组件的职责边界是否清晰
- [ ] 是否有遗漏产物未纳入 PA 组件
- [ ] 产品类型分类体系选择是否正确
- [ ] 树形结构的层级深度和广度是否合理
- [ ] 每个组件的符合性分析是否充分
- [ ] 详细定义中的质量指标是否完整

## 质量门禁

- [ ] PA 架构定义完成（树形结构 + 组件定义）
- [ ] 所有产物已分配到对应的 PA 组件
- [ ] 详细定义已同步编写
- [ ] 映射关系表骨架已预留
- [ ] 组件粒度符合"可独立部署和审查"标准
- [ ] PA 组件划分遵循 §2.4 产品架构分解原则（三因素决策）
- [ ] 运行 rule-conformance-review 进行符合性审查
- [ ] 按审查修复流程处理问题
