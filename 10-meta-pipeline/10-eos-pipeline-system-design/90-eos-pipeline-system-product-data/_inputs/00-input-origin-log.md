# EOS 流水线原始输入来源记录
**EOS Pipeline Original Input Origin Log**

**用途**：登记所有原始输入材料及其处理状态。原始材料文件放入 `_inputs/` 目录，AI 自动登记到此表。

---

## 记录表

| 记录ID | 内容摘要 | 来源渠道 | 接收日期 | 所属域 | AI状态 | 建议状态 | 对应文档 |
|--------|---------|---------|---------|-------|--------|---------|---------|
| | | | | | | | |

**字段说明**：

| 字段 | 维护者 | 说明 |
|------|--------|------|
| AI状态 | AI填写 | "未处理" / "已处理" |
| 建议状态 | **人类填写** | "待处理" / "暂搁置" / "已终止" |

**操作流程**：

1. 人类将原始材料放入 `_inputs/` 目录，告知 AI 登记
2. AI 扫描 `_inputs/`，将新文件追加到此表（AI状态="未处理"）
3. 人类浏览日志，将决定要处理的记录标记为"待处理"
4. AI 启动时筛选"建议状态=待处理"的记录，加载对应文档执行
5. 执行完成后，AI 更新 AI状态="已处理"
