# task-05：系统需求影响分析

**对应场景**：03-agile · 第5步
**对应 AI 模板**：`03-eos-pipeline-system-design-ai-support/03-agile-eos-pipeline-system-design/workflow-prompts.md` 第5步

---

## 目标

分析新需求对系统需求的影响，输出 SysReq 增量定义（同时处理功能和非功能分支）。

## 上下文继承

```
从哪里来：
  - task-04 产出：BA 增量定义 + BA → SysReq 骨架映射
  - 参考基线：现有 SysReq 文档（功能 0-9 级 + 非功能 0-6 级）

去哪里：
  - 本步产出 → task-06（PA 影响分析）的 SysReq 增量定义
  - 同步产出 → SysReq → PA 骨架映射（为 task-06 提供输入）

并行说明：
  - 功能分支（task-03 → task-04 → task-05）与非功能分支（task-03 NFR 部分 → task-05 非功能部分）在此处合并处理
```

---

## 输入

| 输入 | 说明 |
|------|------|
| BA 增量定义 | task-04 产出 |
| 现有 SysReq 文档（基线） | 基线中的 SysReq 层 |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 分析活动：SysReq 影响分析 | AI | 需求说明符合性分析（功能和非功能分支分别分析，BA 增量是否能映射到 SysReq）+ 需求性能符合性分析（性能指标是否从 BA 正确继承） |
| 2 | SysReq 功能增量定义 | AI | 0-9 级架构增量 + 9 级场景活动 |
| 3 | SysReq 非功能增量定义 | AI | 0-6 级架构增量 |
| 4 | 同步设计 PA 骨架 | AI | SysReq → PA 骨架映射 |
| 5 | 结果审核 | 人类 | 确认 SysReq 更新 |

## 人类审核要点

- [ ] 功能分支增量与 BA 一致性
- [ ] 非功能指标量化是否合理
- [ ] 功能/非功能分支分离是否正确
- [ ] PA 骨架是否已同步建立

## 质量门禁

- [ ] SysReq 增量定义完成（功能 + 非功能）
- [ ] 功能架构 0-9 级增量完成
- [ ] 非功能架构 0-6 级增量完成
- [ ] SysReq → PA 骨架映射已更新
- [ ] 运行 rule-conformance-review 进行符合性审查
- [ ] 按审查修复流程处理问题
