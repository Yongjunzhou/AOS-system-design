# task-06：产品架构影响分析

**对应场景**：03-agile · 第6步
**对应 AI 模板**：`03-eos-pipeline-system-design-ai-support/03-agile-eos-pipeline-system-design/workflow-prompts.md` 第6步

---

## 目标

分析新需求对产品架构的影响，输出 PA 增量定义。

## 上下文继承

```
从哪里来：
  - task-05 产出：SysReq 增量定义 + SysReq → PA 骨架映射
  - 参考基线：现有 PA 文档（架构定义 + 详细定义）
  - 涉及规范：设计准则 §2.4（产品架构分解原则）、产品架构规范 §2-§3

去哪里：
  - 本步产出 → task-07（验证）的 PA 增量定义
  - SysReq → PA 映射关系表已更新
```

---

## 输入

| 输入 | 说明 |
|------|------|
| SysReq 增量定义 | task-05 产出 |
| 现有 PA 文档（基线） | 基线中的 PA 层 |

## 操作流程

| # | 操作 | 执行者 | 说明 |
|---|------|--------|------|
| 1 | 分析活动：PA 影响分析 | AI | 需求说明符合性分析（SysReq 增量是否能由现有 PA 组件承接）+ 需求性能符合性分析（修改是否影响性能指标范围） |
| 2 | PA 增量条目定义（含架构分解依据） | AI | 遵循 §2.4 产品架构分解原则（综合信息模型内聚性、NFR Profile 相容性、信息处理模式一致性三个因素）判断构件合并/分离，按产品类型选择分类体系 |
| 3 | 补充组件详细定义 | AI | 因产品类型而异（软件/方法论） |
| 4 | 更新映射关系表 | AI | SysReq → PA 映射 |
| 5 | 结果审核 | 人类 | 确认 PA 更新，涉及技术决策 |

## 人类审核要点

- [ ] 组件划分是否合理
- [ ] 技术决策是否合理
- [ ] 新增组件的 PA 分类体系是否正确
- [ ] 详细定义是否同步更新
- [ ] 映射关系是否正确—SysReq → PA 对应完整

## 质量门禁

- [ ] 分析活动：PA 影响分析已执行
- [ ] PA 增量定义完成（架构定义 + 详细定义）
- [ ] 组件粒度符合"可独立部署和审查"标准
- [ ] PA 组件划分遵循 §2.4 产品架构分解原则（三因素决策）
- [ ] 映射关系表已更新
- [ ] 运行 rule-conformance-review 进行符合性审查
- [ ] 按审查修复流程处理问题
