# Skill: 重复检测

## 目的
检测新增需求与已有需求之间的重复，并进行分类和建议。

## 输入
- 新增需求（规范化和分解后）
- 已有需求列表

## 输出
- 重复列表
- 重复分类
- 处理建议

## 重复类型

### 1. 完全重复
**定义**：新需求与已有需求完全相同

**例子**：
- 已有需求：系统应支持用户登录
- 新增需求：系统应支持用户登录

**处理建议**：删除新增需求，或标记为"已存在"

### 2. 部分重复
**定义**：新需求与已有需求部分相同，新需求包含已有需求的功能

**例子**：
- 已有需求：系统应支持用户登录
- 新增需求：系统应支持用户登录和注册

**处理建议**：合并为"系统应支持用户登录和注册"

### 3. 相似重复
**定义**：新需求与已有需求相似，但表述不同

**例子**：
- 已有需求：用户可以查看订单列表
- 新增需求：系统应显示用户的所有订单

**处理建议**：确认是否为同一需求，如是则合并

## 重复检测规则

### 1. 语义相似度检测
- [ ] 新需求与已有需求的核心功能是否相同
- [ ] 新需求与已有需求的输入输出是否相同
- [ ] 新需求与已有需求的验收标准是否相同

### 2. 表述相似度检测
- [ ] 新需求与已有需求是否使用了相同的关键词
- [ ] 新需求与已有需求是否描述了相同的业务流程

### 3. 范围重叠检测
- [ ] 新需求的范围是否与已有需求的范围重叠
- [ ] 新需求是否是已有需求的子集或超集
