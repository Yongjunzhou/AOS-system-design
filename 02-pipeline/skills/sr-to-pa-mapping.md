# Skill: 系统需求→产品架构映射

## 目的
将系统需求映射到产品架构的组件，确保每条系统需求有唯一的产品实现点。

## 输入
- 系统需求
- 产品架构

## 输出
- 映射关系（SysReq → PA Component）
- 映射验证结果
- 无法映射的需求和建议

## 映射规则

### 1. 多对一映射
- **原则**：N 个产品架构组件 → 1 条系统需求
- **约束**：1 条系统需求 → 1 个产品架构节点（规定层级）
- **验证**：每条系统需求必须有且仅有一个对应的产品架构承接点

### 2. 语义覆盖验证
- 产品架构组件是否完全实现系统需求
- 产品架构组件的功能是否满足系统需求的所有要求
- 是否存在系统需求功能未被产品架构实现

### 3. 无法映射的处理
- 如果系统需求无法映射到任何现有产品架构组件：
  - 选项1：对系统需求进行再分解
  - 选项2：创建新的产品架构组件
  - 选项3：迭代产品架构的设计
- 决策应由架构师确认

## 映射模板

```markdown
### 系统需求→产品架构映射

| SysReq ID | 系统需求 | PA Comp ID | 产品架构组件 | 映射状态 |
|----------|--------|-----------|------------|--------|
| SysReq-001 | 订单创建 | PA-COMP-001, PA-COMP-002 | OrderService, OrderDB | ✓ |
| SysReq-003 | 支付处理 | PA-COMP-003 | PaymentGateway | ✓ |
| SysReq-005 | [需求] | [无映射] | N/A | ✗ 待处理 |

### 无法映射的需求处理

| SysReq ID | 需求 | 原因 | 建议处理方式 |
|----------|------|------|-----------|
| SysReq-005 | [需求] | [原因] | [处理建议] |
```

## 使用场景
- 新增系统需求的产品架构设计
- 产品架构的验证
- 准则4的实现（系统需求与产品架构同步设计）
