# 敏捷式系统设计：产品数据范例
**Agile System Design: Product Data Example**

**用途**：展示一个敏捷式增量系统设计过程的输入和输出范例，AI 参照此范例生成自己的增量设计文档。

**范例项目**：运营体系开发之系统设计流水线（System Design Pipeline）

**基线说明**：本范例基于瀑布式场景（`01-waterfall/product-example.md`）的完整设计输出作为基线。本次增量在基线基础上，新增"设计基线管理"能力。

## 一、第1步产出范例

### 1.1 新需求收集与分类

```
OR-006: 设计基线管理
├─ 需求描述：系统应支持在系统设计过程中创建基线（Baseline），
│           冻结特定版本的设计文档，支持基线间的差异对比和回溯
├─ 原始碎片清单：
│  ├─ "每次迭代完成后要能打标签，方便回溯到那个时间点的设计状态" — 流水线开发者 2026-05-12
│  ├─ "改了一轮需求后，想看看和上一版比改了哪些内容" — AOS架构师 2026-05-13
│  └─ "基线一旦创建就不能再修改，确保评审过的设计不会被意外变更" — AOS开发者 2026-05-14
├─ 包含的性能指标：基线创建 < 2s，差异对比 < 5s
├─ 需求类型：功能需求
├─ 变更类型：新增
├─ 与现有需求的关系：与 OR-004（需求映射与追溯）相关，基线管理可复用现有映射追溯机制
└─ 依赖/冲突关系：依赖 OR-004（需要映射追溯机制作为基线对比的基础）
```

## 二、第2步产出范例

### 2.1 需求规范化与末级分解

```
OR-006: 设计基线管理
├─ 规范化描述：系统提供设计基线的创建、冻结、对比和追溯能力，
│              基线标记特定时间点的设计文档集合，冻结后不可修改，
│              支持对比当前设计与任一基线版本的差异
├─ 类型：功能需求
├─ 优先级：中 ⚠️ 待确认
├─ 末级分解：
│  ├─ OR-006.1: 基线创建与冻结 — 设计者选择当前所有设计文档创建基线，
│  │             基线包含完整的设计快照（SR/BA/SysReq/PA），创建后自动冻结
│  ├─ OR-006.2: 基线差异对比 — 对比当前设计与指定基线版本的差异，
│  │             按层展示新增/修改/删除的内容变化
│  └─ OR-006.3: 基线历史追溯 — 查看基线列表及各基线的创建信息，
│               支持回退到指定基线版本查看历史设计状态
└─ 验收标准：
   ├─ 基线创建后所有设计文档被快照，不可修改
   ├─ 差异对比在 5s 内完成
   ├─ 基线列表按创建时间降序排列
   └─ 历史追溯可完整查看基线时刻的设计文档
```

```
~ BA-012: 验证报告生成（5级·架构末级节点）
├─ 更新说明：验证报告增加"关联基线"标签，支持按基线版本筛选行

├─ 节点类型：配置
├─ 定义：创建当前设计文档集合的基线快照，冻结后不可修改
├─ Input：当前所有设计文档（SR/BA/SysReq/PA）、基线名称、基线描述
├─ Process：
│  1. 设计者输入基线名称和描述
│  2. 系统对当前所有设计文档执行快照（含映射关系）
│  3. 标记快照版本号，写入基线元数据
│  4. 将基线状态置为"已冻结"，所有快照文档设为只读
├─ Output：基线快照（含基线ID、名称、创建时间、版本号、文档快照集合）

├─ 性能指标：基线创建 < 2s
├─ 设计指导：系统应提供基线创建表单、全量快照引擎、状态置为只读

+ BA-016: 基线差异对比（5级·架构末级节点）
├─ 节点类型：配置
├─ 定义：对比当前设计与指定基线版本的差异，按层展示
├─ Input：当前设计文档集、目标基线快照、对比范围（按层筛选）
├─ Process：
│  1. 加载当前设计文档和目标基线快照
│  2. 逐层逐节点对比差异（新增/修改/删除）
│  3. 按变更类型分类展示
│  4. 生成差异报告
├─ Output：差异报告（变更明细、新增/修改/删除统计）
├─ 承接的SR详细定义末级：SR-F-005.2
├─ 符合性分析：通过逐层对比和分类展示满足差异对比需求
├─ 性能指标：差异对比 < 5s
├─ 设计指导：系统应提供差异对比引擎、逐层展开/折叠、变更类型筛选

+ BA-017: 基线历史追溯（5级·架构末级节点）
├─ 节点类型：配置
├─ 定义：查看基线列表，按基线回溯查看历史设计文档

├─ Process：
│  1. 加载基线列表（按创建时间降序）
│  2. 选择目标基线
│  3. 按只读模式加载基线时刻的设计文档快照
│  4. 支持浏览各层文档内容
├─ Output：基线历史视图、基线时刻的设计文档只读展示

├─ 符合性分析：通过基线列表和只读快照加载满足历史追溯需求
├─ 设计指导：系统应提供基线列表、基线版本时间线、只读文档展示
```
