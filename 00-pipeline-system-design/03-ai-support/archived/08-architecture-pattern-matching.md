# 架构模式匹配技能
**Skill: Architecture Pattern Matching**

**编号**：08
**所属步骤**：第 3 步 — 系统需求功能部分架构定义
**引用准则**：系统设计准则 v2.3 — 系统需求架构层级结构（0-9 级）
**引用指南**：瀑布式设计指南 v2.0 第 3 步
**状态**：暂不启用 — 依赖架构模式库（07-shared-assets/patterns/），模式库建立后启用

---

## 用途

将系统需求与架构模式库进行匹配，找到相似的架构模式作为系统需求架构设计的参考。

## 输入

- 系统需求（第 3 步待处理的 IPO 清单）
- 系统特性（规模、并发、性能等）
- 架构模式库（07-shared-assets/patterns/）

## 处理规则

### 匹配维度
1. **系统规模匹配**：功能模块数量、用户规模、数据量级
2. **并发需求匹配**：并发用户数、请求频率
3. **性能需求匹配**：响应时间、吞吐量
4. **可扩展性匹配**：业务增长预期、扩展方式

### 评分标准

| 评分 | 等级 | 说明 | 评分判断标准 |
|------|------|------|------------|
| 90-100% | 高匹配 | 模式直接可用 | **4 个维度都 > 85%** 或 **3 个维度 100% + 1 个维度 > 80%**：系统规模、并发、性能、扩展性都与模式非常接近，架构方案可以直接采用 |
| 70-89% | 中匹配 | 模式需要调整 | **至少 3 个维度 > 70%** 或 **2 个维度 100% + 2 个维度 > 60%**：大部分维度与模式匹配，需要针对 1-2 个维度调整架构 |
| 50-69% | 低匹配 | 模式需要大幅调整 | **2 个维度在 50-70% 范围，另 2 个维度 < 60%** 或 **有 1-2 个维度完全不匹配**：只有部分架构选择相关，需要大幅调整设计 |
| < 50% | 不匹配 | 不建议参考 | **3+ 个维度 < 50%** 或 **有多个关键维度完全不匹配**：架构模式基本不适用，建议寻找其他参考或从头设计 |

### 评分具体判断方法

**系统规模匹配** - 评分从 0% 到 100%：
- **100%**：需求的功能模块数、用户规模、数据量完全与模式一致
- **80%**：需求与模式在规模数量级相同，但具体数字差异 20-30%（例如需要 1000 用户，模式设计 1500 用户）
- **60%**：需求与模式在数量级接近但差异 30-50%（例如需要 5000 用户，模式设计 10000 用户）
- **40%**：需求与模式规模差异较大，超过 1 个数量级
- **0%**：需求与模式的规模完全不同（例如需要百万级用户，模式只设计万级）

**并发需求匹配** - 评分从 0% 到 100%：
- **100%**：需求的并发用户数、请求频率与模式的设计并发完全对应
- **80%**：需求的并发指标与模式相差 20% 以内，模式的并发能力能覆盖需求
- **60%**：需求的并发指标与模式相差 20-50%，模式能基本支持但需要部分优化
- **40%**：需求的并发指标与模式相差较大，超过 50%，需要调整架构
- **0%**：需求的并发要求与模式的设计并发相差超过 1 倍，完全无法支持

**性能需求匹配** - 评分从 0% 到 100%：
- **100%**：需求的响应时间和吞吐量指标与模式的设计指标一致
- **80%**：需求的性能指标与模式相差 10% 以内，模式能满足需求
- **60%**：需求的性能指标与模式相差 10-30%，模式需要部分优化能满足
- **40%**：需求的性能指标与模式相差超过 30%，模式的基础架构可用但需要显著调整
- **0%**：需求的性能指标远高于模式的设计指标，架构完全无法满足

**可扩展性匹配** - 评分从 0% 到 100%：
- **100%**：需求的业务增长预期和扩展方式与模式的设计完全一致
- **80%**：需求的增长方向与模式支持的扩展方式一致，模式有足够的扩展能力
- **60%**：需求的增长方向与模式支持的扩展方式基本一致，需要部分调整
- **40%**：需求的增长预期与模式的扩展能力有差异，需要重新设计扩展策略
- **0%**：需求的扩展需求与模式的设计方向完全不同，模式不适用

## 输出格式

```
| 系统需求 | 匹配模式 | 匹配评分 | 核心参考点 | 建议 |
|----------|---------|---------|-----------|------|
| IPO 去重清单 | 微服务架构 | 85% | 服务拆分方案直接匹配 | 建议参考 |
```
