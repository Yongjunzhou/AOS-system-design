# SR → BA 映射技能
**Skill: SR to BA Mapping**

**编号**：07
**所属步骤**：第 2 步 — 相关方需求功能部分详细定义及业务架构定义
**引用准则**：系统设计准则 v2.3 — 1:1 分配约束、N:1 承接支持、第 3 层业务架构
**引用指南**：瀑布式设计指南 v2.0 第 2 步

---

## 用途

将相关方需求功能部分详细定义末级节点映射到业务架构 IPO，确保每条需求有唯一的业务承接点。同时同步建立系统需求架构骨架。

## 输入

- 相关方需求详细定义末级节点（第 2 步产出）
- 业务模式匹配结果（06-business-pattern-matching.md 的输出）

## 处理规则

### 1. IPO 识别
根据相关方需求详细定义，识别业务处理组件：
- 每个 IPO 是一个独立的业务处理单元
- IPO 包含：Input（输入）、Process（处理）、Output（输出）
- IPO 类型包括：计划定义、流程定义、工单定义、指标定义等

### 2. 1:1 分配约束
每条相关方需求详细定义末级节点只能分配到**唯一一个** BA IPO。

### 3. IPO 去重
IPO 去重优先级：复用 → 改进 → 新增
- 如果已有 IPO 可以承接，直接复用
- 如果已有 IPO 需要扩展，标注改进内容
- 如果无可用 IPO，创建新的 IPO

### 4. 同步建立下层架构骨架
在定义 IPO 的同时，同步在系统需求层建立预计的 5 级节点占位：
- 为每个 IPO 预分配系统需求 5 级节点
- 只有节点名称和映射关系，不填充详细内容

## 输出格式

### IPO 定义

```
BA-001：[IPO 名称]
├─ 定义：[业务组件的定义]
├─ Input：[具体形式、数据结构、来源]
├─ Process：[规则、算法、步骤、约束条件]
├─ Output：[具体形式、数据结构、验收标准]
├─ 承接的相关方需求：SR-F-001.1, SR-F-001.2
├─ 符合性分析：[如何满足所承接的需求]
└─ 为下层提供的设计指导：[对系统需求的指导]
```

### 同步建立的系统需求骨架

```
SysReq 骨架（占位）：
├─ BA-001 → 预计 SysReq 5 级：[组件实例名称]
├─ BA-002 → 预计 SysReq 5 级：[组件实例名称]
```

## 质量检查

- [ ] 1:1 分配约束满足率 100%
- [ ] 每个 IPO 有清晰的 I/P/O 定义
- [ ] IPO 去重率 100%
- [ ] 系统需求架构骨架与 IPO 清单匹配
