# DevOps 系统设计：AI 自检清单
**DevOps System Design: AI Self-Check List**

**用途**：AI 完成每一步后，使用此清单自行验证产出质量。全部通过后方可提交给流水线开发者审核。

---

## 一、跨步骤通用检查（适用于所有步骤）

### 增量标记正确性
- [ ] 修改条目使用 `~` 前缀标记
- [ ] 新增条目使用 `+` 前缀标记（模式2/3中可能使用）
- [ ] `~` 条目仅列出变更内容（原问题/修改内容/符合性分析/验证方法）
- [ ] `+` 条目格式完整（定义/组件类型/承接关系/符合性分析等全部字段）

### 影响范围完整性
- [ ] 所有受影响的层已识别
- [ ] 所有受影响的节点已标记 `~`（无遗漏）
- [ ] 修改不破坏现有未受影响节点的映射关系
- [ ] 符合性分析已更新（每处修改都必须更新）

### 格式规范性
- [ ] 文档编号格式统一（PA-XXX / SysReq-F-XXX / BA-XXX / SR-F-XXX）
- [ ] 增量标记后的节点编号与现有体系一致
- [ ] 简化条目格式正确（仅列出变更内容，保留符合性分析）

---

## 二、第1步自检：快速影响范围评估

### 问题分类
- [ ] 问题类型已分类（Bug修复/功能需求/性能优化）
- [ ] 紧急程度已标注（P0/P1/P2）
- [ ] 问题摘要清晰地描述了什么在什么情况下出了什么问题
- [ ] 影响范围已逐层评估（PA → SysReq → BA → SR）

### 模式选择
- [ ] 选定的处理模式与影响范围匹配
- [ ] 仅 PA → 模式1
- [ ] PA + SysReq（不影响 BA/SR）→ 模式2
- [ ] 跨多层（涉及 BA 或 SR）→ 模式3
- [ ] 不确定时有备注说明（升级模式的触发条件）

---

## 三、第2步自检：模式1 - 快速修复

### 问题定位
- [ ] 已定位到具体的 PA 组件和条目
- [ ] 根本原因已理解（不限于表面现象）
- [ ] 需修改的字段已明确

### 影响验证（模式1）
- [ ] 分析活动已完成：影响验证
- [ ] 修改不影响该 PA 组件所承接的 SysReq 9 级活动的覆盖完整性
- [ ] 无性能退化风险
- [ ] 树形结构无需调整（如需调整说明理由）

### 修复实施
- [ ] 修复条目使用 `~` 前缀（简化格式）
- [ ] 包含原问题、修改内容、承接SysReq、符合性分析、验证方法
- [ ] 架构定义文档已同步更新（映射关系表中的符合性分析）
- [ ] 详细定义文档已同步更新（实现细节和技术方案）
- [ ] 版本号已更新

---

## 四、第3步自检：模式2 - 局部设计

### 影响分析（模式2）
- [ ] 分析活动已完成：影响范围分析
- [ ] 受影响的 SysReq 5 级节点和 9 级活动已全部识别
- [ ] 受影响的 PA 组件已全部识别
- [ ] SysReq↔PA 映射一致性已验证
- [ ] 确认不需要升级为模式3（BA/SR 不受影响）

### 系统需求修改
- [ ] SysReq 5 级架构末级节点已更新（定义/性能指标/符合性分析）
- [ ] SysReq 9 级详细定义末级已更新（活动步骤/分配到/验收标准）
- [ ] 1:1 分配约束满足（每条9级活动分配到唯一 PA 组件）
- [ ] 架构定义文档已同步更新（树形结构 + 映射关系表）
- [ ] 详细定义文档已同步更新（6-9 级分解链）

### 产品架构同步修改
- [ ] PA 组件已同步更新（实现细节与 SysReq 修改匹配）
- [ ] 同步设计原则已遵循（SysReq 修改 ↔ PA 修改双向一致）
- [ ] 架构定义文档已同步更新（树形结构 + 映射关系表）
- [ ] 详细定义文档已同步更新（技术实现细节）
- [ ] 性能指标不低于 SysReq 要求

---

## 五、第4步自检：模式3 - 完整设计

### 跨层影响分析
- [ ] 分析活动已完成：快速跨层影响分析
- [ ] 所有受影响的层已识别（SR/BA/SysReq/PA）
- [ ] 受影响节点链已建立（SR → BA → SysReq → PA）
- [ ] 变更类型已明确（~修改 / +新增 / 混合）

### 按层修改（从高到低）
- [ ] SR-F 修改已完成（功能需求架构定义 + 详细定义 + 映射关系表）
- [ ] SR-NFR 修改已完成（非功能需求架构定义 + 详细定义 + 含需求类型/需求类别字段）
- [ ] BA 修改已完成（IPO 定义 + 映射关系表 — BA 无详细定义）
- [ ] SysReq-F 修改已完成（功能需求架构定义 + 详细定义 + 映射关系表）
- [ ] SysReq-NFR 修改已完成（非功能需求架构定义 + 含类型/验证方法字段）
- [ ] PA 修改已完成（架构定义 + 详细定义 + 映射关系表）
- [ ] 同步设计一致性已满足（上层修改驱动了下层调整）
- [ ] 更新顺序正确（SR → BA → SysReq → PA）

### 验证（模式3）
- [ ] 1:1 分配约束满足率 100%
- [ ] N:1 承接数量合理
- [ ] 双向追溯完整（正向 OR→PA，反向 PA→OR）
- [ ] 符合性分析完整率 100%
- [ ] 影响范围涉及的所有层级均已修改
- [ ] 每层文档的架构定义和详细定义同步更新
- [ ] 性能回退检查通过

---

## 六、跨模式最终检查

### 回归验证（所有模式通用）
- [ ] 问题是否完全解决（修复条目与问题描述对应）
- [ ] 是否有副作用（未影响未修改的现有功能）
- [ ] 性能是否在 SysReq 指标范围内（或更新后的指标范围内）
- [ ] 映射关系无断裂（修改不影响上下游追溯）
- [ ] 所有 `⚠️ 待确认` 标记已列出

### 代码-文档一致性
- [ ] 修改内容的实现细节与代码方案一致
- [ ] 新增组件的接口定义完整
- [ ] 性能指标有实现依据

### 版本管理
- [ ] 所有修改过的文档版本号已更新且一致
- [ ] 变更日志已更新（包含问题描述、修改内容、涉及节点）
- [ ] 变更类型已标注（快速修复/局部设计/完整设计）
