## 项目结构说明

每个项目包含以下文件：

```
project-a/
├── requirements.md                    # 规范化和分解后的相关方需求
├── business-architecture.md           # 业务架构
├── system-requirements.md             # 系统需求
├── product-architecture.md            # 产品架构
├── mappings.md                        # 映射关系矩阵
├── analysis.md                        # 需求分析记录（原始需求、规范化、分解、冲突、重复、分类、确认）
└── changelog.md                       # 版本变更日志
```
