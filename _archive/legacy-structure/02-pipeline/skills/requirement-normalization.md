# Skill: 需求规范化

## 目的
将原始需求转化为规范化的需求，确保需求的完整性、清晰性和可测试性。

## 输入
- 原始需求文本（邮件、会议记录、需求文档等）
- 相关的背景信息

## 输出
- 规范化的需求列表
- 规范化过程中的决策记录

## 规范化规则

### 1. 需求完整性检查
- [ ] 需求是否有明确的描述
- [ ] 需求是否包含背景或原因
- [ ] 需求是否包含验收标准
- [ ] 需求是否包含优先级

### 2. 需求清晰性检查
- [ ] 需求是否避免了模糊表述（如"尽可能"、"大约"等）
- [ ] 需求是否使用了统一的术语
- [ ] 需求是否明确了相关方
- [ ] 需求是否明确了约束条件

### 3. 需求可测试性检查
- [ ] 需求是否包含可测试的验收标准
- [ ] 验收标准是否具体和可量化
- [ ] 是否能够通过自动化测试验证

### 4. 需求独立性检查
- [ ] 需求是否相对独立
- [ ] 需求之间是否存在过度耦合
- [ ] 是否需要进行分解

## 规范化模板

```markdown

| 项目 | 内容 |
|------|------|
| 原始表述 | [原始需求文本] |
| 规范化表述 | [规范化后的需求] |
| 相关方 | [相关方] |
| 优先级 | [P0/P1/P2/P3] |
| 背景/原因 | [背景或原因] |
| 验收标准 | [验收标准] |
| 约束条件 | [约束条件] |
| 规范化决策 | [规范化过程中的决策] |
```

## 使用场景
- 新增需求的初始处理
- 需求文档的标准化
- 需求的质量检查
